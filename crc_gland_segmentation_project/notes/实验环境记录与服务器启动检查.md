# 实验环境记录与服务器启动检查

> 文件性质：实验环境的正式记录和服务器重启后的执行入口说明。  
> 适用范围：GlaS/CRAG 训练、测试、评估、runtime check 和阶段 gate。  
> 当前正式 Python：`/environment/miniconda3/bin/python`

## 1. 固定依赖

当前依赖清单位于项目根目录的 `requirements.txt`，其中关键版本为：

```text
torch==2.2.2
numpy==1.26.4
scipy==1.17.1
pillow==10.3.0
pyyaml
```

`scipy` 是正式评估链的必需依赖，原因包括：

- `scipy.ndimage.label` 用于 8-connectivity 连通域提取；
- 对象级 F1、Object Dice、Boundary F1 和错误案例导出依赖该连通域实现；
- 缺失 `scipy` 时不能开始训练、测试或阶段汇总。

## 2. 每次租用服务器后的固定顺序

服务器环境重新建立后，先执行：

```bash
cd /home/featurize/work/Paper/crc_gland_segmentation_project
/environment/miniconda3/bin/python -m pip install -r requirements.txt
/environment/miniconda3/bin/python tools/stage02_experiment_environment_check.py
```

只有看到：

```text
environment_status=pass
```

才允许执行训练、测试、评估和 gate。

环境检查报告写入：

```text
reports/environment/experiment_environment_check.md
```

如果输出 `environment_status=blocked`，必须先根据报告中的 blocker 安装或修复依赖，不能继续跑实验。

## 3. 自动阻断

以下入口在导入项目模型和评估模块前会自动调用环境检查：

- `scripts/train.py`
- `scripts/test.py`

因此，即使忘记手动执行环境检查，正式训练和测试也会在缺依赖时立即停止，并提示：

```bash
/environment/miniconda3/bin/python -m pip install -r requirements.txt
```

这只是最后一道保护；推荐仍然先执行 `tools/stage02_experiment_environment_check.py`，让环境状态形成独立报告。

## 4. 版本与可复现边界

- 版本严格固定的依赖必须按 `requirements.txt` 安装。
- 没有固定版本的 `pyyaml` 至少必须可安装并成功导入。
- 环境报告记录当前实际安装版本，但不修改实验结果文件。
- 服务器重启后如果虚拟环境被重置，必须重新执行安装命令和环境检查。
- 环境检查通过只表示依赖可用，不表示数据协议、训练、评估或阶段 gate 已通过。

## 5. 与实验门禁的关系

```text
服务器环境安装
-> environment_status=pass
-> smoke/runtime check
-> code quality gate
-> 正式训练或测试
-> 结果独立核查
-> 阶段 gate
```

缺少环境检查报告或自动环境检查失败时，当前正式实验状态必须为 `blocked`。
