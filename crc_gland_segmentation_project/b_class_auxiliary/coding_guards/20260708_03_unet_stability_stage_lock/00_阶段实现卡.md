## 3. 当前阶段唯一目标
- 用一句话说清: 在已严格通过的 `02_UNet流程验证` 之上,只锁定 A2_UNet_GlaS_v1 三次重复(seed3407、seed1234、seed2025)这一条稳定性验证任务,唯一允许变化的变量是 train_seed,产出 TestA 与 TestB 分开的 raw per-seed 表与 mean±std 聚合表,不打开 baseline、模块增强或跨数据集扩展
- 当前阶段进入条件: 研究定标记录已形成,研究结论状态为 `allow_stage_lock`,且 research gate 已经通过;上游 A1 已严格通过并冻结,`crc_gland_segmentation_project/experiments/A1_UNet_GlaS_v1_seed3407/run_meta.yaml` 记录 metric_crosscheck_result 为 pass
- 上一阶段已经交付: 研究阶段已明确 A2 唯一变量、必须继承的 A1 冻结项、sample-only raw 与统计派生 schema、Gate_A2 门槛、当前工程真实缺口以及暂不能直接进入编码与运行的原因,并已形成 allow_stage_lock 结论且通过 research gate,因此当前允许进入阶段锁定

## 4. 为什么现在做这个,而不是下一个阶段
- 当前阶段必须先解决的阻断: 目前还没有把 A2 的唯一变量、允许改动、禁止改动、工程落点和最小运行验证计划写成正式卡片;如果不先锁定,后续 Pre-check 和编码就会在“重复实验”名义下悄悄改动增强、阈值、optimizer 或 batch_size,造成协议漂移
- 如果现在提前做下一个阶段,会破坏什么: 会破坏 `03_UNet稳定性` 作为“同结构同协议下随机波动量化”阶段的角色,把稳定性统计与 `04_Baseline` 的 ResNet34-U-Net、更强模块阶段混成一件事,使 mean±std 结论失去可比性
- 下一个阶段此刻还不允许进入的原因: baseline 与后续更强模型必须以 A2 已成立的稳定基线为对照锚点;在 A2 三次重复未产出 mean±std 并通过 Gate_A2 之前进入 baseline,会失去公平比较的统计基准

## 5. 当前阶段允许改动 / 禁止改动
- 明确允许改: 只允许在后续编码阶段新增三个 A2 experiment 配置（配置路径可含 `v1`，正式 run_name 只允许是 `A2_UNet_GlaS_seed3407`、`A2_UNet_GlaS_seed1234`、`A2_UNet_GlaS_seed2025`，唯一变量为 train_seed），扩展阶段汇总脚本以支持 sample-only raw、七项指标、raw=42 和可选 mean±std=14 派生统计；本轮阶段锁定本身只允许新增当前阶段卡和阶段门禁报告
- 明确禁止改: 不允许改动 GlaS split、数据口径、mask 前景规则、input_size、归一化、light_aug_v1、BCE 与 Dice 组合、AdamW 与学习率与 scheduler、epoch_max、early_stop、batch_size、best_selector=val_objdice_max、threshold_source=val17、eval_cast_policy、boundary_metric_width=3、连通域实现与 UNet 结构;不允许提前进入 `04_Baseline`、`05_LKMA`、`06_Boundary`、`07_Distance`;不允许把某个难看 seed 删除或替换后再汇总
- 如果越界,会破坏哪份正式协议: 会破坏 `结直肠腺体分割_plan_优化版/01_实验执行/03_UNet稳定性/00_阶段总协议.md` 对唯一变量与冻结继承的定义,也会破坏 `结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/04_评估口径与官方脚本对齐.md` 对验证集选模与阈值冻结的正式约束

## 9. 当前未决问题与阻断项
- 当前仍未解决: 三次正式 run 是否在同一 GPU 环境连续执行以保证 cudnn 与 AMP 语义一致的具体执行留痕方式;某个 seed 若出现极端偏离时判为“真实波动”还是“需回退”的量化边界;异常 run 重跑时的新目录命名与作废标签写法,这些都需在 Pre-check 中明确
- 如果这些问题不解,后续会卡在哪: 会卡在 Pre-check 之后的三次正式训练与聚合阶段,因为没有先写清执行环境、异常裁决边界与命名规则,就无法稳定产生可交接的 mean±std 稳定性证据
- 本轮是否允许继续进入 Pre-check: `是`;前提是当前阶段卡通过 `b_class_auxiliary/tools/check_stage_definition_gate.py`,因为阶段锁定的职责是把边界锁死,而不是本轮就开始编码或训练
