# 04_保留或降级标准

本文件不是给 `Distance-aware Loss v1` 补一段“结果看起来还有点帮助”的说明，也不是把现有裁决条目整理成更顺眼的笔记。

它在当前项目中的唯一职责是：

> 作为 `07_Distance` 的正式 decision protocol，把 `结直肠腺体分割_plan_优化版\01_实验执行\07_Distance\03_实验步骤.md` 已冻结的执行链、`结直肠腺体分割_plan_优化版\01_实验执行\07_Distance\02_距离图与lambda范围.md` 已冻结的 target 与 `lambda_dist` 边界、`结直肠腺体分割_plan_优化版\01_实验执行\07_Distance\00_阶段总协议.md` 已冻结的 `Gate_E1` 与 handoff 语义，压成唯一合法的 `keep / downgrade / drop` 判定规则；从现在开始，`Distance-aware Loss v1` 是否进入正式主线、是否只能保留为补充实验、何时必须回退到 `distance_input_base`，都只能由这里写明的证据顺序、主版本优先门、止损边界、独立回退条件和交接字段决定，不能再靠单次最好值、局部好看案例或“理论上可能有帮助”的主观偏好改判。

从现在开始，只要讨论 `Distance` 阶段“什么时候才允许写正式结论、哪些证据优先级最高、主版本与备选版本怎样分层、证据不足时为什么默认降级、以及 `08_外部对比 / 09_CRAG验证` 到底从哪里起步”，都必须以本文件为准；若与同层文件冲突，以 `结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/00_阶段总协议.md` 管阶段身份与 `Gate_E1`，以 `结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/01_设计依据.md` 管为什么距离监督值得单独验证，以 `结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/02_距离图与lambda范围.md` 管 target 与 `lambda` 边界，以 `结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/03_实验步骤.md` 管执行顺序，而本文件负责把这些上游冻结内容压成最终可比较、可止损、可交接的正式裁决逻辑。

## 1. 文件角色与执行边界

### 1.1 当前文件负责什么

当前文件只负责冻结下面八件事：

1. `Distance` 阶段的正式结论最早在什么证据条件下才允许书写。
2. 正式证据必须按什么顺序读取，避免被单个数字、单张图或单个 split 带偏。
3. 什么情况下可以把 `Distance-aware Loss v1` 判为 `keep` 并正式进入后续主线。
4. 什么情况下只能降级为 `downgrade`，而不能冒充正式主线保留。
5. 什么情况下必须直接 `drop`，并停止继续为 `Distance` 扩大搜索边界。
6. 主版本与备选版本之间必须怎样执行“主版本优先门”，避免备选值反客为主。
7. 阶段结论必须输出哪些正式字段，才能被 `Gate_E1`、`08_外部对比` 和 `09_CRAG验证` 稳定消费。
8. 一旦结论不成立，应按什么独立回退条件回到 `distance_input_base`，而不是让下游阶段替当前文件兜底。

### 1.2 当前文件不负责什么

当前文件不重新定义下面这些内容：

- `Distance` 为什么值得做，以及为什么它必须排在 `Boundary` 之后。
- 距离图唯一来源、`Euclidean distance map + norm01`、`lambda_dist` 主版本与备选边界。
- `screening_main -> three_seed_main -> optional_one_backup -> stage_decision` 的执行顺序。
- 数据身份、split、输入尺寸、主 gland 标签定义、训练协议和评估协议本体。
- `08_外部对比`、`09_CRAG验证` 与更后续阶段的结构、训练和结果表规则。

这些职责分别以下列文件为准：

### 1.3 为什么当前文件必须独立存在

因为 `Distance` 阶段最容易出现的伪完成状态不是“完全没写结论”，而是：

- 口头上说只看主版本三 seed，实际却拿 screening run 或备选值单次更好结果强行保留。
- 口头上说优先看边界级和几何级证据，实际却只挑对象级某个更顺眼的数字。
- 口头上说主版本不过线就止损，实际却不断补更多 `lambda` 或偷偷重开 target 定义。
- 口头上说已经能交接到 `08 / 09`，实际却没有把 `decision_level / decision_reason / current_mainline` 写成正式字段。

如果没有一份独立裁决文件，`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/03_实验步骤.md` 会被迫替本文件承担“到底算不算保留”的职责，`08_外部对比` 与 `09_CRAG验证` 会被迫自行决定该从 `distance_kept_base` 还是 `distance_input_base` 起步，而 `Gate_E1` 也就无法真正形成可审计的阶段闭环。

## 3. 判定对象、继承前提与禁止重开

### 3.1 当前文件唯一判定对象

- 当前结论：本文件唯一判定对象是“在 `current_base` 协议完全不动的前提下，已经按固定执行链完成的 `Distance main_variant = current_base + DistanceLoss(euclidean_norm01, lambda=0.1)` 是否值得进入后续主线，以及唯一合法备选版本是否只能作为辅助解释而不能取代主版本结论”。
- 规则类型：`工程冻结规则`
- 适用阶段：`07_Distance`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/00_阶段总协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/02_距离图与lambda范围.md`、`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/03_实验步骤.md`
- 核心公式或定义参考：`decision = f(boundary_metric_trend, geometry_metric_trend, object_metric_trend, seed_consistency, split_consistency, visual_support, cost_change, main_version_priority_pass)`
- 采用原因：只有把判定对象收缩为“已完成正式主链的 Distance 主版本”，后续保留/降级/删除结论才具有单变量与可追溯性。
- 不采用的相邻方案：不把本文件写成执行步骤文件；不把本文件写成 target 协议文件；不把任何未授权扩表变体纳入正式主结论。
- 代码落点：[project_root]/scripts/compare_runs.py、[project_root]/scripts/summarize_stage.py
- 运行记录字段：`decision_level`, `decision_reason`, `seed_consistency`, `split_consistency`, `distance_visual_support`, `cost_assessment`, `main_version_priority_pass`
- 验收方式：检查正式结论只围绕主版本三 seed 的正式资产形成；备选版本若存在，也只能作为辅助裁决输入。

### 3.2 当前文件直接继承哪些冻结项

### 5.5.1 量化参考定义规则

- 当前结论：`keep / downgrade / drop` 虽然不是僵硬的单阈值裁决，但必须至少满足统一的 `seed_support_count + mean_table_direction + split_consistency` 量化参考，不能只靠口头“看起来更好”。
- 规则类型：`量化参考规则 + 裁决解释规则`
- 适用阶段：`Distance` 正式裁决。
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/03_实验步骤.md`、04_Metrics-Reloaded.md
- 核心公式或定义参考：

```text
seed_support_count(metric) = number of seeds whose delta(metric) supports Distance
stable_boundary_gain := seed_support_count(BoundaryF1) >= 2 and mean_delta(BoundaryF1) > 0
stable_geometry_support := [seed_support_count(-HD95) >= 2] or [seed_support_count(-ObjectHausdorff) >= 2]
material_object_regression := [mean_delta(F1) < 0 and mean_delta(ObjectDice) < 0] or object_regression_seed_count >= 2
split_consistency := TestA and TestB do not show direct reversal on main conclusion direction
```

- 采用原因：当前阶段需要的是“可解释但不飘”的量化参考，而不是把三档结论变成完全主观裁决。
- 不采用的相邻方案：不把 `keep` 写成纯视觉印象；不强行发明单一固定百分点阈值覆盖所有指标；不允许无 seed 支持计数的自由裁决。
- 代码落点：[project_root]/scripts/compare_runs.py、[project_root]/scripts/summarize_stage.py
- 运行记录字段：`boundary_support_seed_count`、`geometry_support_seed_count`、`object_regression_seed_count`、`split_consistency`、`mean_table_direction`
- 验收方式：检查正式 `decision_note` 至少写清 seed 支持数、mean 表方向和 `TestA / TestB` 是否一致，而不是只给一句综合判断

### 5.6 直接保留的判定条件

- 当前结论：只有当主版本在固定三 seed 下形成稳定边界级正向趋势、几何级至少一项形成明确支持、对象级不出现材料性退化、局部轮廓可视化支持明确、成本增加仍可接受时，才允许判为 `keep`。
- 规则类型：`工程冻结规则`
- 适用阶段：`Distance` 裁决输出。
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/00_阶段总协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/01_设计依据.md`、`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/03_实验步骤.md`
- 核心公式或定义参考：`keep = stable_boundary_gain and stable_geometry_support and no_material_object_regression and visual_support_clear and cost_acceptable and main_version_priority_pass`
- 采用原因：`keep` 代表后续 `08 / 09` 的正式主线，因此必须建立在稳定、可解释且可复现的收益上。
- 不采用的相邻方案：不因为单个 seed 很好就直接保留；不因为只有可视化变好就直接保留；不因为理论上合理或已经投入较多就默认保留。
- 代码落点：[project_root]/scripts/summarize_stage.py、[project_root]/reports/stage_reports/distance_decision_note.md
- 运行记录字段：`decision_level = keep`, `decision_reason`, `distance_visual_support`, `cost_assessment`, `current_mainline`
- 验收方式：检查 `decision_note` 中明确写出三 seed 趋势、边界级依据、几何级依据、对象级依据、案例依据和成本依据。

### 5.7 降级为补充实验的判定条件

- 当前结论：若边界级或几何级存在一定正向迹象，但收益接近波动范围、跨 seed 或跨 split 支持不够强、或对象级未形成足够强的同步支持，则只能判为 `downgrade`。
- 规则类型：`工程冻结规则`
- 适用阶段：`Distance` 裁决输出。
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/01_设计依据.md`、`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/03_实验步骤.md`、04_Metrics-Reloaded.md
- 核心公式或定义参考：`downgrade = partial_positive_signal and no_clear_mainline_confidence`
- 采用原因：`downgrade` 的意义是承认模块可能有解释价值，但当前证据强度不足以进入正式主线。
- 不采用的相邻方案：不把局部边界改善但主证据不稳的版本升格为 `keep`；不把证据不清的情况直接写成已删除。
- 代码落点：[project_root]/scripts/compare_runs.py、[project_root]/reports/stage_reports/distance_decision_note.md
- 运行记录字段：`decision_level = downgrade`, `downgrade_reason`, `uncertainty_source`
- 验收方式：检查 `decision_reason` 明确写出“为何不足以 keep 但又不直接 drop”。

### 5.8 直接删除的判定条件

- 当前结论：若边界级或几何级收益只出现在单个 seed、`mean +- std` 不支持稳定优势、对象级主指标出现明确退化、可视化不支持、或成本明显上升但无对应收益，则必须判为 `drop`。
- 规则类型：`工程冻结规则`
- 适用阶段：`Distance` 裁决输出。
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/00_阶段总协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/03_实验步骤.md`、04_Metrics-Reloaded.md
- 核心公式或定义参考：`drop = no_stable_gain or material_regression or unsupported_visuals or unjustified_cost`
- 采用原因：若这种情况仍继续保留，后续所有阶段都会建立在不稳定增益之上，污染主线解释与后续比较。
- 不采用的相邻方案：不在主版本明确不过线时继续无限扩 `lambda` 或 target 定义；不靠更多小修小补为删除结论找借口。
- 代码落点：[project_root]/scripts/summarize_stage.py、[project_root]/reports/stage_reports/distance_decision_note.md
- 运行记录字段：`decision_level = drop`, `drop_reason`, `rollback_target`, `current_mainline`
- 验收方式：检查删除结论后不再追加未授权扩表动作，并明确写出回退起点。

### 5.9 “边界涨、对象级不跟”裁决规则

- 当前结论：若 `Boundary F1` 提升，但 `Object Dice / F1` 没有跟上或 `Object Hausdorff` 不支持，当前版本不得直接 `keep`，最多只允许 `downgrade`。
- 规则类型：`工程冻结规则`
- 适用阶段：`Distance` 冲突证据场景。
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/01_设计依据.md`、`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/03_实验步骤.md`
- 核心公式或定义参考：`boundary_gain_with_weak_object_support -> not_keep`
- 采用原因：这说明模块可能只带来局部边界价值，但整体证据不足以进入正式主线。
- 不采用的相邻方案：不因边界指标变好就忽略对象级缺口；不在对象级支撑不足时继续写成主线模块。
- 代码落点：[project_root]/scripts/compare_runs.py
- 运行记录字段：`boundary_f1_delta`, `objdice_delta`, `objf1_delta`, `objhaus_delta`
- 验收方式：检查任何 `keep` 结论都没有出现“边界涨但对象级无支撑”的冲突模式。

### 5.10 “对象级略涨但边界/几何证据不清”裁决规则

- 当前结论：若对象级主指标略有提升，但 `Boundary F1`、`HD95 / Object Hausdorff` 和局部轮廓可视化都不提供清晰支持，则不得直接 `keep`，通常只能 `downgrade`。
- 规则类型：`工程冻结规则 + 评价解释规则`

- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/01_设计依据.md`、02_Distance-Map-Loss.md
- 核心公式或定义参考：`object_gain_without_boundary_or_geometry_support -> not_keep`
- 采用原因：这与当前模块“首先作用在边界邻域连续性”的理论职责不匹配。
- 不采用的相邻方案：不只凭对象级轻微上升就宣布距离监督保留；不让对象级单项数字覆盖主职责证据。
- 代码落点：[project_root]/scripts/compare_runs.py
- 运行记录字段：`objdice_delta`, `objf1_delta`, `boundary_f1_delta`, `hd95_delta`, `distance_visual_support`
- 验收方式：检查任何 `keep` 结论都同时具备边界/几何级支撑，而不是只具备对象级轻微上升。

### 5.11 “图更好看但指标不支持”裁决规则

- 核心公式或定义参考：`visual_only_without_metric_support -> not_keep`
- 采用原因：可视化是关键解释证据，但不能替代跨 seed 的正式统计证据。
