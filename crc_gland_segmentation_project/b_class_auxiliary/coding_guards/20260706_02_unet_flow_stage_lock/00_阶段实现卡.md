## 3. 当前阶段唯一目标
- 用一句话说清: 在已放行的 `01_数据协议` 之上，只锁定 A1_UNet_GlaS_v1_seed3407 这一条标准 UNet 首版训练与评估闭环，不打开稳定性统计、baseline、模块增强或 CRAG 扩展
- 当前阶段进入条件: 研究定标记录已形成，研究结论状态为 `allow_stage_lock`，且 research gate 已经通过
- 上一阶段已经交付: 研究阶段已经把当前唯一目标、必须继承的数据和评估冻结项、当前工程真实缺口、为何不能直接进入编码与运行说清，因此当前允许进入阶段锁定

## 4. 为什么现在做这个,而不是下一个阶段
- 当前阶段必须先解决的阻断: 目前还没有把 A1 的唯一目标、允许改动、禁止改动、工程落点和最小运行验证计划写成正式卡片；如果不先锁定，后续 Pre-check 和编码就会在边界漂移下进行
- 如果现在提前做下一个阶段,会破坏什么: 会破坏 `02_UNet流程验证` 作为“流程排雷 + 最小可复现基线”阶段的角色，把 stage01 预飞入口、A1 首版协议、A2 稳定性统计和更强模型阶段混成一件事
- 下一个阶段此刻还不允许进入的原因: Pre-check 之前必须先有一张正式阶段卡，把唯一目标、允许改动、禁止改动、依据、工程落点和最小运行验证计划写死；否则 Pre-check 只能检查一组未锁定的口头假设

## 5. 当前阶段允许改动 / 禁止改动
- 明确允许改: 只允许在后续编码阶段新增或更新标准 UNet 模型实现、seg loss 模块、engine 模块、eval 模块、object metrics 模块、train/model/eval/experiment 配置集合，以及训练、测试、可视化、阶段汇总脚本；本轮阶段锁定本身只允许新增当前阶段卡和阶段门禁报告
- 明确禁止改: 不允许重开 split、数据口径、mask 前景规则、input_size、ImageNet mean/std、best_selector、threshold_source；不允许提前进入 `03_UNet稳定性`、`04_Baseline`、`05_LKMA`、`06_Boundary`、`07_Distance`；不允许本轮进入 Pre-check 之后的编码、运行证据、代码质量和说明文档
- 如果越界,会破坏哪份正式协议: 会破坏 `结直肠腺体分割_plan_优化版/01_实验执行/02_UNet流程验证/00_阶段总协议.md` 对 A1 角色和 A2 交接边界的定义，也会破坏 `结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/04_评估口径与官方脚本对齐.md` 对验证集选模与阈值冻结的正式约束
