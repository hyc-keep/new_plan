# 06_数据产物清单

## 文档身份与当前状态
A 类数据资产交接协议。`status=planned`、`handoff_gate=blocked`、`formal_result=not_run`、`result_eligibility=false`。清单中的“必须交付”是计划要求，不代表当前资产已存在或通过。

## 本轮重写直接依赖的前置文件
active 00-07规范；数据协议00-05、07；路线02-03；文献 GlaS/MILD-Net/TA-Net；active project 的 `splits`、`src/data`、`reports` 和运行资产索引。旧计划/旧工程仅作 provenance。

## 1. 文件角色与执行边界（角色边界）

本文件冻结交接资产类别、schema、digest、消费者和消费边界；不生产数据、不重开 split 或标签规则。

## 2. 本轮直接依赖的前置文件（前置依赖）

前置为 active 00-07、数据协议00-05/07、路线/文献核心文件和 active project 资产索引。

## 3. 本阶段唯一允许处理的变量（固定参数与变量边界）

六类资产和必填字段固定；仅报告文件名和压缩格式可调，缺失资产必须阻断 handoff。

## 4. 阶段门控表达式（数据阶段门控/通过线）

source、split、check、label、preview、manifest/config均有真实资产且互相绑定时才可 accepted；当前为 planned/blocked。

## 角色边界
本文件登记 source、split、check、label、preview、manifest/config 六类资产及其交接关系，不生产数据或改变协议。

## 前置依赖
资产清单依赖 00-05 的输入输出契约和 07 验收规则；active project 文件只有经 manifest 绑定后才能进入 handoff。

## 变量或边界
固定六类资产、必填 schema、七字段 lineage、消费者和 blind 边界；只允许变更文件版本与可追溯 digest。

## 门控/通过线
六类资产均有真实路径、schema、sha256、producer、consumer 和消费边界才可 handoff；当前 planned/blocked。

## 上游、同层与下游
- 上游：数据源、检查、GlaS/CRAG split、标签/预览协议，决定清单节点和版本。
- 同层：00聚合、07验收；02-05负责生产清单所登记的各类资产。
- 下游：02-04训练、06 val筛查、07/08正式测试、10汇总；只消费通过 manifest/handoff 的资产。

## 路线依据
数据阶段交付后才允许进入02；路线继续按01-11执行。TestA/TestB/CRAG test盲态，val只做选择。清单不允许扩展模型或指标。

## 文献依据
GlaS/CRAG文献只决定 benchmark身份、split和对象级任务背景；资产清单是工程可追溯实现，不把文献数字当项目资产。

## 固定、可调、禁止参数
- 固定六类资产：source、split、check、label、preview、manifest/config。每类必须有路径、schema、protocol_version、sha256和消费边界。
- 可调：报告文件名、压缩格式、manifest生成时间字段；不得省略必填字段。
- 禁止：只交CSV、目录存在即交接、手工填pass、把历史运行资产作为当前数据包、test标签进入筛选。

## 输入/输出资产 schema 与消费边界
| 资产类 | planned/current schema | 当前状态 | 消费者 |
|---|---|---|---|
| source | dataset/version/root/source_note/digest | planned_not_created或existing_unverified | 所有阶段只读来源 |
| split | `sample_id,image_relpath,mask_relpath,source_partition,grade或source_subset,split,blind_state` | 现存CSV为existing_unverified | train/val/test按边界 |
| check | sample级六步结果、统计、人工抽查 | planned_not_created | 07和preflight |
| label | mask_rule,dtype,resize,target_identity,sha256 | planned_not_created | 02-08按协议 |
| preview | image/mask/overlay + sample_id回指 | planned_not_created | QC，不调test |
| manifest/config | asset list、schema、digest、consumer、lineage | planned_not_created | Gate/handoff |

总 manifest必须包含 `asset_id,asset_type,path,status,sha256,schema_version,producer,consumer,consumption_boundary,blind_state`。当前正式交接状态只能 `planned/blocked`。

## 代码落地接口
- current/existing：active project 的 `src/data/csv_loader.py`、`src/data/datasets.py`、`scripts/train.py`、`scripts/test.py`、`src/eval/run_eval.py`，只能作为潜在消费者。
- planned_not_created：scripts/build_manifests.py、scripts/validate_handoff.py、planned_not_created:reports/gates/data_stage_gate.json、planned_not_created:configs/data_protocol.yaml；planned artifact directory reports/gates。
- historical provenance：旧工程清单/入口不可作为 current producer或consumer。

## 运行记录字段与 lineage
每个资产和交接记录包含 `source_stage/source_manifest/source_protocol_version/source_run_name/consumer_stage/consumer_file/consumption_boundary` 及 `asset_id,asset_type,path,status,planned_or_actual,protocol_version,schema_version,sha256,producer,consumer,blind_state,formal_result,result_eligibility`。当前 `source_run_name=not_run`、路径未创建资产标 `planned_not_created`。

## 验收
逐类确认文件可读、schema完整、digest可重算、路径属于 active root、消费者和边界匹配、Test blind、manifest与config一致。任何一类缺失为 `handoff_gate=blocked`，不允许进入02。

## 独立回退条件
manifest缺节点、digest不一致、split/label版本混用、路径越界、消费者未登记、盲态破坏、状态与实际文件冲突时，回退到最早缺失的源/检查/split/标签层，重建清单；不得手工改Gate。

## 冲突裁决记录
- 现存 active project 文件只标 existing_unverified，不能绕过当前 manifest。
- `planned_not_created` 不等于空缺被忽略，而是明确阻断下游。
- 旧计划/旧工程资产不进入 current manifest。

## 字段级验收清单
- 核对 source、split、check、label、preview、manifest/config 六类资产各有 asset_id、status、schema_version、sha256 和 consumer。
- 核对每个资产都有七字段 lineage，且 consumption_boundary 具体到 train/val 或 blind test。
- 核对 planned_not_created 与 existing_unverified 的状态和路径语义一致，不以目录存在代替交付。
- 核对 GlaS 数量、CRAG 数量、mask>0、CSV schema、报告 hash 和 blind 字段均能回指上游协议。
- 核对 02-04 只消费 train/val，06 只消费 val，07/08 只消费冻结后的 blind test。
- 核对 manifest/config 绑定的 protocol_version、split_sha256、asset_manifest_sha256 一致。
- 核对缺失资产触发 handoff_gate=blocked，并将 next_action 指向最早缺失层。
- 核对清单不会生成数据、启动训练或把 checker pass 写为数据验收。
- 核对每个 planned_not_created 节点均有 producer、consumer、next_action 和阻断原因。

## 复核动作
- 复核六类资产均有 producer、consumer、schema、sha256 和状态。
- 复核七字段 lineage 与 consumption_boundary 不缺失。
- 复核 planned_not_created 节点有阻断原因和 next_action。
- 复核 GlaS/CRAG 目标数量和 blind 字段来自当前协议。
- 复核 handoff_gate 只能由真实资产证据改变。
- 复核清单不生成数据、不启动训练、不替代 Gate。
- 复核 handoff 失败时 next_action 指向最早缺失的 source、check 或 manifest 节点。
