# 01_数据协议 操作说明（如何运行本阶段）

> 目的：让任何人（包括第二次重做的你）不看源码也能把 01 阶段完整跑一遍。
> 所有命令都在项目根目录执行：`cd /home/featurize/work/Paper/crc_gland_segmentation_project`
> 本阶段脚本全部带默认参数，**绝大多数命令直接运行、不需要传参**。

## 一、这个阶段在做什么

把原始 GlaS / CRAG 数据整理成正式的「数据协议」：切分 split、二值化掩码、派生边界/距离目标、校验数据配对、生成数据验收报告，并让训练入口 `scripts/train.py` 能做一次「入口预飞（preflight）」。

本阶段**不训练模型**，产物是：`splits/`（切分 CSV）、`reports/data_checks/`（校验报告）、`reports/stage_reports/data_stage_acceptance.md`（验收）、`reports/stage_reports/asset_manifest.json`（资产清单）。

## 二、先读哪些文件（推荐阅读顺序）

1. `00_交付范围内正式对象清单.md` —— 本阶段正式交付对象
2. `README.md` —— 阶段总览
3. `implementation_status.md` —— 当前状态与诚实边界
4. `当前阶段为什么能pass以及下一步怎么看.md` —— 通过依据
5. 各脚本逐文件说明：`tools_prepare_glas_split.py.md` 等（想看某个脚本细节再点）

## 三、运行顺序总览

```
prepare_glas_split / prepare_crag_split   （切 split）
        ↓
convert_masks                             （二值掩码协议校验）
        ↓
check_dataset_pairs                       （图像/掩码配对校验）
        ↓
build_boundary_targets / build_distance_targets  （派生目标样例）
        ↓
preview_dataset_samples                   （抽样预览，人工核对）
        ↓
validate_data_assets                      （汇总验收，出 data_stage_acceptance）
        ↓
train.py --runtime-check                  （训练入口预飞，非真正训练）
```

## 四、逐脚本命令（含参数说明）

所有脚本位于 `tools/stage01_data_protocol/`。每个参数都有默认值，下面「最简命令」可直接运行；「可选参数」仅在需要改输出路径时用。

### 1. 切分 split

```bash
python tools/stage01_data_protocol/prepare_glas_split.py
python tools/stage01_data_protocol/prepare_crag_split.py
```
- `--project-root`：项目根，默认自动推断，一般不用传。
- `--config`：数据配置，GlaS 默认 `configs/data/glas.yaml`，CRAG 默认 `configs/data/crag.yaml`。
- `--output-dir`：split 输出目录，默认 `splits/glas`（或 `splits/crag`）。
- `--report`：切分报告，默认 `reports/data_checks/glas_split_report.md`（或 crag）。
- 产物：`splits/glas/glas_{train68,val17,testA60,testB20}.csv`、`splits/crag/crag_{train153,val20,test40}.csv`。

### 2. 二值掩码协议校验

```bash
python tools/stage01_data_protocol/convert_masks.py
```
- `--project-root`：默认自动推断。
- `--summary-output`：默认 `reports/data_checks/binary_mask_summary.csv`。
- `--report`：默认 `reports/data_checks/label_protocol_report.md`。

### 3. 图像/掩码配对校验

```bash
python tools/stage01_data_protocol/check_dataset_pairs.py
```
- `--project-root`：默认自动推断。
- `--output-dir`：默认 `reports/data_checks`。

### 4. 派生边界/距离目标样例

```bash
python tools/stage01_data_protocol/build_boundary_targets.py
python tools/stage01_data_protocol/build_distance_targets.py
```
- `--num-per-split`：每个 split 导出几个样例，默认 `2`。
- `--output-dir`：边界默认 `reports/data_targets/boundary`，距离默认 `reports/data_targets/distance`。
- `--report`：边界默认 `reports/data_checks/boundary_target_report.md`，距离同理。

### 5. 抽样预览（人工核对）

```bash
python tools/stage01_data_protocol/preview_dataset_samples.py
```
- `--project-root`：默认自动推断。
- `--split`：默认 `None`（全部 split）；只看某个 split 时传，如 `--split glas_val17`。
- `--output-dir`：默认 `reports/data_preview`。
- `--notes`：默认 `reports/data_checks/manual_audit_notes.md`。

### 6. 数据资产汇总验收（关键，决定阶段能否 pass）

```bash
python tools/stage01_data_protocol/validate_data_assets.py
```
- `--project-root`：默认自动推断。
- `--dataset`：校验哪个数据集，默认 `all`（GlaS+CRAG）。
- `--report`：默认 `reports/data_checks/data_asset_validation_report.md`。
- `--acceptance-output`：默认 `reports/stage_reports/data_stage_acceptance.md`（**看这里的 `data_stage_pass=True`**）。
- `--manifest-output`：默认 `reports/stage_reports/asset_manifest.json`。

### 7. 训练入口预飞（不是真正训练）

```bash
python scripts/train.py --config configs/experiment/A1_UNet_GlaS_v1_seed3407.yaml --device cuda --runtime-check
```
- `--config`：**必填**，实验配置相对路径。
- `--runtime-check`：本阶段关键开关，只做「正式资产检查 + 轻量 payload 落盘」的入口预飞，**不跑完整训练**。
- `--device`：默认 `cuda`（CUDA 不可用自动回退 CPU）。
- `--runtime-check-output`：payload JSON，默认 `b_class_auxiliary/runtime_checks/train_runtime_payload.json`。
- 诚实边界：本步只证明「数据链能喂进训练入口」，**不代表 forward/loss/backward/optimizer.step 已成立**，那是 02 阶段的事。

## 五、跑完怎么确认成功

- `reports/stage_reports/data_stage_acceptance.md` 里 `data_stage_pass=True`、`handoff_ready=True`、`preflight_pass=True`
- `reports/stage_reports/asset_manifest.json` 已生成
- 各 `reports/data_checks/*_report.md` 无报错项

## 六、常见坑

- 命令必须在项目根目录 `crc_gland_segmentation_project/` 下跑，不能在上一级 `Paper/`（否则 `python: can't open file 'scripts/...'`）。
- 数据根路径写在 `configs/data/glas.yaml` / `crag.yaml`，换机器重做时先确认这两个 yaml 里的数据目录指向真实存在的原始数据。
