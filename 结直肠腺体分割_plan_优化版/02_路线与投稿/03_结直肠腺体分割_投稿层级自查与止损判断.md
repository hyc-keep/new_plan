# 结直肠腺体分割投稿层级、自查与止损判断

这份文档不是继续讨论“还要不要补很多论文”，而是把当前项目真正走到投稿口时需要面对的三个判断压实：

1. 现在这套结果更适合投哪个层级
2. 哪些结果形态可以进入写稿，哪些还不够
3. 如果某些模块或外部对比不理想，应该如何止损和回退

## 文档定位

- 类型：投稿判断文档
- 核心作用：回答“现在够不够写、先投哪一层、哪些内容必须保留、哪些内容可以止损”
- 前置阅读：`02_结直肠腺体分割_分阶段实验路线与执行标准.md`
- 配套文档：`01_结直肠腺体分割_高性价比路线总锁定.md`

---

## 1. 一页结论

### 1.1 当前最适合进入投稿判断的主线

- 任务固定：`2D` 结直肠组织病理腺体分割
- 主数据集固定：`GlaS`
- 第二 benchmark 固定：`CRAG`
- 流程排雷模型固定：`UNet`
- 主 baseline 固定：`ResNet34-U-Net`
- 主方法表达固定：`形态感知大核建模 + 边界分离辅助监督`
- `Distance-aware Loss` 固定定位：`可选增强项，不预设一定保留`

一句话说：

> 当前项目最稳的投稿版本，不是追最重模型，而是把 `baseline -> LKMA -> Boundary Head` 这条证据链做扎实，再决定是否保留 `Distance-aware Loss`，然后按结果形态在“中文保底收口”和“直接按普通英文 / SCI 低区组织”之间做判断。

### 1.2 当前最推荐的最终模型表达

投稿前优先按下面顺序判断最终模型：

1. 首选：`Baseline + LKMA + Boundary Head`
2. 次选：`Baseline + LKMA + Boundary Head + Distance-aware Loss`
3. 回退：`Baseline + Boundary Head`
4. 再回退：`Baseline + LKMA`

默认理解：

- `LKMA` 负责整体形态感知
- `Boundary Head` 负责边界分离与对象解耦
- `Distance-aware Loss` 只是补轮廓监督，不要预设它一定是核心贡献

### 1.3 当前最现实的投稿层级

- 默认第一目标：`中文学报 / 应用型中文期刊`
- 条件满足时的并行目标：`普通英文应用期刊 / SCI 低区`
- 当前暂不建议：`直接冲高难度英文目标`

这里不要再机械理解成“必须先写中文，再考虑英文”，更准确的裁决口径是：

- 计划执行阶段默认按中文保底闭环组织
- 结果裁决阶段允许直接判断是否够到英文起步
- 只有在结果不够强、任务内 direct comparison 不足或 `CRAG` 支撑偏弱时，才回落到中文优先收口

### 1.4 当前最重要的不是继续扩路线，而是判断结果形态

投稿前真正需要的不是“更多模块”和“更多文献”，而是明确下面三件事：

- 最终模型到底停在哪一版
- 现在的结果能不能支撑中文保底稿
- 是否值得再加 `1-2` 个增强对比去冲普通英文

## 2. 投稿层级怎么判断

### 2.0 先给最重要的裁决原则

投稿层级不是按主观信心定，而是按结果形态定。

你后面只允许在下面三类判断中选一类：

1. `直接按英文起步组织`
2. `先按中文保底收口，再追加英文增强`
3. `暂不进入写稿，继续回到实验阶段`

不允许出现：

- baseline 还不稳就硬按英文写
- 主线其实够英文了，却因为惯性只敢当中文看
- 明明主线还没闭环，却因为焦虑提前扩很多对比和文献

### 2.1 中文保底稿的硬门槛

如果准备把稿件按“中文保底”推进，下面这些不是建议项，而是最低门槛：

1. `GlaS` 上必须有 `UNet / ResNet34-U-Net / 最终模型` 三组完整结果
2. 内部主线至少 `UNet / baseline / final` 需要 `3` 个 seed 的 `mean +- std`
3. 主线消融必须完整，至少要覆盖 `baseline`、`+ LKMA`、`+ Boundary Head`、`+ Distance-aware Loss`
4. 主指标必须齐：`F1 / Object Dice / Object Hausdorff`
5. 至少一类边界证据必须齐：`Boundary F1` 或 `HD95`
6. `CRAG` 至少要有 `UNet / baseline / final` 的固定协议结果
7. 图表必须成套：主结果表、消融表、对象级/边界级指标表、失败案例图
8. 中文方法和中文实验两章必须已经能直接落笔

如果上面 `8` 条里有 `2` 条及以上明显缺失，就不要把项目判断成“已经够中文投稿”。

### 2.2 普通英文起步稿的硬门槛

如果准备从中文保底往普通英文应用期刊推进，除了满足中文保底的全部门槛，还应尽量满足下面这些硬条件中的大部分：

1. 外部主表必须有 `3` 个代表性对比：`UNet++ / DeepLabV3+ / Attention U-Net`
2. 主结果表里至少要有 `2-3` 个 gland-specific 直接量化对照，优先从 `DCAN / MILD-Net-RTS / DEA-Net / SDT / TA-Net` 里选，不能只在 related work 里提一下
3. 内部主线必须有 `3` seed 的 `mean +- std`
4. `CRAG` 不能只是“没跑崩”，而要能说明最终模型相对 baseline 的优势没有消失，并且能和 `MILD-Net-RTS / TA-Net / DEA-Net` 这类任务内方法形成直接参照
5. 方法贡献不能只靠像素 Dice，要能落到对象级或边界级证据
6. 如果外部增强对比来得及，可以再补 `1` 个：`nnU-Net / TransUNet / Swin-Unet` 中择优，不把它当成替代 gland-specific 对照的手段
7. related work 需要补齐近两年直接相关工作，避免审稿时被指出漏文献
8. 英文摘要、方法和实验必须能写成不过度夸张的版本

如果上面大部分都不满足，就不要一开始按英文稿组织全文。

### 2.2.1 可以直接按英文起步组织的触发条件

如果下面这些条件大部分成立，就不必强行“先写完中文再升级”，而可以直接按英文低区 / 普通英文应用期刊组织：

1. `LKMA` 或 `Boundary Head` 至少有一层在对象级主指标上表现出清楚且多 seed 一致的正增益
2. `CRAG` 上最终模型相对 baseline 的优势没有消失，或者至少主要优势仍可解释
3. 主结果表里已经具备通用外部对比和任务内 direct comparison 的最小集合
4. 你的结论不再只靠 `Dice / IoU`，而能落到对象级、边界级或黏连分离可视化
5. 英文 related work 已经把 `MAC-Net / DEA-Net / TA-Net / MILD-Net / SDT` 这类直接参照对齐清楚

这不是说英文一定比中文更优先，而是说：

- 只要结果真的够，就允许直接按英文整理
- 中文此时仍然可以作为保底出口，而不是唯一顺序

### 2.3 当前不建议直接冲更高目标的原因

当前阶段不建议直接按更高目标组织全文，主要不是因为路线错误，而是因为：

- 数据体量和任务范围都偏聚焦
- 主 benchmark 仍然较少
- 方法更像“高性价比任务导向改进”，不是重型新范式
- 现阶段最稳的优势在可复现和可解释，不在绝对 novelty 拉满

### 2.3.1 为什么当前主干本身仍有英文机会

虽然当前不建议一开始就按更高难度目标组织全文，但这不等于这条主干只能做中文。已有直接竞品说明：

- 围绕 attention、context fusion、boundary recovery 的 gland-specific 路线仍在持续发表
- 你这条线如果把“大核形态建模 + 边界分离 + 对象级/边界级证据链”讲清楚，是有英文空间的
- 真正限制英文上限的不是“有没有第四个模块”，而是结果是否稳定、direct comparison 是否够强、写作有没有把差异和边界讲清楚

### 2.4 经验预警线，不当作硬阈值

下面这些不是投稿门槛，也不是写进论文里的正式标准，只是内部排查时的经验预警线：

- 如果最终模型在 `GlaS TestA` 的对象级结果明显低于你当前预期区间，优先排查指标实现、数据协议和训练配置，而不是继续叠模块
- 如果最终模型在 `GlaS TestB` 的对象级结果明显低于 `UNet` 和 `ResNet34-U-Net` 应有的常见水平，优先怀疑代码、split、评估脚本或 checkpoint 选择
- 如果 `TestA` 和 `TestB` 差距异常大，也优先检查数据划分、增强和评估脚本的一致性

更稳妥的理解方式是：

- 预警线只用于提醒“结果可能有实现问题”
- 不用于直接判定“这篇稿一定不能投”
- 真正是否保留结果，仍以统一协议、三次重复、对象级与边界级证据为准

## 3. 现在够不够进入写稿

### 3.1 允许进入写稿的最低条件

只有同时满足下面这些，才建议进入正式写稿：

- 主 baseline 已稳定
- 最终模型已锁定
- 每个模块去留已经有明确结论
- `GlaS` 主结果表已成型
- `CRAG` 补充验证已完成
- 至少一组对象级可视化和一组边界级可视化已经准备好
- 你已经能用一句话解释每个模块各自解决什么问题

### 3.2 直接否决写稿的信号

只要出现下面任意一条，就不建议进入正式写稿：

- 最终模型还没锁定
- `baseline` 还不稳定
- `Distance-aware Loss` 还没判清是主线还是补充
- `CRAG` 结果还没出来
- 主结果表和消融表还没有初版
- 可视化还只是零散截图，不能支持方法故事

### 3.3 写稿前最后一次自问

进入写稿前必须能明确回答下面五个问题：

1. 最终模型到底停在哪一版
2. 为什么 `LKMA` 要保留或删除
3. 为什么 `Boundary Head` 要保留或删除
4. `Distance-aware Loss` 最终是主线、补充还是删除
5. 当前稿件是按中文保底收口，还是已经具备普通英文起步条件

## 4. 投稿版实验包怎么收

### 4.1 内部主线实验名单

这部分必须跑完整，不是可选项：

1. `UNet`
2. `ResNet34-U-Net` 主 baseline
3. `Baseline + LKMA`
4. `Baseline + LKMA + Boundary Head`
5. `Baseline + LKMA + Boundary Head + Distance-aware Loss`

要求：

- 每个主线版本尽量保持统一训练协议
- 主结果至少 `3` 个 seed
- 至少在 `GlaS` 上完整产出
- `CRAG` 至少覆盖 `UNet / baseline / final`

### 4.2 主表第一层必跑外部对比名单

优先跑下面这三类，够构成投稿主表：

1. `UNet++`
2. `DeepLabV3+`
3. `Attention U-Net`

这样选的原因：

- `UNet++`：代表更强的 U 形多尺度跳连基线
- `DeepLabV3+`：代表经典通用语义分割强基线
- `Attention U-Net`：代表注意力增强 U 形结构，和你的主线更贴近

### 4.3 主表第二层任务内直接量化对照

这层是当前普通英文稿最容易被审稿人盯的一块，因此不能再写成“只引用不强求复现”：

- `GlaS` 主表优先对齐：`DCAN / MILD-Net-RTS / DEA-Net / SDT`
- `CRAG` 主表优先对齐：`MILD-Net-RTS / TA-Net / DEA-Net`

执行上更稳的理解是：

- 能统一协议复现的，优先复现
- 暂时不统一复现的，也至少要作为主结果表里的任务内文献 direct comparison 明确列出
- 不要只拿 `UNet++ / DeepLabV3+ / Attention U-Net` 来支撑英文稿，因为那样更像“通用分割对比”，还不够像“腺体任务内直接对话”

写进论文时还要遵守三条表格规则：

- 统一协议复现实验结果标为 `reproduced`，直接引用原文的结果统一用 `†` 标记
- 只有在同一 benchmark、同一 split、同一指标口径下的数字，才允许进入主结果表做 direct comparison
- 如果协议差异明显，例如 split 不同、输入设置差异太大或本身带特殊测试流程，就降级到引用对比或 discussion，不硬写成同一口径横比

### 4.4 选跑增强名单

如果时间、算力和代码条件允许，再从下面只选 `1-2` 个：

- `TransUNet`
- `Swin-Unet`
- `nnU-Net`
- 一个近两年腺体分割代表方法

使用原则：

- 只补最能增强说服力的，不要全堆
- 如果复现成本过高，就降级为引用对比，不强求都跑

### 4.5 只引用不强求复现的名单

这类更适合放在 related work、discussion 和对比表引用列中：

- `SCAU-Net`
- `Prompted SAM-H`
- `AttentionBoost`
- 近两年直接在 `GlaS / CRAG` 上验证的腺体分割工作

这类结果在论文里如果直接引用，也建议统一：

- 用 `†` 标记为原文报告值
- 在表注中说明是否存在协议差异
- 不和你自己统一协议下复现的结果混写成“完全公平对比”

这类文献的价值主要是：

- 说明任务脉络
- 说明你的方法和已有“边界 / 拓扑 / 形态”路线的关系
- 说明你没有漏掉直接相关工作

## 5. 自查与止损判断

### 5.1 哪些模块值得保留到最终模型

只有当下面条件成立时，才建议把模块保留进最终模型：

- `3` 个 seed 中至少大多数方向一致
- 相对上一版不是只涨一个边角指标
- 可视化能够支持它的职责解释
- 训练成本没有显著失控

更具体地说：

- `LKMA`：至少应在对象级结果、轮廓完整性或边界相关指标里体现出稳定正向价值，否则不建议保留
- `Boundary Head`：至少应带来 `Boundary F1`、`HD95 / Object Hausdorff` 或黏连分离可视化中的一类明确改善，否则不建议保留
- `Distance-aware Loss`：只有在上一版基础上还能提供额外稳定增益时才保留；如果只是接近波动范围，直接降为补充或删除

### 5.2 哪些情况需要回退

出现下面这些情况时，应该主动回退，而不是继续叠东西：

- `Distance-aware Loss` 收益接近波动范围
- `Boundary Head` 没有边界证据支撑
- `LKMA` 成本明显增加但收益有限
- `CRAG` 一加模块就明显崩掉
- 外部对比复现成本过高，已经拖慢主线闭环

推荐回退顺序：

1. 先删除 `Distance-aware Loss`
2. 再判断是否只保留 `LKMA + Boundary Head`
3. 如果仍不稳，回退到最清楚的一层改进

更具体的止损规则：

- 如果 `Distance-aware Loss` 在 `3` 个 seed 中只有 `1` 个方向变好，直接降级
- 如果 `Boundary Head` 没有任何边界指标改善且可视化也不能支撑，直接回退
- 如果 `LKMA` 让训练成本明显上升但主指标和对象级指标都不稳定，直接回退
- 如果最终模型在 `CRAG` 上相对 baseline 于多个核心指标同时变差，就不要写强泛化，只保留补充验证定位

### 5.3 哪些情况说明该停下补文献

从当前阶段看，下面这些情况出现时，就不要再继续泛补 PDF：

- 主线已经锁定
- 对比名单已经锁定
- 缺的是实验结果，不是文献名录
- 缺的是图表整理，不是 related work 规模

后续只建议补很少量定制项：

- 目标期刊近两年相似论文
- 你最终保留的增强对比原始论文
- 写稿时需要的指标出处或背景出处
- 任务内 direct comparison 的最终原文表格和页码来源

## 6. 投稿版结果包最低要求

### 6.1 主结果表最低组成

至少要有下面四张核心表：

- `表 1` 内部主线逐步消融表
- `表 2` 外部方法主结果对比表
- `表 3` 边界指标与对象级指标表
- `表 4` `GlaS + CRAG` 联合结果表

### 6.2 核心图片最低组成

至少要有下面六类图：

- 数据样例图
- 主结果可视化图
- 黏连腺体分离对比图
- 边界局部放大图
- 失败案例图
- `CRAG` 成功/失败案例图

### 6.3 结论保留规则

只有在下面条件成立时，才建议把模块写成正式贡献：

- 指标有支持
- 可视化有支持
- 多个 seed 方向一致

如果只满足一部分：

- `LKMA`：可以写成“增强整体形态建模的尝试”
- `Boundary Head`：可以写成“边界辅助监督分支”
- `Distance-aware Loss`：优先写成“可选补充约束”

### 6.4 投稿前必须齐的最小结果包

投稿前至少要明确确认下面这些都已经齐：

- `GlaS` 主结果表
- 主线逐步消融表
- `CRAG` 补充验证表
- 对象级与边界级指标表
- 成功案例图
- 失败案例图
- 黏连腺体分离图
- 中文摘要和中文结论初稿

## 7. 最终判断模板

### 7.1 如果结果达到这一级

- 主线消融完整
- 最终模型明确
- `CRAG` 不崩
- 外部对比已有代表性
- 任务内 direct comparison 已进入主表
- 图表齐

优先动作：

- 直接判断是否按英文低区 / 普通英文应用期刊组织全文
- 同步保留中文保底出口
- 如果英文 still 缺一层说服力，再补 `1-2` 个增强对比或形态学下游分析

### 7.2 如果结果只达到这一级

- `GlaS` 主结果清楚
- `CRAG` 只有补充验证
- 外部对比还不算很强
- 但主线故事完整

优先动作：

- 先按中文保底稿收口
- 不继续无止境加模块
- 把更多精力放在图表和写作表达上
- 再判断是否只补最小英文增强包

### 7.3 如果结果还没达到这一级

- baseline 还不稳
- 最终模型还没锁
- `CRAG` 明显有问题
- 图表和可视化还不成体系

优先动作：

- 回到 `02` 的阶段执行标准
- 不进入正式写稿
- 不继续扩大外部对比范围

## 8. 最终建议
