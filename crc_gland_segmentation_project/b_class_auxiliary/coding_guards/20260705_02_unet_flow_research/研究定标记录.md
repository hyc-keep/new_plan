## 3. 当前阶段研究目标
- 当前阶段唯一研究问题: 在 `01_数据协议` 已正式放行之后,如何把 `02_UNet流程验证` 的首版 `UNet` 训练与评估闭环研究清楚,使后续阶段锁定只围绕标准 `UNet`、固定训练评估协议和正式工程缺口展开
- 这一轮研究要先回答什么: `02` 阶段现在为什么必须先做标准 `UNet`,哪些默认值已经被上游冻结,对象级评估和 TestA / TestB 边界为什么必须先继承,当前工程到底缺哪些正式对象,以及为什么本轮不能直接进入阶段锁定之后的编码/运行
- 如果现在不先研究清楚,后面会在哪一步开始靠猜: 会在阶段锁定里把“标准 `UNet` 首版协议”写成泛泛的“开始训练”,会在 Pre-check 和编码时临时决定 optimizer/loss/threshold/best_selector,也会把当前 stage01 预飞入口误当成 stage02 训练入口
