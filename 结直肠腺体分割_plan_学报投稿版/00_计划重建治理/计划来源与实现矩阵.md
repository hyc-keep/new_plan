## 7. 当前主要冲突待裁决

1. N1 的 checkpoint selector：旧协议为 `val_objdice_max`，当前 journal smoke 曾使用 `val_loss_min`；必须由正式 N1 协议裁决，未裁决不得编码。
2. N1/N2 是否三 seed：必须在 05/00 和 06/00 预注册，不能 test 后追加。
3. N1 contour target：实例轮廓宽度/radius、空实例、边缘实例和损失形式必须由论文/验证规则冻结。
4. N3 watershed：marker、elevation、阈值、最小面积和形态学操作只能在 val17 选择。
5. 旧 11 的细粒度 readiness 文件是否恢复为 active：在批次 7 重新裁决。
6. 旧工程完整评估链迁移到 journal 的 API 名称、目录和 manifest 字段必须先写入新代码映射表。

本矩阵只定义来源和待裁决，不代表任一实验结果已成立。
