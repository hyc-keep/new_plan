# 基于62篇正式深提取的关键词反查索引

这份文档的目标只有一个：

> 当你脑子里只有一个关键词、一个指标名、一个模块名或一个写作问题时，能最快反查到应该先看哪几篇正式深提取稿。

这份文档不重复抄正式稿内容，只负责做“关键词 -> 类别 -> 代表论文 -> 进入后重点看什么”的路由。

---

## 1. 使用方法

### 1.1 这份文档负责什么

- 当你脑子里只有关键词、指标名、模块名或术语时，提供第一跳入口
- 把关键词映射到“优先目录 + 代表正式稿 + 进入后重点看什么”
- 降低后续在 `62` 篇正式稿里盲搜的成本

### 1.2 什么时候先看这份

- 当你不知道该先开哪篇论文，只知道一个词时
- 当你知道要找某个模块或指标，但不知道它属于哪个目录时
- 当你想把“实验问题”快速转成“应查看的正式稿”时

### 1.3 这份文档的输入和输出

- 输入问题：`这个词、这个模块、这个指标该去哪里查`
- 输出结果：`关键词 -> 目录 -> 论文 -> 重点字段`

### 1.4 推荐使用顺序

1. 先在本文件搜索关键词
2. 找到对应条目后，先看“优先目录”
3. 再打开“代表正式稿”
4. 进入正式稿后，优先看：
   - `1. 论文信息`
   - `6. 训练协议`
   - `9. 主表结果与对比`
   - `15. 对我们项目的落地价值`
   - `16. 关键图表索引`

如果一个关键词同时涉及多个层面，优先顺序如下：

- 先看任务内直接相关论文
- 再看通用方法或损失论文
- 最后再看综述、病理或下游应用论文
