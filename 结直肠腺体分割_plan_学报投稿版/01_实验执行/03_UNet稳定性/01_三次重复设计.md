>
> 当前身份同步：本文件当前正式 run_name、配置路径和 experiments 路径统一使用 `A2_UNet_GlaS_seed3407 / seed1234 / seed2025`。旧身份只允许在本冻结声明和历史映射说明中出现。当前正式结果使用 `eval_proto_v1`；`protocol_v3` 只用于历史追溯，不能进入当前汇总或 Gate。`stage_pass_a2/Gate_A2` 与 `workflow_gate_status` 属于不同层级，必须分别记录。

本文件不是 `结直肠腺体分割_plan_优化版/01_实验执行/03_UNet稳定性/00_阶段总协议.md` 的摘要版，也不是对 `A1` 阶段“再补两次运行”的口头备注。

## 1. 文件角色与执行边界

### 1.1 当前文件负责什么

当前文件只负责冻结下面六件事：

1. `A2` 阶段唯一合法的三组正式训练 seed。
2. 三次正式 run 的固定执行顺序与命名。
3. 每个 seed 必须独立交付的最小资产清单。
4. 三次 run 必须共享的正式协议字段，以及唯一允许变化的字段。
5. 异常 run 的判定、留痕和是否允许进入同层汇总的前置边界。
6. `结直肠腺体分割_plan_优化版\01_实验执行\03_UNet稳定性\01_三次重复设计.md` 向 `结直肠腺体分割_plan_优化版\01_实验执行\03_UNet稳定性\02_mean_std汇总规则.md` 和 `结直肠腺体分割_plan_优化版\01_实验执行\03_UNet稳定性\03_阶段验收.md` 的交接条件。

### 1.2 当前文件不负责什么

当前文件不重新定义以下内容：

- `UNet` 结构、输入规格、损失、优化器和评估链本身。
- `mean +- std` 的聚合公式、表结构和阶段稳定性总说明写法。
- `A2` 阶段最终是否放行到 `04_Baseline` 的总门控结论。
- `Baseline`、`LKMA`、`Boundary`、`Distance` 等下游阶段的方法设计。

这些职责分别以下列文件为准：

- `结直肠腺体分割_plan_优化版/01_实验执行/03_UNet稳定性/00_阶段总协议.md`
- `结直肠腺体分割_plan_优化版/01_实验执行/03_UNet稳定性/02_mean_std汇总规则.md`
- `结直肠腺体分割_plan_优化版/01_实验执行/03_UNet稳定性/03_阶段验收.md`
- `结直肠腺体分割_plan_优化版/01_实验执行/04_Baseline/00_阶段总协议.md`

### 1.3 为什么当前文件必须独立存在

因为 `A2` 阶段最容易出现的偷换是：把“固定协议下的三次重复”偷改成“想到什么跑什么的多次试验”。

如果没有一份独立的 seed 执行细则，后面就会出现下面几类风险：

- 某个 seed 结果难看时被临时替换。
- 三次 run 的协议字段不一致，却仍被送去做 `mean +- std`。
- 某个 seed 没有 `TestA / TestB`、可视化或错误记录，却照样进入阶段结论。
- 异常 run 被覆盖或删除，导致 `A2` 的波动范围失真。

所以本文件的任务不是解释 `UNet` 为何成立，而是把“什么才算正式三次重复”写成不可漂移的执行清单。

## 3. 本阶段唯一允许处理的变量

### 3.1 三次重复的核心定义

`A2` 阶段三次重复的核心定义固定为：

- `S_repeat = {3407, 1234, 2025}`
- `theta_k = Train(P_fixed, seed = s_k)`，其中 `s_k in S_repeat`
- 对任意两次正式 run `r_i, r_j`，必须满足 `proto(r_i) = proto(r_j)`，且唯一允许差异是 `train_seed(r_i) != train_seed(r_j)`

### 3.2 当前阶段真正测的是什么

当前阶段测的是“固定协议下的随机波动范围”，不是重新调参。

因此，`A2` 只能回答：

- 同一 `UNet` 协议在不同随机种子下是否稳定。
- 结果波动是否仍在可解释范围内。
- 错误模式是否跨 seed 保持大体一致。

它不能夹带回答：

- 换增强是否更好。
- 改阈值是否更好。
- 上新后处理是否更好。
- 改 batch size 或 optimizer 是否更好。

### 3.3 为什么必须固定为三组 seed

固定为 `3407 / 1234 / 2025` 的原因是：

- 这三组 seed 已在全局冻结层正式写死。
- 三次重复是当前路线下第一版 `mean +- std` 的最低统计入口。
- 计算成本仍可控，不会在 `UNet` 阶段提前消耗过多算力。
- `3407` 与 `A1_UNet_GlaS_v1_seed3407` 直接对齐，便于检查是否发生协议漂移。

### 3.4 当前文件直接继承的冻结字段

本文件继承并禁止改动的正式协议字段包括：

- `GlaS train68 / val17 / testA60 / testB20`
- `mask > 0`
- `RGB`
- `512 x 512`
- `ImageNet mean/std`
- `light_aug_v1`
- `L_seg = L_BCE + L_Dice`
- `AdamW`
- `lr = 1e-3`
- `weight_decay = 1e-4`
- `ReduceLROnPlateau`
- `epoch_max = 120`
- `early_stopping = 20`
- `AMP on`
- `best_selector = val_objdice_max`
- `threshold_source = val17`
- `threshold_value`
- `eval_cast_policy = logits/probabilities must be kept or cast to float32 before thresholding`
- `boundary_metric_width = 3 px`
- `boundary_metric_impl = np.logical_xor(mask, ndimage.binary_erosion(mask, structure, border_value=0)) + ndimage.binary_dilation`
- `connected_components_impl = scipy.ndimage.label`
- `connected_components_connectivity = 8`
- `eval_proto_version = eval_proto_v1`
- `postprocess_version = none_in_v1`

## 4. 阶段门控表达式

### 4.1 固定三组 seed

- 当前结论：`A2` 阶段只允许使用 `3407 / 1234 / 2025` 三组正式训练 seed。
- 规则类型：`路线层已锁定 + 工程冻结规则`
- 适用阶段：`03_UNet稳定性`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/02_参数冻结总表.md`，`结直肠腺体分割_plan_优化版/02_路线与投稿/02_结直肠腺体分割_分阶段实验路线与执行标准.md`
- 核心公式或定义参考：`S_repeat = {3407, 1234, 2025}`
- 采用原因：三次重复已满足当前阶段最小统计入口，且能和 `A1` 的 `seed3407` 直接对照。
- 不采用的相邻方案：不采用临时换 seed；不采用结果难看时加跑第四或第五个 seed；不采用删除低结果 seed 后再汇总。

- 运行记录字段：`run_name`，`train_seed`，`seed`，`config_version`，`aggregation`
- 验收方式：检查正式 run 名称和结果表中只出现这三组 seed，不存在替换 seed 或额外补跑混算。

### 4.2 固定执行顺序

- 当前结论：三次正式 run 的执行顺序固定为 `seed3407 -> seed1234 -> seed2025`。
- 规则类型：`工程冻结规则`
- 适用阶段：`03_UNet稳定性`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/02_UNet流程验证/00_阶段总协议.md`，`结直肠腺体分割_plan_优化版/01_实验执行/03_UNet稳定性/00_阶段总协议.md`

- 采用原因：先与 `A1` 的 `seed3407` 对齐，确认协议未漂移，再继续后两次重复，最利于定位问题来源。
- 不采用的相邻方案：不采用随机顺序；不采用先跑更高 seed；不采用先把三个目录名占位后混合补跑。

## 5. 逐 run 执行与观察要求

### 5.1 每个 seed 必须独立完成的动作

每个 seed 都必须独立完成：

1. 训练。
2. 验证。
3. `TestA` 测试。
4. `TestB` 测试。
5. split 级可视化导出。

- 只给某个 seed `TestA`，不给 `TestB`。
- 只保留“最好看”的一个目录，其余两个只留数字。
- 可视化和错误案例只对单个 seed 补做。

### 5.2 每个 seed 至少要记录的观察字段

每个 seed 至少要补齐下列观察字段：

- `best_epoch`

- `boundary_metric_width`
- `boundary_metric_impl`

- val_metrics.csv 中的验证趋势
- testA_metrics.csv
- testB_metrics.csv
- summaries/error_cases.md

### 5.3 当前阶段最需要观察的三类一致性

三次重复至少要回答以下三类一致性问题：

1. 主指标方向是否大体一致，而不是某次突然崩掉却解释不清。

## 6. 回退条件

### 6.1 独立回退触发条件

只要出现下面任意一条，本文件负责的三次重复执行就必须先回退处理，不能直接流入 `结直肠腺体分割_plan_优化版\01_实验执行\03_UNet稳定性\02_mean_std汇总规则.md` 或 `结直肠腺体分割_plan_优化版\01_实验执行\03_UNet稳定性\03_阶段验收.md`：

- 某个 seed 缺失最小资产，导致该 run 不能独立复核。
- 三次 run 的正式协议字段不一致。
- 指标与可视化长期明显矛盾。
- 某个 seed 出现明显全背景预测、异常抖动或中途逻辑错误，却没有排查记录。
- 某次 run 被覆盖、删除或替换，导致无法回查原始异常。
- `threshold_source`、`threshold_value`、`best_selector` 与 `A1` 冻结协议不一致。
- `eval_cast_policy` 没有保持 `threshold` 前 `float32` 边界，或 `Boundary F1 / connected components` 实现字段与 `A1` 冻结版本不一致。

### 6.2 固定回退顺序

发生回退时，统一按下面顺序检查：

1. 检查是否误改了 `A1` 已冻结协议。
2. 检查 seed 设置、随机性控制和配置分发是否一致。
3. 检查训练、验证、测试、可视化和错误案例资产是否齐全。
4. 检查异常 run 是否属于 `data / target / loss_eval / engine / model` 中的流程问题。
5. 只有确认前面都没有问题后，才允许把差异解释为真实 seed 波动。

### 6.3 回退后的重新放行条件

回退后，只有同时满足下面条件，才允许恢复“可进入同层汇总/验收”的状态：

- 问题来源已按固定顺序定位并修复。
- 修复动作和影响范围已写入 notes/debug_note.md 或逐 run 总结。
- 三次 run 的正式协议字段重新对齐。
- 缺失资产已经补齐，或作废原因与替代处理已显式记录。
- 当前三次 run 重新满足第 `4.6` 节的交接条件。
