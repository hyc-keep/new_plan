## 2. 当前已成立结论

### 2.1 A2/B1 训练协议控制变量基本一致

已确认两组使用同一：

- 数据源和 split；
- train68/val17/TestA60/TestB20；
- 输入 512x512；
- normalization；
- 数据增强；
- BCE + Dice；
- AdamW；
- ReduceLROnPlateau；
- threshold 0.5；
- val17 threshold source；
- best selector `val_objdice_max`；
- early stopping patience 20；
- seed 集合 3407/1234/2025；
- 当前共同评估入口和聚合函数。

未发现会直接使两组训练比较失效的 split、尺寸、增强、loss、optimizer、scheduler 或 early stopping 配置差异。

### 2.2 B1 v2 三个正式 run 已真实完成

三个 B1 v2 run 均：

- 使用 CUDA AMP；
- 以 `early_stopping` 自然结束；
- `epoch_count = best_epoch + 20`；
- 生成 best/last checkpoint；
- 测试 CSV 的 checkpoint epoch、metric 和 SHA256 与 run_meta 对齐；
- TestA/TestB 样本数为 60/20；
- sample mean 与 aggregate 行内部 crosscheck 通过。

### 2.3 当前均值方向结论成立

当前比较表显示 B1 v2 相比 A2：

- TestA F1、Object Dice 改善；
- TestA Object Hausdorff 下降；
- TestB F1、Object Dice 改善；
- TestB Object Hausdorff 下降。

因此 `main_metrics_not_worse=true` 的方向判断与当前聚合数字一致。

### 2.4 当前三 seed 波动数字是真实产物

当前聚合脚本确实按 `ddof=0` 计算三 seed population std，且 A2/B1 共用该函数。当前阻断数字来自真实 v2 产物，不能删除或手改。

但“数字真实产生”不等于“指标定义已经完全科学闭合”。下面的口径缺口必须先解决。

## 3. 当前不能直接当作最终科学结论的项目

### 3.1 Object F1 口径冲突

当前代码/参考代码更接近：

```text
overlap / GT_area > 0.5
```

而总览/协议文字存在：

```text
overlap / prediction_area >= 0.5
```

并且协议文字要求显式一对一去重。两者在过分割、黏连和对象面积不相等时会给出不同 TP/FP/FN。

裁决状态：`resolved`。正式冻结为 `overlap / GT_area > 0.5`，每个预测对象取最大重叠 GT，不使用 Hungarian matching。

### 3.2 Object Hausdorff 匹配策略冲突

当前代码在多个候选对象中选择 Hausdorff 最小者；GlaS/DCAN 参考代码更接近选择像素重叠最多者。

Object Hausdorff 是当前阻断指标之一，因此当前阻断不能完全解释为模型性能差异。

裁决状态：`not_resolved`。

### 3.3 HD95 实现与协议命名不一致

协议曾写 `medpy.metric.binary.hd95`，当前代码使用自定义边界提取、EDT 和 percentile，不是直接 MedPy 实现。空前景时当前使用有限图像对角线，参考资料常见语义是异常或 `inf`。

裁决状态：`not_resolved`。

### 3.4 空前景距离规则没有充分外部依据

当前双空/单侧空规则对距离指标使用有限工程值。该规则可以作为项目自定义协议，但不能继续同时声称等价于 MedPy/GlaS 官方距离指标。

裁决状态：`not_resolved`。

### 3.5 Boundary F1 connectivity 未完全写入冻结字段

代码使用 8-connectivity 的 erosion-xor；协议文字主要写 `find_boundaries(mode=inner)`。已有等价说明依赖“同一结构元”前提，但该前提应显式写入协议并通过构造性测试。

裁决状态：`needs_explicit_freeze`。

### 3.6 A2 测试 CSV 的身份链弱于 B1 v2

A2 测试 CSV 缺少：

- run_name；
- seed；
- model identity；
- checkpoint path；
- checkpoint epoch；
- checkpoint metric；
- checkpoint SHA256；
- config version。

这不证明 A2 数字错误，但不满足与 B1 v2 同等强度的逐结果 provenance。当前只能说 A2 通过 run_meta、目录、配置和旧测试 CSV 间接追溯。

裁决状态：`provenance_partial`。

### 3.7 稳定性 Gate 的精确规则未在协议中充分冻结

代码采用：

```text
所有主指标 × TestA/TestB 都必须 B1_std <= A2_std
```

但计划文字使用“方向稳定”“波动不增”“稳定性不弱于”，没有明确：

- 是否所有指标都必须不增；
- 允许多少项波动增加；
- 允许增加多少；
- 是否需要浮点容差；
- 是否要使用 per-seed 配对差值；
- 三 seed 对 TestB=20 的解释边界。

裁决状态：`protocol_not_frozen`。

## 5. 构造性对账后的正式裁决

### 5.1 已通过对账的非空路径

构造性对账记录 [04_Baseline_指标构造性对账记录_20260712.md](04_Baseline_指标构造性对账记录_20260712.md) 显示：

- 当前 F1 与 DCAN 在非空构造样例上一致；
- 当前 Object Dice 与 DCAN 在非空构造样例上一致；
- 当前 Object Hausdorff 与 DCAN 在非空构造样例上一致；
- 当前离散 HD95 与项目内 MedPy 源码样例一致；
- 当前逐样本均值和三 seed `ddof=0` 聚合链内部一致。

这些结论支持“现有非空主路径实现不是明显错误”，但不等于所有官方实现和所有退化情况都已等价。

### 5.2 必须在正式协议中明确、但不能后验放宽的项目

以下项目仍必须正式裁决：

1. 双空和单侧空的 Object F1、Object Dice、Object Hausdorff、HD95 返回值；
2. `hd95` 的正式名称必须明确为 MedPy-style discrete surface distance 或项目自定义实现，不能同时宣称 surface-distance robust Hausdorff 等价；
3. Boundary F1 的边界提取、8-connectivity 和 width=3 必须作为完整冻结字段；
4. GlaS gist 因环境缺失 sklearn 暂记无法验证，不用替代结果冒充官方验证；
5. A2 旧 CSV 的 provenance 只能判 `partial`，不回写旧 CSV 伪造身份字段；
6. 稳定性 Gate 必须在重新评估前冻结，禁止根据当前 blocker 选择更宽松规则。

### 5.3 稳定性 Gate 的保守处理

当前代码的 `B1_std <= A2_std` 逐项规则虽然比协议自然语言更严格，但它已经实际产生了当前 blocker。为防止后验改规则迎合结果，在研究定标完成前：

- 不删除当前 blocker；
- 不把 `stability_not_weaker` 改成通过；
- 不引入“允许若干指标波动”这类未经预注册的宽松规则；
- 只允许补充极小浮点比较容差，并且必须预先写入协议；
- 若研究定标最终决定修改稳定性定义，必须对 A2/B1 在新冻结规则下重新生成完整结果，旧 v2 blocker 保留为历史轮次。

### 5.4 当前结果的准确表述

当前准确结论是：

> B1 v2 在当前非空指标实现和当前严格逐项标准差判定下触发 stability blocker；构造性对账支持主要指标实现的非空路径与 DCAN/MedPy 样例一致，但空前景、Boundary F1、官方 GlaS gist 可验证性、A2 provenance 和稳定性规则的正式冻结仍未闭合。因此当前 blocker 不能被扩写为模型最终不稳定，也不能被改写为阶段通过。

## 7. 2026-07-12 正式冻结裁决

- Object F1：`DCAN/GlaS` 非空口径；每个预测对象选择最大重叠 GT，`overlap / GT_area > 0.5`，不使用 Hungarian matching。
- Object Dice：双向对象面积加权最大重叠。
- Object Hausdorff：有重叠时选择最大重叠对象，无重叠时选择 Hausdorff 最近对象，双向面积加权。
- HD95：正式名称为 `MedPy-style discrete surface distance`，`pixel spacing = 1`。
- Boundary F1：项目自定义 `erosion_xor`，`3x3 ones`，`8-connectivity`，`border_value = 0`，`tolerance = 3px`。
- 空集 totalization：项目自定义；双空相似指标 `1`、距离指标 `0`；单侧空相似指标 `0`、距离指标 `Dmax = sqrt((H-1)^2 + (W-1)^2)`。
- 标准差：`np.std(values, ddof=0)`。
- 稳定性 Gate：对 `{F1, Object Dice, Object Hausdorff} x {TestA, TestB}` 六个独立单元，全部满足 `sigma_B1 <= sigma_A2 + 1e-12 * max(1, sigma_A2)`；禁止多数通过、平均标准差、额外宽限、删除 split 或删除 seed。
- A2 身份结论保持 `provenance = partial`，不修改旧 CSV，不伪造身份字段。
- 本次冻结只关闭研究协议未决项，不改变历史实验数字，不改变当前 `Gate_B1=false`，不允许进入 `05`。
