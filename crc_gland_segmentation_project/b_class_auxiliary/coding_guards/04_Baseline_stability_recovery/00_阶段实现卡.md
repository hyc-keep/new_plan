## 8. 本轮最小运行验证计划

| 验证项 | 计划动作 | 预期物理证据 |
|---|---|---|
| import / py_compile | 对 `scripts/train.py`、`src/engine/trainer.py` 和配置加载入口执行编译/导入检查。 | 退出码 0 和当前轮日志。 |
| smoke run | 用独立 smoke identity 完成最小 train、val、checkpoint 闭环。 | smoke 目录、run_meta、日志和 checkpoint。 |
| dataloader batch | 抽查 train batch 的 shape、dtype、finite 和标签值。 | runtime evidence 记录真实 batch。 |
| loss / backward / optimizer.step | 执行 forward、loss、backward 和 optimizer.step，并核对两个学习率参数组。 | runtime evidence、日志和退出码。 |
| 正式 run 完整性 | 六个 run 逐一运行，检查 `experiments/` 下对应正式目录中的 run metadata、train/validation CSV、best/last checkpoint 和 summary。 | epoch 从 1 开始严格递增，无重复、无并发写入。 |
