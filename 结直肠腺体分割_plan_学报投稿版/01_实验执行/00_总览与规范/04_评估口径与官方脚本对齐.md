## 路线依据与文献依据
GlaS TestA/TestB 分开且对象级评价优先；DCAN 提供 object-contour 参考实现；MILD-Net 提供同任务对象级报告语境。surface-distance/MedPy 只用于明确的 surface/HD95 实现，不替代对象 Hausdorff。当前所有结果均 `not_run`。

## 固定评估链
1. 读取 logits；2. 在 threshold 前转或保持 `float32`；3. sigmoid；4. 应用仅由 val 锁定的 threshold；5. 生成二值 mask；6. 使用二维 8-connectivity 连通域；7. 计算对象级和像素级指标；8. 写 per-image、aggregate、crosscheck。默认无 TTA、无额外后处理；引入后处理必须升级 `eval_proto_version`。

## 指标与定义
主指标：`F1/Object F1`、`Object Dice`、`Object Hausdorff`；补充：`Dice`、`IoU`、`HD95`、`Boundary F1`。`Dice=2|P∩G|/(|P|+|G|)`，`IoU=|P∩G|/|P∪G|`，`F1=2PR/(P+R)`。Object F1 primary identity 使用挑战论文语义；DCAN 本地 `>0.5` 只作 compatibility audit，必须有 exactly-0.5 边界测试和差异报告。Boundary F1 身份固定为 `project_custom_erosion_xor_3x3_ones_8conn_border0_tol3px`，其评估宽度为 3 px；不得与 target width 混名。

## 评估失败与回退规则
- 如果对象级实现无法证明与 GlaS/DCAN 语义一致，则停止正式测试，不能用像素 Dice 代替 Object Dice。
- 如果 `Object Hausdorff`、通用 HD95、Boundary F1 的实现身份混淆，则当前评估版本判 invalid，回到评估协议重写。
- 如果 empty-mask policy、connected-components connectivity、threshold 或 cast policy 缺失，不能生成可消费 raw CSV。
- 如果独立复算与正式 CSV 在样本 ID、指标值或容差上不一致，整个 split 判失败，禁止汇总或 handoff。
- 只有在 val17 选择规则、TestA/TestB blind 和评估版本都冻结后，才允许进入正式 GlaS。
