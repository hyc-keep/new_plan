- 上一阶段已经交付: historic v1 recovery 保留为审计对象；其指标、checkpoint、run_meta、metrics 和 Gate 不进入 future round 证据链。

## 4. 为什么现在做这个,而不是下一个阶段
- 当前阶段必须先解决的阻断: future round 的文档门禁此前为过简汇总件，无法证明阶段边界、独立四件套和真实目录扫描。
- 如果现在提前做下一个阶段,会破坏什么: 直接运行 runtime/smoke 或正式训练会越过尚未完成的 Pre-check，且可能错误消费 historic v1 资产。
- 下一个阶段此刻还不允许进入的原因: 正式训练、测试、runtime、smoke、独立复核和正式 Gate 都未运行；本卡仅锁定其前置准备。
