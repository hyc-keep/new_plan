## 一、这个阶段在做什么

用**三个 seed（3407 / 1234 / 2025）**各跑一次完整 UNet 训练→测试，然后把三次结果聚合成 mean±std，验证「训练与评估链路对随机种子是稳定的」（换 seed 指标抖动小），并产出论文用的稳定性表。当前正式 run：`experiments/A2_UNet_GlaS_seed{3407,1234,2025}/`。配置文件名中的 `v1` 仅表示配置版本，不是当前 run_name。

这是「稳定性验证」（A2），前置是 02 阶段（A1，单 seed 流程验证）已通过。A2 只是把 A1 的完整链在多 seed 上复跑并聚合，**评估协议与 A1 完全一致**。

注意：本阶段主指标 F1 是 **GlaS 官方对象级检测 F1**（F1_obj = 2TP/(2TP+FP+FN)，对象匹配 overlap>0.5），不是像素 Dice。三者是不同口径，别混（详见 `实现依据记录.md` 第 13 节）。

## 三、配置引用关系（重要）

三个 seed 各有一个 experiment 入口配置，除 seed 值外其余完全同构：
- `configs/experiment/A2_UNet_GlaS_v1_seed3407.yaml`
- `configs/experiment/A2_UNet_GlaS_v1_seed1234.yaml`
- `configs/experiment/A2_UNet_GlaS_v1_seed2025.yaml`

每个 experiment 入口都引用同一套子配置：
- `configs/model/unet_v1.yaml` —— 模型结构
- `configs/train/unet_flow_v1.yaml` —— 训练超参（epoch、patience=20、lr 等）
- `configs/eval/eval_proto_v1.yaml` —— 评估协议（阈值来源、connectivity=8、cast policy、指标口径）
- `configs/data/glas.yaml` —— 数据（继承自 01 阶段）

三个 seed 必须共用**同一份 `eval_proto_v1.yaml`**，否则聚合失去可比性（协议红线字段不一致会让 Gate_A2 的 `proto_consistent` 变 false）。改超参改子配置，别散着改。

## 四、完整执行流程与命令

> 二次重做直接照抄本节。前置：`cd /home/featurize/work/Paper/crc_gland_segmentation_project`

### 步骤 0（若 checkpoint 已存在可跳过）：三 seed 正式训练

只有当三个 seed 目录的 checkpoint 缺失/损坏时才需要重训。已有 checkpoint 就直接跳到步骤 1。

```bash
python scripts/train.py --config configs/experiment/A2_UNet_GlaS_v1_seed3407.yaml --device cuda
python scripts/train.py --config configs/experiment/A2_UNet_GlaS_v1_seed1234.yaml --device cuda
python scripts/train.py --config configs/experiment/A2_UNet_GlaS_v1_seed2025.yaml --device cuda
```
- `--config`：**必填**。
- `--device`：默认 `cuda`，服务器务必保持 cuda。
- `--smoke-check`：**不要加**，加了就只是 smoke 不是正式训练。
- `--resume-from-last`：仅训练中断需续跑时加。
- 早停 patience=20，best_selector=`val_objdice_max`（选 checkpoint 只看验证 Object Dice，与 F1 口径无关，所以修 F1 口径**不需要重训**）。

### 步骤 1：三 seed 测试评估（用现有 best.ckpt，不训练）

```bash
python scripts/test.py --config configs/experiment/A2_UNet_GlaS_v1_seed3407.yaml --device cuda
python scripts/test.py --config configs/experiment/A2_UNet_GlaS_v1_seed1234.yaml --device cuda
python scripts/test.py --config configs/experiment/A2_UNet_GlaS_v1_seed2025.yaml --device cuda
```
- `--config`：**必填**。
- `--checkpoint`：默认 `None`（自动加载该 run 的 `checkpoints/best.ckpt`）；评估特定权重才传。
- `--max-visual-samples`：每 split 导出多少张最差案例可视化，默认 `5`。
- `--skip-visuals`：只要指标、不要可视化时加。
- `--max-samples-per-split`：默认 `None`（全量）；仅本地 CPU preflight/连通性联调时限制样本数；该运行不得生成或覆盖当前正式 A2 评估证据。
- 产物：每个 `experiments/A2_UNet_GlaS_seed*/` 下的 `testA_metrics.csv`、`testB_metrics.csv`（"w" 覆盖写）、`metric_crosscheck_note.md`。每条输出应含 `metric_crosscheck_result=pass`。

### 步骤 2：聚合三 seed，生成论文表与稳定性说明

```bash
python scripts/summarize_stage.py --stage a2
```
- `--stage a2`：**A2 聚合模式的关键参数**（自动扫描三个 seed 目录做聚合；02 阶段用的是 `--config` 单 run 模式，别用错）。
- `--debug-note` / `--stage-summary-output` / `--manifest-output`：都有默认路径，一般不用改。
- 产物：`reports/tables/unet_seed_results.csv`（逐 seed 原始行）、`reports/tables/unet_mean_std_summary.csv`（`n_runs=3` 聚合行）、`reports/stage_reports/unet_stability_note.md`、`reports/tables/unet_stage_manifest.csv`。输出末尾应为 `stage_pass_a2=true`。

### 步骤 3：总放行门禁（六步串行，一次给全 6 个参数）

```bash
python b_class_auxiliary/tools/enforce_workflow_gate.py \
  --stage-card b_class_auxiliary/coding_guards/20260708_03_unet_stability_stage_lock/00_阶段实现卡.md \
  --research-record b_class_auxiliary/coding_guards/20260708_03_unet_stability_research/研究定标记录.md \
  --precheck-guard b_class_auxiliary/coding_guards/20260708_03_unet_stability_stage_lock/20260708_03_unet_stability_pre_check_guard.md \
  --post-qc-guard b_class_auxiliary/runtime_checks/post_qc_guard.md \
  --experiment-config configs/experiment/A2_UNet_GlaS_v1_seed3407.yaml \
  --runtime-split testA
```
- 内部自动串联 research → stage_definition → precheck → runtime_check → code_quality → workflow 六步，无需单独跑各步。
- **只有 `--stage-card` 是必填**，其余四个（research-record / precheck-guard / post-qc-guard / experiment-config）默认值为空会让对应步骤 `blocked`，所以必须显式给全 6 个。
- 报错读法：报 `missing_xxx` 即缺对应参数；报 `blocked_by_xxx` 是被上一步阻断，修好上一步即可。
- 产物：`b_class_auxiliary/runtime_checks/workflow_gate_report.md`，末尾应为 `workflow_gate_status=pass`（报告的 `command:` 字段会记录本次完整命令，可直接照抄还原）。

## 六、常见坑

- **F1 是对象级不是像素 Dice**：主表 F1（testA 0.5291 ± 0.0653；testB 0.5865 ± 0.0177）、Object Dice、像素 Dice 是三种口径，看结果别张冠李戴。历史文档曾把 F1 误写成像素 Dice（0.8908/0.8854）；该旧数字仅作历史更正记录，不得当前消费。
- **summarize_stage 两种模式**：03 阶段用 `--stage a2`（多 seed 聚合）；02 阶段用 `--config`（单 run）。用错模式聚合结果不对。
- **改 F1 口径不需要重训**：checkpoint 由 `val_objdice_max` 选出，与 F1 无关；只需重跑步骤 1（test）+ 步骤 2（聚合）即可刷新 F1 列。
- **gate 六参数要给全**：漏任一个都会 blocked。别只给 `--stage-card` 就跑。
- **别把 `__runtime_probe` 目录当第 4 个 seed**：`experiments/A2_UNet_GlaS_seed3407__runtime_probe/` 只是工具链探针目录，不承担正式训练身份。
- **三 seed 必须共用同一份 eval_proto_v1.yaml**：协议红线字段（阈值/connectivity/cast policy）不一致会让 `proto_consistent` 变 false，聚合失去可比性。
- 命令必须在 `crc_gland_segmentation_project/` 目录下跑（不是外层 `Paper/`）。
