# 增强对比进入条件

这份文件不是给 `08_外部对比` 再加一页“有空再跑更强模型”的备忘录，也不是把 `nnU-Net / TransUNet / Swin-Unet` 先记进候选池、以后看心情再决定。

它在当前阶段中的唯一职责是：

> 作为 `08_外部对比` 阶段的增强外部准入门控协议，把 `nnU-Net / TransUNet / Swin-Unet` 这类更强、更重、更难公平适配的方法，何时允许进入、进入前必须满足哪些阶段门、进入后必须如何记录最小必要差异、何时必须止损退出，一次写死。后续代码、阶段总结、主表草稿和下游 handoff 只允许忠实执行本文件，不允许在首批外部层尚未收口时，一边扩名单一边重新解释为什么这些增强外部现在值得跑。

从现在开始，增强外部不是默认主线，而是 `结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/00_阶段总协议.md` 和 `结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/01_统一公平协议.md` 之下的条件触发分支。若本文件与这两个上位协议冲突，以它们为准回改本文件。

## 1. 文件角色与执行边界

### 1.1 当前文件负责什么

当前文件只负责冻结下面八件事：

1. `增强外部对比` 在 `08_外部对比` 阶段中的唯一身份，也就是“首批外部层收口后，按必要性补强说服力的条件分支”，而不是默认必跑层。
2. `nnU-Net / TransUNet / Swin-Unet` 的默认候选池、优先级和最多允许进入的数量上限。
3. 增强外部正式进入前必须同时满足的阶段门、资源门、实现可信度门和公平协议门。
4. 增强外部进入后必须继承的统一数据、训练、评估、结果记录和来源标记协议。
5. `enter_reason / stop_reason / entry_decision / resource_budget_ok / implementation_trustworthy` 等一级决策字段的强制留痕。
6. 增强外部结果进入主表、补充表、参考说明区或被直接止损的唯一分流规则。
7. 当前门控协议在代码与结果层中的入口函数、决策文件、最小交付物和下游 handoff 资产。
8. 当前文件自身的独立 `回退条件`、`冲突裁决记录`、`文件质量自检` 和独立 `Diagnostics 闭环`。

### 1.2 当前文件不负责什么

当前文件不重新定义下面这些内容：

- `current_mainline` 的来源；它只能由 `07_Distance` 的正式结论唯一给出。
- 首批外部对比 `UNet++ / DeepLabV3+ / Attention U-Net` 的实现细则与最小必要差异。
- `reproduced / † / *` 的表注体系本体。
- 外部主表第一层是否已经形成；那由 `结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/07_阶段验收.md` 和阶段汇总共同裁决。
- `CRAG` 阶段和 `10_结果汇总` 阶段对增强外部结果的最终写作使用方式。

### 1.3 为什么当前文件必须独立存在

增强外部最容易出现的伪完成状态不是“完全没跑”，而是：

- 首批外部层还没收口，就提前把 `nnU-Net / TransUNet / Swin-Unet` 当成新的主要工作量。
- 因为增强外部本身更重、更强，就默认放宽统一公平协议。
- 只说“需要更强说服力”，却没有把“为什么现在必须补、补哪个、补到什么程度、何时止损”写成硬规则。
- 跑出一个增强外部数字后，既没有明确它是否具备主表资格，也没有说明它只是补充对比还是正式正文证据。

如果没有一份独立门控协议，后续 `09_CRAG验证`、`10_结果汇总` 和 `11_总验收与止损` 就无法判断增强外部到底属于：

- 合法进入的补强证据；
- 仅作补充表说明；
- 还是应该直接止损退出的高成本分支。

## 3. 本阶段唯一允许处理的变量

### 3.1 当前文件真正要证明什么

当前文件要证明的不是“增强外部有文献价值”，而是：

> 在 `current_mainline` 已冻结、首批外部层已成型、任务内 direct comparison 已经补足基本说服力边界之后，增强外部是否仍然有必要作为额外补强层进入；如果进入，它必须以统一协议下的最小必要差异方式存在，并且有明确的进入理由、退出理由和结果分流位置。

### 3.2 当前文件唯一允许变化的变量

本文件唯一允许处理的核心变量固定为：

```text
plus_external_entry = persuasion_need + resource_budget + implementation_trust + fairness_adaptability
```

其中：

- `persuasion_need`：首批外部层和任务内直比层之后，当前主线说服力是否仍明显不足。
- `resource_budget`：时间、显存、算力、存储和整理成本是否允许继续扩。
- `implementation_trust`：是否有可信实现来源，而不是高风险重造。
- `fairness_adaptability`：是否能纳入统一协议，或者至少能把最小必要差异说清。

### 3.3 当前默认结论

当前默认结论固定为：

- 增强外部不是默认必做项。
- 只有在首批外部层和任务内 direct comparison 都已经成型后，才允许讨论增强外部。
- 如果任一关键门未通过，当前正式结论应明确写为 `不进入增强外部对比`，而不是保留模糊的“后面再看”。

### 3.4 增强外部正式进入门

当前文件把增强外部进入门固定为：

```text
Gate_H1_plus = current_mainline_ready
               AND core_external_ready
               AND external_main_table_ready
               AND task_specific_direct_ready
               AND persuasion_gap_still_exists
               AND resource_budget_ok
               AND implementation_trustworthy
               AND fairness_adaptable
```

各子门解释如下：

- `current_mainline_ready`：`Distance` 结论已收口，当前主线唯一合法。
- `core_external_ready`：`UNet++ / DeepLabV3+ / Attention U-Net` 已形成最小闭环。
- `external_main_table_ready`：首批外部主表草稿已经形成。
- `task_specific_direct_ready`：任务内 direct comparison 区域已经不缺位。
- `persuasion_gap_still_exists`：当前主线与第二强方法差距仍不足以支撑投稿说服力。
- `resource_budget_ok`：时间、算力、存储和整理成本足以承担增强外部。
- `implementation_trustworthy`：能找到可信实现来源，而不是高风险从零重写。
- `fairness_adaptable`：该方法能纳入统一协议，或最小必要差异可以被严格记录。

### 3.5 增强外部不进入门

只要下面任一条件成立，当前正式结论就固定为：

```text
plus_external_denied = NOT core_external_ready
                       OR NOT external_main_table_ready
                       OR NOT task_specific_direct_ready
                       OR persuasion_gap_already_clear
                       OR resource_budget_not_ok
                       OR implementation_not_trustworthy
                       OR fairness_gap_too_large
```

这类结论不是“暂时不做”的弱口吻，而是正式止损结论。

## 4. 阶段门控表达式

### 4.1 当前允许进入候选池的方法

当前增强外部候选池只允许包括：

- `nnU-Net`
- `TransUNet`
- `Swin-Unet`

除非上游路线层和阶段母协议同时升级，否则当前文件不接受新增第四类增强外部候选。

### 4.2 当前固定优先级

当前优先级固定为：

1. `nnU-Net`
2. `TransUNet`
3. `Swin-Unet`

### 4.3 当前优先级的固定理由

- `nnU-Net`：最能补“强 pipeline baseline”的说服力，也最能校准“是否是协议不够强，而不是方法不够强”。
- `TransUNet`：最能补“CNN vs Transformer hybrid”比较，适合在需要更强结构路线对照时进入。
- `Swin-Unet`：更偏 pure Transformer U-shape 的补充路线，优先级低于前两者。

### 4.4 当前最多允许补多少

当前增强外部最多只允许补：

- `1-2` 个

不允许因为“都很经典”而把 `3` 个全部同时转成正式执行任务。

## 8. 回退条件

### 8.1 独立回退触发条件

下面任意一条成立，都必须先回退修订本文件，而不是继续推进增强外部执行：

- 首批外部层尚未完成最小闭环，却已经尝试放行增强外部。

- 资源预算、实现可信度或公平适配边界仍然模糊。
- 增强外部进入后仍没有明确的主表 / 补充表 / 参考区分流规则。
- “不进入”与“进入”两种状态都没有留下结构化字段和正式 decision note。

### 8.2 固定回退顺序

回退时统一按下面顺序排查：

1. 先检查 `current_mainline`、首批外部层和任务内直比层是否真的已收口。
2. 再检查说服力缺口是否真实存在，而不是主观焦虑。
3. 再检查资源预算和实现可信度是否真的足够。
4. 最后检查统一公平协议能否覆盖该增强外部，或最小必要差异是否可被严格记录。

### 8.3 回退后的重新放行条件

发生回退后，只有同时满足下面条件，才允许重新声明本文件通过：

- 问题来源已经按固定回退顺序定位并修复。
