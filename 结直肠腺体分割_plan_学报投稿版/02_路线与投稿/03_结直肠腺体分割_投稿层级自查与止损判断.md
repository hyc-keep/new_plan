## 2. 投稿裁决的统一证据语义

### 2.1 数据和盲态

- screening 只使用 `train68 / val17`；模型、loss、threshold、轮廓和 watershed 参数只能在 val 锁定。
- `GlaS TestA / TestB / CRAG test` 全部盲态，最终配置和 checkpoint 锁定前不得读取、比较、调参或触发回退。
- TestA、TestB 必须分开列报，CRAG test 单列；不得用合并平均掩盖数据集间差异。

### 2.2 指标身份

主指标：`F1 / Object Dice / Object Hausdorff`。补充指标：`Dice / IoU / HD95 / Boundary F1`。全项目唯一 Boundary F1 身份为：

`project_custom_erosion_xor_3x3_ones_8conn_border0_tol3px`

写作、表格和复核时必须同时显示实现身份；同名不同实现不得直接横比。任何 direct comparison 都必须标为 `reproduced` 或 `quoted from original paper`（引用值使用 `†`），协议不同则降级为参考，不伪装公平比较。

## 3. 学报写作的最低证据包

“允许写作”不等于“保证录用”。进入学报正式写稿前，必须同时具备：

1. B1 只读资产、lineage、checkpoint 和评估身份可回查。
2. GlaS screening 的 train68/val17 运行记录完整，且无盲态泄漏。
3. B1/N1/N2/N3 的预注册矩阵、实际状态、配置和回退关系清楚；未运行版本不得进入结果表。
4. N3（若被验收）必须有三 seed `3407/1234/2025` 的独立 bundle、mean±std 统计语义和失败案例。
5. GlaS TestA/TestB 分开结果和 CRAG test 盲态结果均有 manifest、checkpoint、评估脚本身份；没有就不得宣称泛化。
6. 主线消融表、对象级/边界级表、代表性可视化和失败案例图能够由正式资产重建。
7. Boundary F1 明确采用唯一 identity；Object Dice/Object Hausdorff 的对象匹配规则和统计方式写入表注。
8. 任务内 direct comparison 至少有来源、benchmark、split、指标口径和 `reproduced/quoted` 标签；不满足等价条件的放入讨论或参考表。
9. 中文方法、实验、局限和结论均能只使用已验收结果落笔；不得把预期方向写成实际提升。
10. 成本、参数、统计语义和失败模式齐全，能解释每一阶段的职责，而不是只给 Dice。

## 4. 投稿层级判断（非承诺性门控）

### 4.1 学报保底收口

当最低证据包齐全、主线至少有一个通过验收的当前版本、图表和失败案例可审计，且结论不过度外推时，可进入中文学报写作。若 N1-N3 均未运行，不能进入正式结果写稿，只能写计划/方法草案。

学报稿必须明确：任务范围、B1 历史只读限制、当前路线的机制假设、TestA/TestB/CRAG 盲态边界、指标 identity、统计与失败案例。学报层级是当前救援优先出口，不表示录用概率或指标下限。

### 4.2 普通英文应用期刊起步

仅当学报最低证据包之外，结果已具备可复核的主线机制证据、GlaS 与 CRAG 的合理解释、通用外部对比和任务内 direct comparison 的最小集合、对象级/边界级证据及不过度夸张的英文写作条件时，才可选择英文起步。不能以增加模块数量替代 direct comparison、盲态测试或失败分析。

### 4.3 暂不写稿

出现以下任一情况，结论只能是“暂不进入正式写稿，回到实验/证据修复”：B1 lineage 或指标身份缺失；N1-N3 状态混乱；baseline/当前版本不可复现；测试集被调参；主表依赖未运行结果；CRAG test 未盲态；关键图表无法从正式资产重建；或结果解释只剩像素 Dice。

## 5. N3 机制门与 Plan B 裁决

N3 只在 N2 验收通过后进入。先在 val17 预注册并锁定 marker、轮廓阈值、连通性、距离和最小对象规则，再执行 TestA/TestB/CRAG test。只有当 N3 在 val 显示轮廓生成/marker/分水岭机制系统性失效，且排除了 target、代码和评估错误，才记录 `val_mechanism_gate.json` 并触发 Plan B。

Plan B 必须实现并准确命名为 `DCAN-inspired object-contour joint inference adaptation`，独立记录与原文的差异、状态、配置、seed、checkpoint、指标和失败案例；不得声称 strict replication。没有机制门，不得运行；运行后不能因 TestA/TestB/CRAG test 不理想而临时触发。N3 与 Plan B 均失败时回到 N2 或 B1，不扩大路线。

## 6. 模块/阶段止损规则

- **N1**：轮廓 target、联合监督或实现身份无法验证时停止；修复后仍不成立则保留 B1，不进 N2。
- **N2**：区域/轮廓融合成本不可解释、轮廓证据丢失或 val 无机制支持时删除融合，独立回 N1。
- **N3**：marker/分水岭造成不可解释过分割、欠分割或多 seed 机制不稳定时，不继续调大后处理空间；按机制门考虑 Plan B。
- **Plan B**：仅门控运行；不通过则回到 N2/B1，不把它写成默认主线。
- **任何阶段**：盲态泄漏、指标 identity 漂移、配置/manifest/run_meta 不同步、无法回溯 checkpoint 时，当前结果作废并回到最近未泄漏阶段。
- **外部对比**：复现成本失控时降级为有来源的引用对比，不拖慢主线，也不把引用值当作统一协议结果。

## 7. 投稿主表、图表与写作交接

至少准备：

- 内部主线表：B1（历史只读）与实际已验收的 N1/N2/N3/Plan B；未运行行明确标 `planned/not_run`，不得填数字。
- GlaS TestA、TestB 分开主结果表；CRAG test 独立表。
- 对象级与边界级指标表，显式写 Boundary F1 identity、对象匹配和 mean±std 语义。
- 任务内 direct comparison 表：来源、协议差异和 `reproduced/quoted` 标记齐全。
- 图像样例、轮廓/黏连分离、成功/失败案例、成本或复杂度图；均须能从 artifact manifest 重建。

论文段落的可写边界：未运行阶段只能写“拟采用/拟验证/ planned”；只有 accepted run bundle 才能写“观察到/结果显示”；不能写“显著提升”“优于所有方法”“保证录用”等无证据结论。

## 8. 运行字段与验收记录

每个投稿候选版本必须具有：`run_name`、`stage`、`status`、`planned_or_actual`、`source_stage`、`source_manifest`、`source_protocol_version`、`source_run_name`、`consumer_stage`、`consumer_file`、`consumption_boundary`、`dataset/split`、`blind_state`、`seed`、`config_digest`、`code_commit_or_digest`、`checkpoint`、`metric_identity`、`val/test metrics`、`runtime/cost`、`failure_cases`、`decision`、`rollback_target`、`artifact_manifest`。缺任一关键字段，投稿 gate 为 `blocked`。

验收需由独立记录完成：数据和盲态、配置一致性、指标 identity、对象/边界证据、可视化、失败案例、成本、统计和 lineage 逐项核对。投稿判断只消费 `accepted` 资产，不消费截图、临时 notebook 输出或口头结论。

## 9. 冲突裁决记录

1. 当前路线与旧 LKMA→Boundary→Distance 冲突时，以 B1→N1→N2→N3 为唯一当前主线，旧结果保留为历史来源。
2. `valid_with_stability_warning` 不得被改写成“已稳定”或“当前最佳”；它的限制必须进入方法/局限表述。
3. 原 04 std gate 只做内部稳定性参考，不转换成投稿门槛、指标预承诺或录用保证。
4. screening 与盲态测试冲突时，train68/val17 负责选择，TestA/TestB/CRAG test 只在配置锁定后一次性评估。
5. 文献引用值与复现值冲突时分栏并加来源标记；协议不等价不得做直接排名。
6. 投稿层级冲突时，证据完整性优先于主观信心：不足则先学报保底或暂不写稿，不因焦虑扩模块。

## 10. 独立回退条件

- **协议回退**：数据、split、指标、blind_state、lineage 或 run bundle 不完整，回到 pre-check，不进入写作。
- **主线回退**：N1 不成立回 B1；N2 不成立回 N1；N3 机制失败才允许 Plan B；Plan B 失败回 N2 或 B1。
- **投稿回退**：图表、CRAG、TestA/TestB 分开结果或 direct comparison 不足，回到学报证据包整理；若连最低包都不齐，暂不写稿。
- **表述回退**：无法由 accepted 资产支持的“提升/泛化/优于”全部改成假设、观察范围或局限，不补写数字。
