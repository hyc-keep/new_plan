## 7. 第三阶段：代码修复清单

### 7.1 checkpoint/resume

修改 checkpoint schema，至少保存：

- epoch
- model state
- optimizer state
- scheduler state
- GradScaler state
- early stopper state
- Python random state
- NumPy random state
- Torch CPU RNG state
- CUDA RNG state（CUDA 可用时）
- DataLoader generator state
- run_name
- seed
- config version
- best metric、best epoch
- checkpoint schema version

resume 时必须：

- 校验 run_name、seed、config version；
- 恢复全部训练状态；
- 将 optimizer state 移到目标 device；
- 处理早停边界，避免多跑一轮；
- 过滤并立即重写孤儿 CSV 行；
- 记录 resume source checkpoint 和恢复状态。

### 7.2 runtime probe

修改 runtime 工具和训练入口：

- 强制 probe run_name 与正式 run_name 不同；
- 在进入 runtime 分支前拒绝正式 run_name；
- 从配置解析并记录 `model_name=resnet34_unet`；
- 记录最终 factory 路由和模型模块路径；
- 在 CUDA 可用时默认使用 CUDA；
- runtime payload 显式记录 amp_requested、amp_active、grad_scaler_used、model_identity；
- CPU fallback 必须标记为不能证明 CUDA AMP 正式链路；
- 只有覆盖正式 profile 的 runtime 才能标记 `full_training_runtime`。

### 7.3 模型接口

- 对 GlaS 固定 `in_channels=3` 做显式断言；
- 若要支持其他通道，替换并初始化 ResNet `conv1`，同时定义预训练权重适配策略；
- 记录 decoder 完整结构；
- 记录预训练权重版本、URL/本地路径和 SHA256；
- 记录 PyTorch/torchvision 版本。

### 7.4 结果绑定

测试脚本必须在输出中写入：

- run_name

- model identity
- checkpoint relative path
- checkpoint epoch
- checkpoint metric value

### 7.5 参考代码和指标

- 固定 GlandSegBenchmarks 参考实现的 commit 或本地文件 hash；
