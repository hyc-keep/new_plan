## 5. 阶段 02：A1 U-Net 流程验证

```text
runtime probe: A1_UNet_GlaS_seed3407_chain__runtime_probe

```

### 5.3 执行顺序

1. **隔离 runtime**：运行一个真实训练 step，输出独立 runtime 三件套；不得复用正式 run 名。
2. **smoke**：以同一配置和独立 smoke run 验证最小训练、验证、checkpoint 和测试入口可连通。
3. **冻结**：runtime 和 smoke 都通过后，记录当前 commit、config hash、数据 freeze manifest 与环境信息；开始正式运行后不得改代码或配置。
4. **正式训练**：只运行正式 A1 run；由训练脚本写 checkpoint、`run_meta.yaml`、训练日志与验证日志。
5. **正式测试**：只读取本轮 A1 best checkpoint，生成 testA/testB sample CSV、预测、GT、独立复核记录。
6. **收口**：检查 run identity、checkpoint SHA256、样本数、CSV schema 和独立复核；生成 A1 manifest，才可向 03 交接。

### 5.4 禁止事项

- 不得使用历史 `A1_UNet_GlaS_v1_seed3407` 作为本轮输入或结果。
- 不得把 runtime probe/smoke 当作正式 A1。
- 不得在正式 A1 已开始后修改代码、配置、数据、指标口径或环境。
- A1 失败时不得启动 A2。

## 8. 每次命令前后的检查清单

### 命令前

1. 确认当前阶段前置 manifest 和 Gate 为真实可读状态；
2. 确认本阶段 config、seed、run name 和 `experiment_root` 与 contract 一致；
3. 运行全链 checker；
4. 确认本批次根不存在，或当前操作是允许的 resume；
5. 设置配置对应的 `PYTHONHASHSEED` 与 `CUBLAS_WORKSPACE_CONFIG=:4096:8`；
6. runtime 必须使用 `__runtime_probe`，smoke 必须使用 `__smoke`，均不可使用正式 run 名。

### 命令后

1. 记录命令退出码，不得忽略失败；
2. 读取脚本生成的真实输出路径；
3. 对正式训练确认 `run_meta.yaml`、best checkpoint、checkpoint SHA256、训练/验证日志；
4. 对测试确认 testA/testB sample CSV 的样本数和身份字段；
5. 运行独立复核与阶段聚合/Gate；
6. 仅将真实状态、路径、blocker 同步到本文相应小节；
7. 每个完成阶段后提交并推送 Git；不提交数据集、checkpoint、实验输出或调试文件，除非另行明确决定。
