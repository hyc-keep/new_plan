## 0. 先把锁定边界说清楚

### 0.1 这份文档现在锁死什么

这份文档负责锁死的是路线层内容，而不是训练脚本里每一个超参数。当前在这里直接锁死的只有下面八类：

1. 任务边界：`2D` 结直肠组织病理腺体分割
2. 数据边界：`GlaS` 为主，`CRAG` 为第二 benchmark
3. baseline 阶梯：`UNet -> ResNet34-U-Net -> 外部对比`
4. 方法主线：`LKMA -> Boundary Head -> Distance-aware Loss(可选)`
5. 指标主线：`F1 / Object Dice / Object Hausdorff` 为主，`Dice / IoU / HD95 / Boundary F1` 为辅
6. 投稿层级：先完成中文保底主干，同时保留“结果够强时直接按 `SCI` 低区组织”的判断口
7. 外部对比分层：哪些必跑，哪些选跑，哪些只引用
8. 模块去留原则：必须由多 seed、对象级或边界级指标、可视化和成本共同决定

### 0.2 这份文档明确不在这里锁死什么

下面这些内容后面一定要定，但不应该在路线层直接写成永久固定值：

- 最终 `batch size`
- 最终训练 epoch 实际停在哪一轮
- `LKMA` 的最终插入位置、核大小和实现细节
- 边界标签的最终生成算法、边界宽度和边界损失权重
- 距离图的最终生成方式和 `lambda_dist`
- 外部增强对比最终到底保留哪 `1-2` 个

原因很简单：

- 这些内容如果现在写死，后面很容易和真实训练稳定性冲突
- 这些内容如果一直不写，执行阶段又会摇摆

所以正确做法不是“不锁”，而是：

- 现在先锁“允许的候选范围”
- 等 `UNet` 跑通、主 baseline 稳定后，再在执行层锁成最终协议

- 默认主 loss 先按 `BCE + Dice`

- 默认主线顺序不允许改成“先边界后形态”或“先距离再边界”

- 保证路线不散
- 保证后面执行时不是从零开始想
- 保证执行层只是在小范围内定最终协议，而不是重新找方向

## 1. 一页结论

### 1.1 当前最推荐的主路线

- 任务固定：`2D` 结直肠组织病理腺体分割
- 主数据集固定：`GlaS`
- 第二 benchmark 固定：`CRAG`
- 流程排雷模型固定：`UNet`
- 主 baseline 固定：`ResNet34-U-Net`
- 主方法固定表达：`形态感知大核建模 + 边界分离辅助监督`
- `Distance-aware Loss` 固定定位：`可选增强项，不预设一定保留`
- 主实验顺序固定：`UNet -> baseline -> + LKMA -> + Boundary Head -> + Distance-aware Loss(可选) -> 投稿第一批外部对比 -> CRAG -> 投稿增强对比`

### 1.2 当前最值得优先追求的最终模型

默认优先级按下面顺序判断：

1. 首选：`Baseline + LKMA + Boundary Head`
2. 次选：`Baseline + LKMA + Boundary Head + Distance-aware Loss`
3. 回退：`Baseline + Boundary Head`
4. 再回退：`Baseline + LKMA`

一句话理解：

> 先把整体形态建模和对象边界分离做扎实，再判断距离约束有没有额外收益；不要把一开始就把第三个模块预设成必须保留。

### 1.3 当前最现实的投稿目标

- 默认收口目标：`中文学报 / 应用型中文期刊`
- 默认升级目标：`普通英文应用期刊 / SCI 低区可行目标`
- 当前不建议一开始就按更高难度英文目标把全部实验包做得过重

更准确的理解不是“这条线只能做中文”，而是：

- 在计划阶段，默认先按中文保底的闭环来组织资源
- 在结果判断阶段，如果主线提升清楚、任务内 direct comparison 够强、`CRAG` 不崩，就允许直接按 `SCI` 低区 / 普通英文应用期刊组织全文
- 如果结果只是稳但不惊艳，再按中文保底收口并追加小幅英文增强

### 1.4 这条路线为什么是高性价比路线

基于现有文献，当前最稳的不是追最重的 foundation model，而是追一条“故事完整、实现成本可控、结果可预测”的路线。并且近期直接竞品已经证明，这一级别的任务导向结构改进在真实投稿中是有空间的。原因主要有七个：

- `GlaS` 和 `CRAG` 已经足够支撑一条完整腺体分割论文主线
- 腺体任务的核心痛点已经被文献收束到：`形态变化大 + 对象黏连 + 边界模糊`
- `ResNet34-U-Net` 作为主 baseline 成本低、可复现性高、也容易和后续模块公平对比
- `LKMA` 和 `Boundary Head` 分别对应形态建模与对象分离，逻辑顺序清楚
- `Distance-aware Loss` 在文献中更像增强项，放在最后判断最符合收益预期
- 这条路线更容易形成“方法设计 -> 指标提升 -> 可视化支撑 -> 写作表达”闭环
- `MAC-Net`、`DEA-Net` 这类近期工作说明：围绕 morphology、attention、boundary recovery 的腺体任务路线仍然能形成可投稿故事，关键在于差异化表达和证据链强度

## 2. 文献已经帮你锁定了什么

### 2.1 已被文献锁定的任务边界

根据 `GlaS`、`MILD-Net`、`TA-Net`、`DEA-Net` 等文献，当前任务边界已经足够明确：

- 主任务不是通用病理分割，而是 `gland segmentation`
- 主难点不是简单前景分割，而是：
  - 恶性腺体形态变化大
  - 相邻腺体容易黏连
  - 边界模糊且实例分离困难
- 这意味着你的主方法不应该写成“更强 backbone”，而应该写成“更好处理 gland morphology 和 boundary separation”

### 2.2 已被文献锁定的 baseline 结构

根据 `U-Net`、`ResNet`、`UNet++`、`DeepLabV3+`、`Attention U-Net`、`nnU-Net`、`TransUNet`、`Swin-Unet` 等文献，当前实验主表基础模型已经足够明确：

- 最小流程基线：`UNet`
- 主 baseline：`ResNet34-U-Net`
- 投稿主表必跑对比：`UNet++ / DeepLabV3+ / Attention U-Net`
- 投稿增强对比：`nnU-Net / TransUNet / Swin-Unet` 中择优 `1-2` 个

这意味着你现在不需要再花时间讨论“到底该用什么 baseline”，而是该讨论：

- 先跑哪几个
- 哪些必须跑
- 哪些只引用不复现

### 2.3 已被文献锁定的主改进方向

根据 `Large Kernel Matters`、`RepLKNet`、`WTConv`、`DCAN`、`Gated-SCNN`、`Distance Map Loss`、`TA-Net`、`DEA-Net` 等文献，当前主改进方向已经基本定住：

- 第一层改进：`LKMA / 大核形态建模`
- 第二层改进：`Boundary Head / 边界辅助监督`
- 第三层改进：`Distance-aware Loss / 距离约束`

这里最重要的不是“模块越多越好”，而是：

- 第一层负责补整体形态与感受野
- 第二层负责补对象边界与黏连分离
- 第三层只是在边界附近再做额外约束

### 2.4 已被文献锁定的指标体系

根据 `GlaS challenge`、`MILD-Net`、`TA-Net` 和辅助层里边界指标文献，当前指标体系可以固定为：

- 主指标：`F1 / Object Dice / Object Hausdorff`
- 补充重叠指标：`Dice / IoU`
- 边界补充指标：`HD95 / Boundary F1`

这意味着你后面不应该再把故事写成“Dice 提升了所以方法有效”，而要写成：

- 对象级结果有没有改善
- 边界级结果有没有改善
- 黏连腺体分离是不是更好

### 2.5 当前必须正面对齐的直接竞品

按目前已经补到位的文献证据，下面这些不是“可有可无的 related work”，而是你在路线层就必须心里有数的直接参照：

- `MAC-Net`：近期 attention + context fusion 路线，说明类似量级的结构改进在真实期刊里仍有发表空间
- `DEA-Net`：近期 gland-specific 的 dual encoder + boundary-enhanced attention 路线，是你当前主线最直接的任务内强对照
- `MILD-Net-RTS / TA-Net / SDT`：分别对应经典任务强基线、topology-aware 路线和 skeleton-aware 强路线，是后续主表必须正面对话的对象

这一步的意义不是把自己吓住，而是把差异说清：

- 你不是做“泛 attention 拼装”
- 你要做的是“单主干下的形态感知大核建模 + 边界分离辅助监督”
- 你的优势候选应当落在：更轻、更好解释、对象级和边界级证据更成体系

## 3. 最终方法故事应该怎么讲

### 3.1 不要讲成“堆模块”

你现在最应该避免的写法是：

- baseline 上加一个大核模块
- 再加一个边界分支
- 再加一个 distance loss

这种写法的问题是：

- 容易被看成零散拼装
- 每个模块都很难讲清主次
- 一旦其中一个模块收益小，整条路线就会变散

### 3.2 应该讲成一条连续证据链

当前最合理的主叙事是：

1. 腺体分割首先受限于整体形态和上下文建模不足
2. 因此先用 `LKMA` 提升形态感知和感受野表达
3. 但仅有更大感受野还不够，腺体黏连与边界模糊仍然存在
4. 因此再引入 `Boundary Head` 做边界辅助监督和对象分离
5. 最后再检验 `Distance-aware Loss` 能否作为轮廓附近的补充增强

这样写的好处是：

- `LKMA` 和 `Boundary Head` 各有明确职责
- `Distance-aware Loss` 不会被错误写成主贡献
- 就算最后删掉 distance 项，整条主线也不会塌

### 3.3 当前最稳的创新边界

当前项目更适合追求的是“实用型、组合型、任务导向创新”，而不是“提出一个颠覆式新网络”。更合适的贡献边界是：

- 把 gland morphology 和 boundary separation 组织成一条更统一的设计链
- 在 `GlaS + CRAG` 上给出稳定、对象级和边界级都支持的证据
- 通过消融说明每一层改进是否值得保留

不建议硬写的贡献：

- “提出全新 SOTA 大模型结构”
- “强泛化到所有病理分割任务”
- “距离约束是整篇文章最核心创新”

## 4. 投稿版实验名单怎么锁

### 4.1 内部主线实验

这部分必须跑完整，不是可选项：

1. `UNet`
2. `ResNet34-U-Net`
3. `Baseline + LKMA`
4. `Baseline + LKMA + Boundary Head`
5. `Baseline + LKMA + Boundary Head + Distance-aware Loss`

如果 `Distance-aware Loss` 不过线，最终主模型就回退到第 `4` 个版本。

### 4.2 投稿主表第一层：通用外部必跑对比

第一版主表建议只锁三类：

1. `UNet++`
2. `DeepLabV3+`
3. `Attention U-Net`

这样做的好处是：

- 三类方法代表性足够
- 复现和对比成本还能控制
- 和你的主线有天然可比性

### 4.3 投稿主表第二层：任务内直接量化对照

这一层和上面那三类通用外部对比不是一回事。

按当前 `05/06` 的主结果表口径，下面这些 gland-specific 方法不应该再只停留在 related work 里，而应作为主表里的直接量化对照对象：

- `GlaS` 优先对齐：`DCAN / MILD-Net-RTS / DEA-Net / SDT`
- `CRAG` 优先对齐：`MILD-Net-RTS / TA-Net / DEA-Net`

这里要特别注意两点：

- 最理想状态当然是统一协议下能复现多少就复现多少
- 但就算其中某些方法因为代码、时间或算力原因不做统一复现，也至少必须作为任务内 direct comparison 明确进入主结果表，不能只留在 related work 和 discussion

换句话说：

- `UNet++ / DeepLabV3+ / Attention U-Net` 负责补“通用外部基线代表性”
- `DCAN / MILD-Net-RTS / DEA-Net / SDT / TA-Net` 负责补“腺体任务内直接比较说服力”

这层对照不再是“英文增强时再想”的事，而是路线层现在就要预留的强约束：

- 中文稿可以先完成通用外部对比并同步整理任务内 direct comparison 引用值
- 一旦主线结果足够强，英文稿就直接把这层对照抬进主表
- 不允许到写稿时才发现自己只和通用语义分割网络比较过

### 4.4 高阶增强对比

如果时间、算力、代码条件允许，再从下面只选 `1-2` 个：

- `nnU-Net`
- `TransUNet`
- `Swin-Unet`
- 一个近两年直接在 `GlaS / CRAG` 上验证的腺体分割工作

推荐优先级：

1. `nnU-Net`
2. `TransUNet`
3. `Swin-Unet`

原因：

- `nnU-Net` 最能补“强基线与流程公平性”
- `TransUNet` 最能补“CNN vs Transformer”
- `Swin-Unet` 更像锦上添花，不是第一必要项

### 4.5 只引用不强求复现的文献组

当前更适合保留在 related work 和 discussion 里的论文包括：

- `SCAU-Net`
- `AttentionBoost`
- `Prompted SAM-H`
- 其他复现代价高、指标口径不完全一致或更适合写扩展讨论的 foundation model / zero-shot 路线

它们的核心价值不是一定都要复现，而是：

- 说明任务发展脉络
- 说明你的方法和边界 / 拓扑 / attention / gland-specific 路线的关系
- 提醒你避免把模型写成现有近期方法的近似复刻

## 5. 结果大概应该期待到什么程度

### 5.1 可以合理期待的不是“暴涨”，而是“稳定改善”

基于现有文献，更合理的预期不是：

- 每加一个模块都大幅提升

而是：

- baseline 先稳定
- `LKMA` 带来小到中等幅度、方向稳定的收益
- `Boundary Head` 在对象级和边界级上更容易体现价值
- `Distance-aware Loss` 可能有补充收益，也可能收益接近波动范围

### 5.2 更可能出现收益的地方

相对更值得期待的改善位置是：

- `Object Dice`
- `Boundary F1`
- `HD95` 或 `Object Hausdorff`
- 黏连腺体分离的局部可视化

相对不该过度期待的地方是：

- 单纯像素 Dice 出现特别夸张提升
- `CRAG` 上完全无代价保持和 `GlaS` 一样的收益幅度

### 5.3 最终更像保底稿件的结果形态

如果最后拿到下面这种结果形态，就已经比较像“高性价比保底稿件”：

- `UNet` 稳定
- `ResNet34-U-Net` 稳定
- `LKMA` 有明确结论
- `Boundary Head` 有对象级或边界级证据
- `Distance-aware Loss` 有结论，保留或删除都能说清
- `CRAG` 不崩，且最终模型相对 baseline 的优势没有完全消失
- 中文方法和实验两章能顺畅写出来

## 6. 当前不建议做的事

### 6.1 不建议一开始就押重模型

暂时不建议把主线改成：

- `nnU-Net` 为全文主 baseline
- `TransUNet` 或 `Swin-Unet` 为全文中心
- `SAM` / prompt / diffusion 路线为当前主稿主线

原因不是这些方向没价值，而是：

- 复现和调参成本更高
- 更容易拖慢闭环
- 对“保底可发”帮助不一定比当前路线更大

### 6.2 不建议把论文主线写成“泛化”

当前更合适的写法是：

- 主结论基于 `GlaS`
- `CRAG` 是同任务补充验证
- 不要把 `CRAG` 一组结果过度写成“强泛化到其他病理场景”

### 6.3 不建议继续大面积补路线 PDF

从路线角度看，当前最需要补的不是“大量路线论文”，而只是后续一个小规模定制包：

- 目标期刊近两年相似论文
- 任务内直接竞品的最终引用页和表格来源
- 如果要上英文，再补 `1-2` 个增强外部对比
- 如果要补写作说服力，再补少量近年 gland-specific 论文

## 7. 你接下来应该怎么理解整个项目

### 7.1 现在不是找方向阶段

当前阶段已经不是“探索有什么方向可做”，而是：

- 方向已经基本锁定
- 方法主线已经基本锁定
- 实验名单已经大部分锁定
- 指标体系已经锁定

所以后面真正该做的是：

- 用实验去验证这条路线
- 用结果去决定哪一层保留
- 用写作去收束主故事

### 7.2 剩下最重要的不是想法，而是执行质量

对这个项目来说，后面最决定结果上限的因素更可能是：

- baseline 是否稳定
- 对比是否公平
- 对象级和边界级指标是否齐
- `CRAG` 是否不崩
- 图表与失败案例是否整理好

而不是再去找一个“第四个模块”。
