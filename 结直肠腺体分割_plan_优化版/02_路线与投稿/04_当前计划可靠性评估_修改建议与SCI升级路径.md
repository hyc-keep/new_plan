# 当前计划可靠性评估、修改建议与 SCI 升级路径

## 1. 文档目的

本文件用于一次性回答下面四个问题：

1. 当前 `结直肠腺体分割_plan` 这条主路线本身是否可靠。
2. 当前计划是否足以支撑一篇中文学报或应用型中文期刊的保底稿。
3. 如果需要修改，究竟应该改哪些地方，工作量大不大。
4. 是否可以先完整实现当前计划，再在此基础上补强到 SCI 低区，而不必推翻重做。

本文件不是新的实验协议文件，也不是立即替代 `01_实验执行` 全套文档的上位母协议。它的作用是把当前计划的可信度、投稿边界、修改范围和后续升级路径一次讲清楚，避免后续反复在“要不要重做”和“是不是内容不够”之间摇摆。

---

## 2. 先给最终结论

### 2.1 总结论

当前计划总体上是可靠的，且具有明确的实验主线、合理的阶段顺序、可执行的止损逻辑和可写作的结构基础。它不是“方向错误”或“内容空洞”的计划，而是一条偏稳妥、偏工程闭环、优先服务中文保底的高性价比路线。

### 2.2 中文保底判断

如果严格按当前计划完成以下内容：

- `UNet / ResNet34-U-Net / final model` 的完整主线结果；
- `3-seed mean +- std`；
- `GlaS` 主结果；
- `CRAG` 补充验证；
- 对象级主指标；
- 至少一类边界证据；
- 失败案例和局部可视化；

那么当前计划是有现实机会支撑中文学报或应用型中文期刊保底投稿的。

### 2.3 SCI 低区判断

当前计划原始版本不应直接被理解为“天然 SCI 低区路线”，但它可以作为一个足够稳的中文保底主干。更重要的是，这个主干是可升级的。只要从现在开始按“后续可补强”的方式记录资产，后续完全可以在不推翻主路线的前提下，通过补充任务内直接对照、病理意义分析或更强结果包，升级到更像 SCI 低区要求的版本。

### 2.4 是否需要重做

不需要推翻重做。

更准确地说：

- 不需要重做主任务；
- 不需要重做数据主线；
- 不需要重做 baseline 和主模块顺序；
- 不需要整体重写 `01_实验执行` 全套文档；
- 只需要做少量关键口径修正和结果资产要求补强。

换句话说，当前问题不是“方向错了”，而是“当前计划更像执行方案，还需要稍微收紧为投稿方案”。

## 3. 为什么说当前计划本身是可靠的

### 3.1 任务边界可靠

当前计划把任务明确限定为：

- `2D` 结直肠病理图像腺体分割；
- `GlaS` 为主数据集；
- `CRAG` 为第二 benchmark；
- 以对象级和边界级指标为重点。

这一边界是合理的。它没有把任务扩展成“泛病理分割”，也没有在数据规模、跨任务、跨中心泛化等层面一开始就立过高主张，因此适合作为一篇以任务为中心的完整研究起点。

### 3.2 实验顺序可靠

当前计划的主顺序是：

`UNet -> ResNet34-U-Net -> LKMA -> Boundary Head -> Distance-aware Loss(可选) -> 外部对比 -> CRAG -> 结果汇总 -> 总验收`

这条链的优点是：

- 先排除流程问题，再做方法改进；
- 先固定 baseline，再加模块；
- 每次只增加一层核心因素，便于解释收益来源；
- 先有内部主线，再做外部比较；
- 先形成主结论，再进入结果汇总和写稿判断。

这是一条典型的、规范的实验推进链，而不是“想到什么加什么”的松散路线。

### 3.3 问题抓手可靠

从任务本质看，结直肠腺体分割的核心难点长期集中在：

- 腺体形态变化大；
- 邻近腺体黏连，实例分离困难；
- 边界模糊；
- 对象级结构恢复比像素级重叠更重要。

当前计划对应的模块抓手是：

- `LKMA` 对应整体形态和大感受野建模；
- `Boundary Head` 对应边界恢复与对象分离；
- `Distance-aware Loss` 对应边界附近的额外约束。

也就是说，当前计划不是无目标地堆模块，而是围绕腺体任务的长期主矛盾在设计。

### 3.4 执行闭环可靠

当前 `01_实验执行` 目录已经把项目拆成了：

- 数据协议；
- 流程验证；
- 稳定性；
- baseline；
- 模块阶段；
- 外部对比；
- `CRAG`；
- 结果汇总；
- 总验收与止损。

这种结构的优点在于，它不仅有“怎么做”，还有“什么时候停”“什么时候回退”“什么时候允许进入写稿”。这对保底中文尤其重要，因为很多项目最后不是死在想法不够，而是死在没有正式结果包、没有统一口径和没有止损条件。

因此，从工程可靠性和论文落地性看，当前计划是可靠的。

## 4. 当前计划是否足够支撑中文保底

### 4.1 结论

是的，当前计划有机会支撑中文保底。

但这里的判断是“在结果真正跑出来并形成正式资产的前提下”，不是“仅凭计划文本就自动可投”。

### 4.2 为什么足够支撑中文保底

中文学报或应用型中文期刊通常更看重：

- 研究对象明确；
- 方法设计有明确动机；
- 实验设计完整；
- 对比合理；
- 指标充分；
- 图表齐全；
- 结论克制、可解释。

当前计划在这些方面已经具备较好的基础：

- 主任务明确；
- 主 baseline 明确；
- 模块职责清楚；
- 数据集和 benchmark 合理；
- 有对象级和边界级评价意识；
- 有 `CRAG` 作为复杂场景补充验证；
- 有外部对比阶段和结果汇总阶段；
- 有总验收与中文保底条件判断。

这些内容如果全部落成真实结果，是足够组成一篇中文保底稿骨架的。

### 4.3 中文保底的真实前提

当前计划要想真正形成中文保底稿，至少要做到下面这些：

- `UNet / baseline / final` 三层核心主线完整；
- `Baseline + LKMA`、`Baseline + LKMA + Boundary`、`Baseline + LKMA + Boundary + Distance` 的逐步消融完整；
- `3-seed mean +- std`；
- `GlaS TestA / TestB` 分开报告；
- 至少一类边界证据，比如 `Boundary F1` 或 `HD95`；
- `CRAG` 至少覆盖 `UNet / baseline / final`；
- 有主结果图、边界局部放大图、失败案例图；
- 能清楚解释每个模块解决什么问题。

如果上面这些资产不齐，那么“计划合理”不等于“中文可投”。

### 4.4 中文层面的创新性判断

从中文保底角度看，当前计划的创新性是够用的，但其创新边界应当被定义为：

- 任务导向的结构改进；
- 形态建模和边界分离的组合式设计；
- 在 `GlaS + CRAG` 上形成对象级和边界级证据链；
- 通过消融和失败案例说明模块去留。

它不应被拔高为：

- 全新分割范式；
- 大模型级方法创新；
- 跨病理场景强泛化结论；
- foundation model 级突破。

也就是说，中文层面“创新够不够”的答案是：

- 作为保底中文，够；
- 前提是写作口径要克制，主张要与结果包强度匹配。

## 5. 当前计划是否天然足够支撑 SCI 低区

### 5.1 结论

当前计划原始版本不应直接判断为“天然足够 SCI 低区”。

更准确地说：

- 它是一个可靠的主干；
- 但默认更像中文保底路线；
- 若要更像 SCI 低区稿，需要在主干完成后再补一层更强的说服力。

### 5.2 为什么当前版本还不够稳

核心原因不在于任务无意义，而在于当前方法如果只停留在：

- `ResNet34-U-Net`；
- `+ LKMA`；
- `+ Boundary Head`；
- `+ Distance-aware Loss`；

那么在英文审稿语境下，容易被看成：

- 组合式增量改进；
- 工程优化感较强；
- 新范式或强新颖性不足；
- 如果数值提升有限，则说服力会进一步下降。

因此，当前计划对中文足够，对英文低区则仍需要至少补一层“文章身份增强”。

### 5.3 哪种情况下有机会升级到 SCI 低区

如果当前主干最终结果满足下面条件，则具备进一步升级的潜力：

- `LKMA` 和 `Boundary Head` 至少有一层贡献非常清楚；
- 主线提升不是只体现在一个偶然指标上；
- `CRAG` 上没有明显崩溃；
- 有对象级与边界级证据；
- 失败案例和局限性可解释；
- 外部对比层至少有基本说服力。

在这个前提下，再补一层增强，文章就可能从“中文保底稿”向“低区 SCI 候选稿”移动。

## 6. 当前计划的内容是否足够、创新是否够

### 6.1 内容是否足够

内容是够的，但这里的“够”要分两层理解：

- 作为一个完整研究项目的主干，够；
- 作为最终英文投稿包，当前还不够。

当前内容已经包含：

- 数据；
- baseline；
- 多阶段模块验证；
- 外部对比；
- `CRAG`；
- 结果汇总；
- 中文/英文写稿门槛；
- 总验收与止损。

这说明“研究内容为空”这个判断并不成立。

真正的问题在于：这些内容目前主要存在于“执行设计层”，而不是“正式结果层”。

### 6.2 创新是否够

也要分层判断：

- 作为中文保底稿，创新够；
- 作为默认 SCI 低区稿，当前还偏保守。

当前创新的合理定义应该是：

- 以形态建模与边界分离为中心的任务导向改进；
- 强调对象级和边界级证据，而不是只追像素级高分；
- 用 `GlaS + CRAG` 形成一个相对完整的任务内方法链。

如果把创新定义成这个层级，它是成立的。

如果把创新定义成：

- 明显超越主流近年 gland-specific 方法；
- 或 foundation model 级别新思路；
- 或跨数据、跨中心强泛化；

那当前原始计划并不支持这种强表述。

### 6.3 最准确的判断

当前计划的创新强度更像：

- 对中文：够且合理；
- 对英文：需要补一层更强证据，而不是重做任务。

当前最合理的改法不是推翻主线，而是：

因此，修改重点应当是“关键母文件”和“阶段总协议”，而不是几十篇全部重写。

### 7.2 完全不用改或只需极小改的部分

这些内容对中文保底和后续升级都是底盘，不需要重做。

### 7.3 需要优先小改的部分

下面这些是最值得调整的关键位置：

#### A. `02_路线与投稿`

应优先修改：

- `01_结直肠腺体分割_高性价比路线总锁定.md`

重点不是推翻，而是确认：

- `LKMA` 的职责是“形态建模第一增强层”；

总体来看，当前 `05_LKMA/00_阶段总协议.md` 已经比较强，更多是无需大改，只需确保后续写作口径不把它夸成整篇论文的唯一核心创新。

#### C. `06_Boundary`

应明确：

- `Boundary Head` 是当前计划里最有机会成为正式主模型组成部分的一层；
- 边界证据必须进入主模型保留判断；
- 如果对象级或边界级收益更稳定，它在写作中的优先级应高于 `Distance`。

当前 `06_Boundary/00_阶段总协议.md` 的总体方向是对的，只需在后续路线层和结果汇总层更明确其优先级。

#### D. `07_Distance`

这是最值得收紧口径的地方。

建议改动原则：

- 不再默认把它当成主模型必须保留的一环；
- 把它稳定定义为“第三层可选增强项”；
- 只要收益接近波动范围，就优先 `downgrade` 或 `drop`；
- 中文写作层应默认允许没有 `Distance` 也构成完整主稿。

也就是说，这一阶段不是删除，而是要更明确它的“可选性”。

#### E. `08_外部对比`

这是后续能否平滑升级 SCI 的关键。

建议修改内容：

- 首批外部对比明确服务中文保底主表；
- 把 task-specific direct comparison 的作用从“准备层”进一步前置为“英文升级接口”；
- 不要求当前就全部复现 gland-specific 方法，但要明确后续升级时优先补什么；
- 增强外部对比保持止损边界，避免拖慢主线闭环。

#### F. `10_结果汇总`

这是当前最值得强化的阶段之一。

建议改动内容：

- 明确结果资产不仅服务中文投稿，还必须服务后续升级；
- 强化对象级、边界级、失败案例、样本级图像留档；
- 提前留好后续形态学分析接口；
- 明确哪些图和表是中文必需，哪些是英文增强可追加。

#### G. `11_总验收与止损`

重点不是重写，而是调整中文和英文的真实边界。

建议优先修改：

- `03_中文保底条件.md`
- `04_英文起步条件.md`

需要强调：

- 中文保底不要求现在就完成英文级补强；
- 英文起步必须建立在中文主干结果已经稳住的基础上；
- 英文升级的必要补充项要写清，但不要提前把它们写成中文起步的前置条件。

## 9. 在当前计划基础上如何升级到 SCI 低区

### 9.1 核心原则

SCI 升级不应该通过“再加第四个模块”来完成，而应通过“补一层更强证据”来完成。

### 9.2 最推荐的两条升级路径

#### 路径 A：任务内 direct comparison 增强

在当前中文主干完成后，补强：

- `DCAN`
- `MILD-Net`
- `TA-Net`
- `DEA-Net`
- `SDT`

中的关键方法对照。

目标不是全部统一复现，而是至少在主表中与 gland-specific 方法形成直接对话。

优点：

- 更符合英文审稿人对任务内比较的预期；
- 不改变当前主线；
- 属于结果层增强，不是任务重做。

#### 路径 B：病理意义/下游价值增强

在当前分割结果基础上导出腺体形态学特征，例如：

- 面积；
- 周长；
- 圆度；
- 密度；
- 边界复杂度。

然后证明：

- 最终模型比 baseline 更能支持良恶性差异分析；
- 或更稳定地支持结构量化；
- 或更适合作为病理下游分析入口。

优点：

- 与综述主线高度一致；
- 容易把文章从“分得更准一点”提升到“分得更有病理意义”；
- 最有机会提升英文稿说服力。

### 9.3 哪条更推荐

如果只能选一条，我更推荐优先补“病理意义/下游价值增强”。

原因是：

- 当前主线已经围绕 morphology 和 boundary；
- 补形态学分析与主线最一致；
- 更容易形成与综述一致的高层叙事；
- 比单纯多跑几个网络更容易让英文稿显得“有任务深度”。

但如果资源有限，先补 task-specific direct comparison 也完全合理。

### 9.4 SCI 升级的最小充分条件

从当前中文主干升级到更像 SCI 低区的版本，建议至少补足下面之一：

- `主线完整 + 通用外部对比 + task-specific direct comparison`
- `主线完整 + 通用外部对比 + 病理意义/形态学分析`

如果两者都能补一点，英文说服力会更强。

## 10. 推荐的执行顺序

### 10.1 第一阶段：按当前主线做出中文保底主干

目标：

- 完成 `UNet / baseline / LKMA / Boundary / Distance` 逐步验证；
- 优先锁定正式主模型；
- 完成 `GlaS + CRAG`；
- 做齐主结果表、消融表、边界证据和失败案例。

### 10.2 第二阶段：形成正式中文结果包

目标：

- 整理结果汇总；
- 判断中文保底是否成立；
- 写出一版中文可投稿骨架。

### 10.3 第三阶段：判断是否进入英文升级

判断条件：

- 主模型稳定；
- `CRAG` 不崩；
- 对象级/边界级证据清楚；
- 图表齐；
- 主线消融完整。

如果这些条件成立，再补英文增强层。

### 10.4 第四阶段：按所选路径做英文增强

二选一或两者兼做：

- task-specific direct comparison；
- 病理意义/形态学分析。

这样做的好处是：

- 中文保底不会被英文扩展拖死；
- 英文升级建立在真实主干之上；
- 整体节奏稳，返工风险低。

## 11. 最后结论

当前计划不是不可靠，也不是内容空，更不是必须推翻重做。

最准确的判断是：

- 它是一条可靠的、高性价比、偏中文保底的主干路线；
- 它的内容足以支撑一篇完整的项目主线；
- 它的创新对于中文保底是够用的；
- 它对 SCI 低区而言不是天然充分，但完全可以在当前主干完成后通过增量补强升级；
- 真正需要修改的不是几十篇全部重写，而是少数关键文件的口径收紧；
- 最该收紧的是主模型优先级、`Distance` 定位、外部对比分层、结果资产要求和中文/英文边界。

因此，最推荐的策略不是“重做路线”，而是：

1. 保留主干；
2. 小改关键口径；
3. 先完成中文保底主干；
4. 再补英文增强层。

一句话版本：

> 当前计划可以继续执行，并且有现实机会保底中文；后续如要冲 SCI 低区，也完全可以在现有主干基础上增量补强，不必推翻重做，但应尽快对少数关键文件做小范围口径修正。
