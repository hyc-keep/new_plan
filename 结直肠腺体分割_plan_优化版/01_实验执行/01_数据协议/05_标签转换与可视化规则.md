# 标签转换与可视化规则

本文件是 `01_数据协议` 中用于锁定正式标签资产的执行协议，不是“导几张图看看”的经验记录。它负责把 `GlaS / CRAG` 的原始标注统一约束为后续各阶段共同继承的一条正式链路：

`raw mask -> binary semantic mask -> tensor dtype/resize -> derived boundary target -> derived distance target -> preview assets -> asset_manifest / handoff_ready`

只要这一层没有写死，后续最容易出现的不是样式问题，而是训练、评估、可视化、边界监督和距离监督各自消费不同口径的标签资产，最终导致结果不可比较、问题不可追溯、下游阶段不得不重算上游资产。

---

本文件直接服务以下数据阶段门控：

### 1.4 当前文件对下游的硬约束

- 在下游阶段重新发明一套二值化规则
- 在下游阶段重算另一版边界图或距离图却不升级版本号
- 用训练时一套标签、评估时另一套标签
- 把数据阶段已经产出的正式标签资产降格为“临时中间结果”

## 4. 文献依据与不采用项说明

### 4.1 二值语义主线的依据

- `GlaS Challenge` 提供官方 benchmark 背景，任务核心仍是腺体区域分割与对象级评价
- 现阶段路线要求先稳定语义分割主线，再在后续阶段叠加边界或距离监督

### 4.2 边界目标的依据

- `DCAN` 说明 gland object 与 contour 联合建模可缓解 touching glands 难分问题
- `MILD-Net` 进一步说明 gland / contour 双输出在腺体任务中的延续价值
- `Gated-SCNN` 从更一般的分割场景支持显式 boundary stream 的合理性

### 4.3 距离目标的依据

- `Distance-Map-Loss` 支持把距离变换作为边界敏感监督目标
- `TA-Net` 说明拓扑与距离类信息对粘连对象分离有帮助，也提醒当前版本不应把距离目标和完整拓扑约束混为一体

### 4.4 当前明确不采用的相邻方案

当前版本不采用以下方案作为主线：

- 直接把训练主标签改成实例 mask
- signed distance map 作为第一版正式距离标签
- 方向场、多级边界带、marker-based 分离作为当前阶段正式协议
- `GlaS` 与 `CRAG` 使用不同前景阈值

不采用原因是：这些方案要么超出当前路线阶段，要么会让训练、评估与资产交接口径变复杂，无法满足当前“先完成稳定闭环”的要求。

## 5. 正式标签转换总链路

当前唯一允许的正式链路如下：

1. 读取 `GlaS / CRAG` 原始单通道标注
2. 执行二值化：`y(p) = 1[m(p) > 0]`
3. 形成正式二值标签资产：张量口径 `0 / 1`，磁盘可视化口径 `0 / 255`
4. 对 image 与 mask 执行同步 resize / augmentation，其中 image 用连续插值、mask 用离散插值
5. 从正式二值标签派生边界图
6. 从正式二值标签派生距离图
7. 导出 `raw / mask_bin / overlay` 最小预览证据包；若已生成派生标签，则追加 `boundary_overlay / distance_heatmap`
8. 写入 `asset_manifest`、运行记录字段与交接状态

任何阶段都不允许绕开第 `2-4` 步直接临时生成另一套训练标签，也不允许边界图和距离图脱离正式二值标签单独定义。

与 `结直肠腺体分割_plan_优化版\01_实验执行\01_数据协议\00_阶段总协议.md`、`结直肠腺体分割_plan_优化版\01_实验执行\01_数据协议\07_数据阶段验收.md`、`结直肠腺体分割_plan_优化版/01_实验执行/02_UNet流程验证/00_阶段总协议.md` 的门控绑定固定为：

## 6. 二值语义标签规则

### 6.1 规则卡片

- 当前结论：当前主线统一使用二值语义标签，前景定义为 gland foreground，规则固定为 `mask > 0`
- 规则类型：`路线层已锁定 + 工程冻结规则`
- 适用阶段：`01_数据协议` 至少继承到 `07_Distance`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/01_数据协议/00_阶段总协议.md`，`结直肠腺体分割_plan_优化版/02_路线与投稿/01_结直肠腺体分割_高性价比路线总锁定.md`，`结直肠腺体分割_plan_优化版/03_文献证据/05_腺体任务经典论文/01_GlaS-Challenge.md`
- 核心公式或定义参考：对任意像素 `p`，正式标签定义为 `y(p) = 1[m(p) > 0]`，否则 `y(p) = 0`
- 采用原因：当前路线要求先稳定统一语义分割闭环，让 `UNet / Baseline / LKMA / Boundary / Distance` 都继承同一基础标签
- 不采用的相邻方案：不采用实例标签直训；不采用按数据集分别设阈值；不采用“脚本里看情况再转”
- 代码落点：`crc_gland_segmentation_project/tools/stage01_data_protocol/convert_masks.py`，`crc_gland_segmentation_project/src/data/mask_ops.py`
- 运行记录字段：`mask_rule_version=mask_rule_v1`，`mask_positive_rule=gt_zero`
- 验收方式：抽查正式 mask 是否仅含 `{0,1}`；核对下游阶段文档与配置继续继承同一版本号

### 6.2 当前正式要求

`GlaS` 和 `CRAG` 在执行层统一遵守同一二值化规则：

- `mask > 0` 记为前景 `1`
- 其余记为背景 `0`

本文件不允许出现“`GlaS` 一套、`CRAG` 一套”的主标签定义。

## 7. `dtype`、存储口径与张量规范

### 7.1 规则卡片

- 当前结论：image 训练张量固定为 `float32`，mask 在进入训练前也必须统一为 `float32`；磁盘可视化导出使用单通道 `uint8`
- 规则类型：`工程冻结规则`
- 适用阶段：`01_数据协议` 至少继承到 `02_UNet流程验证`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/02_参数冻结总表.md`，`结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/05_代码工程映射与实现策略.md`，`结直肠腺体分割_plan_优化版/01_实验执行/01_数据协议/00_阶段总协议.md`
- 核心公式或定义参考：若 `mask_disk in {0,255}` 或 `mask_disk in {0,1}`，则训练前统一映射到 `mask_tensor in {0.0, 1.0}`
- 采用原因：训练接口、数据增强与后续派生标签生成都要求稳定、唯一的张量类型
- 不采用的相邻方案：不采用 `uint8 mask` 直接入 loss；不采用一部分脚本用 `0/1`、另一部分脚本用 `0/255` 却不设转换层
- 代码落点：`crc_gland_segmentation_project/tools/stage01_data_protocol/convert_masks.py`，[project_root]/src/data/transforms.py
- 运行记录字段：`image_dtype=float32`，`mask_dtype=float32`，`mask_disk_value_rule=0_255`
- 验收方式：检查训练前 image/mask 张量类型已统一；检查磁盘导出与训练张量口径被显式区分

### 7.2 正式口径

当前正式口径固定为：

- 内存与训练张量：`mask in {0,1}`，`dtype=float32`
- 磁盘导出与人工检查：`mask in {0,255}`，`dtype=uint8`

禁止出现：

- 训练脚本直接读取 `0/255 uint8` mask 却不转换
- 可视化导出与训练张量混用同一路径却不标记版本
- 不记录 `mask_rule_version` 和 `mask_disk_value_rule`

## 8. resize 与 augmentation 同步规则

### 8.1 规则卡片

- 当前结论：默认输入尺寸固定为 `512 x 512`；image resize 使用双线性插值，mask resize 使用最近邻插值；几何增强必须同步作用于 image 与 mask
- 规则类型：`参数冻结 + 工程冻结规则`
- 适用阶段：`01_数据协议` 至少继承到 `03_UNet稳定性`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/02_参数冻结总表.md`，`结直肠腺体分割_plan_优化版/01_实验执行/01_数据协议/00_阶段总协议.md`，`结直肠腺体分割_plan_优化版/02_路线与投稿/02_结直肠腺体分割_分阶段实验路线与执行标准.md`
- 核心公式或定义参考：`x' = BilinearResize(x, 512, 512)`；`y' = NearestResize(y, 512, 512)`
- 采用原因：连续图像与离散标签必须分开处理，才能避免边界污染并保持派生标签可复算
- 不采用的相邻方案：不采用 image/mask 同用双线性；不采用 resize 后再靠阈值修补标签；不采用 augmentation 只变 image 不变 mask
- 代码落点：[project_root]/src/data/transforms.py，`[project_root]/configs/data/*.yaml`
- 运行记录字段：`input_size=512x512`，`image_interp=bilinear`，`mask_interp=nearest`，`stain_normalization=off`
- 验收方式：检查 transform 配置显式写出插值方式；抽查 overlay 无系统性边界漂移

### 8.2 几何增强与颜色增强边界

几何增强可包含：

- `flip`
- `rotate`
- `scale`
- `crop`

它们必须满足：

- image 与 mask 同步变换
- mask 始终采用离散标签插值

颜色相关增强只允许作用于 image，不允许作用于 mask。

### 8.3 `stain_normalization` 当前口径

当前版本固定：

- `stain_normalization = off`

如果后续要引入 stain normalization，只能作为新变量单列，不允许回写到本文件当前正式协议里伪装成原有默认项。

## 9. 边界目标规则

### 9.1 规则卡片

- 当前结论：边界图必须从正式二值 mask 派生；首轮默认边界宽度 `w = 3 px`，唯一备选为 `5 px`
- 规则类型：`论文支持的候选范围 + 工程冻结规则`
- 适用阶段：数据阶段派生资产，正式服务 `06_Boundary`
- 直接依据：`结直肠腺体分割_plan_优化版/03_文献证据/05_腺体任务经典论文/03_DCAN.md`，`结直肠腺体分割_plan_优化版/03_文献证据/05_腺体任务经典论文/04_MILD-Net.md`，`结直肠腺体分割_plan_优化版/03_文献证据/02_边界_形状_损失支撑/01_Gated-SCNN.md`，`结直肠腺体分割_plan_优化版/01_实验执行/01_数据协议/00_阶段总协议.md`
- 核心公式或定义参考：`b_w = 1[(Dilate(y, K_w) - Erode(y, K_w)) > 0]`，其中 `y` 为正式二值 mask
- 采用原因：需要为后续边界分支提供唯一、可复算且可检查的派生监督目标
- 不采用的相邻方案：不采用脚本各算一套 boundary；不采用多级边界带作为第一版正式协议；不采用脱离正式 mask 单独标注 contour
- 代码落点：`crc_gland_segmentation_project/tools/stage01_data_protocol/build_boundary_targets.py`，`crc_gland_segmentation_project/src/data/boundary_targets.py`
- 运行记录字段：`boundary_target_version=boundary_target_v1`，`boundary_width=3`
- 验收方式：抽查 boundary overlay 与原始 gland mask 对齐；检查训练、检查和预览共用同一生成函数

### 9.2 当前默认生成方式

当前边界图统一按以下顺序生成：

1. 读取正式二值 mask
2. 提取前景边界
3. 形成单像素边界
4. 使用固定宽度 `w` 膨胀成边界带

数据阶段只要求把边界目标稳定生成、稳定记录和稳定可视化，不在本文件里冻结后续 boundary loss 权重。

## 10. 距离目标规则

### 10.1 规则卡片

- 当前结论：距离图由正式二值 mask 生成 Euclidean distance transform，并归一化到 `[0,1]`
- 规则类型：`论文支持的候选范围 + 工程冻结规则`
- 适用阶段：数据阶段派生资产，正式服务 `07_Distance`
- 直接依据：`结直肠腺体分割_plan_优化版/03_文献证据/02_边界_形状_损失支撑/02_Distance-Map-Loss.md`，`结直肠腺体分割_plan_优化版/03_文献证据/05_腺体任务经典论文/07_TA-Net.md`，`结直肠腺体分割_plan_优化版/03_文献证据/02_边界_形状_损失支撑/04_Shape-Aware-SDM.md`，`结直肠腺体分割_plan_优化版/01_实验执行/01_数据协议/00_阶段总协议.md`
- 核心公式或定义参考：`d(p) = min_{q in \partial Y} ||p-q||_2`；`d_hat(p) = d(p) / (max(d) + eps)`
- 采用原因：为后续距离敏感监督提供统一派生标签，同时保持当前版本简单、可解释、可复现
- 不采用的相邻方案：不采用 signed distance map；不采用方向场；不采用多种距离定义混合为第一版正式协议
- 代码落点：`crc_gland_segmentation_project/tools/stage01_data_protocol/build_distance_targets.py`，`crc_gland_segmentation_project/src/data/distance_targets.py`
- 运行记录字段：`distance_target_version=distance_target_v1`，`distance_type=euclidean`，`distance_norm=zero_one`
- 验收方式：检查 `npy` 数值范围稳定在 `[0,1]`；检查 distance heatmap 与训练输入共用同一生成函数

### 10.2 当前默认生成方式

当前距离图统一按以下顺序生成：

1. 读取正式二值 mask
2. 计算 Euclidean distance transform
3. 归一化到 `[0,1]`
4. 以 `float32` 正式保存

当前版本明确不把 signed distance map 纳入主线正式协议，只保留为后续相邻方案。

## 13. 当前明确禁止的做法

以下做法一律不得出现：

- `GlaS` 与 `CRAG` 使用不同二值化阈值却不单独升级版本
- resize mask 时使用双线性或其它连续插值
- 训练、检查、可视化各自重算不同版本的边界图和距离图
- 训练阶段读取一套标签，评估阶段又切换另一套二值规则
- 把连续距离图压成低精度整型图再回读训练
- 把 stain normalization 混入当前默认协议却不单列成新变量
- 在 `02_UNet流程验证` 才临时决定标签 `dtype`、插值方式或派生标签定义

## 14. 回退条件

### 14.1 数据阶段内回退触发条件

只要出现下面任意一条，本文件对应的标签协议就不得放行，必须先回退到当前文件修正，而不是把问题带进 `UNet`：

- 二值化结果不再满足 `mask > 0`
- 磁盘口径与训练张量口径混用，且无法通过版本字段区分
- image / mask resize 或几何增强不同步
- boundary target 不是由正式二值 mask 派生
- distance target 不是 Euclidean distance transform + `[0,1]` 归一化
- 预览图无法回指 `dataset / split / sample_id`
- `asset_manifest` 缺少 `mask_rule_version / boundary_target_version / distance_target_version / visual_proto_version`

### 14.2 下游异常时的强制回退顺序

如果 `02_UNet流程验证` 出现全背景预测、可视化明显错位、对象级指标与肉眼严重矛盾等异常，必须按下面顺序强制回退：

1. 检查 `mask > 0` 二值化是否被下游改写
2. 检查磁盘口径与训练张量口径是否混用
3. 检查 image / mask 的插值与同步增强是否分裂
4. 检查 boundary target 是否仍由正式二值 mask 派生
5. 检查 distance target 是否仍沿用 Euclidean + `[0,1]` 归一化
6. 检查 preview 资产与 `asset_manifest` 是否还能回指当前版本
7. 检查训练入口是否真的只消费正式 split 与正式标签资产

### 14.3 回退后的重新放行条件

发生回退后，只有同时满足下面条件才允许重新把 `handoff_ready` 改回 `True`：

- 相关标签资产已经按当前正式版本重导出
- 预览资产已同步更新，且抽查无系统性错位
- `asset_manifest` 已更新到新一轮正式登记结果
- `run_unet_data_preflight()` 重新通过，`preflight_pass=True`
