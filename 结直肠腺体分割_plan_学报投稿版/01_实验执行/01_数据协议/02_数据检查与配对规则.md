## 固定、可调、禁止参数
- 固定：六步顺序：数量 -> 配对 -> 可读性 -> 前景 -> 统计 -> 人工抽查；前景定义 `mask>0`；sample_id唯一。
- 可调：人工抽查样本清单和报告版本，须预注册并记录原因；不调样本配对规则。
- 禁止：只检查文件存在、用训练异常代替数据检查、忽略空mask/错配、测试集参与选择、把检查退出0写成 data pass。

## 检查规则与资产 schema
1. 数量：按预期 split 目标检查 GlaS 68/17/60/20、CRAG 153/20/40，但目标数量不等于实际通过。
2. 配对：`sample_id`由规范化 stem 得到；image/mask 一一对应，CSV至少含 `sample_id,image_relpath,mask_relpath,source_partition,grade或source_subset`。
3. 可读性：RGB image、mask可解码，尺寸/通道记录，路径必须在 dataset_root内。
4. 前景：记录 `foreground_pixels=(mask>0).sum()`、空mask、非有限值和尺寸不一致；mask原值可保留但主标签只用 `mask>0`。
5. 统计：每split计数、前景像素统计、grade/source_subset分布、重复sample和跨split交集。
6. 人工抽查：从每个 dataset/split 预注册样本输出 image/mask/overlay；报告逐样本回指，不以截图存在代替结论。
输出 directory_contract planned artifact directory reports/data_checks 下的 planned_not_created:<dataset>_check.json/csv/md，每条记录含 `check_name,sample_id,split,observed_value,expected_rule,status,warning_code`。

## 验收
`pass_check`要求六步均有报告、样本级异常可回指、重复/跨split为零或有正式阻断解释、抽查结果可追溯。实际 Gate 只能 `planned/blocked`；未运行检查不允许填数字或 pass。

## 独立回退条件
任一错配、路径越界、空mask异常未解释、尺寸不符、跨split重复、报告和CSV digest不一致、人工抽查缺失、blind字段缺失，回退到数据源或配对规则，重新生成检查资产；不进入 split 写入。

## 实施与状态转移
- 对每条 CSV 行先规范化 `sample_id`，再解析 image/mask；解析失败必须保留 `warning_code`，不能静默丢弃样本。
- `foreground_pixels=(mask>0).sum()`、shape、dtype、重复键和跨 split 交集写入样本级报告；报告缺行即维持 `planned/blocked`。
- 检查状态按 `planned_not_created -> running -> partial/blocked -> accepted` 记录；`partial` 只表示证据不完整，不是可消费状态。
- 预览抽查至少回指 dataset、split、sample_id 和 report row；test 只验证 blind 字段，不读取标签做选择。
- 下游消费前必须同时读取 CSV、check report 和 report sha256；单独存在的 CSV 仍为 `existing_unverified`。

## 字段级验收清单
- 核对每条记录包含 `check_name,sample_id,split,observed_value,expected_rule,status,warning_code`。
- 核对 image/mask 解码、shape、dtype、mask>0 前景像素和路径越界均有样本级 observed_value。
- 核对重复 sample_id、跨 split 交集、空 mask、不可读文件和尺寸不一致均有计数与 warning_code。
- 核对六步检查顺序不可跳过；人工抽查样本必须有 overlay 回指和审核结论。
- 核对 report_sha256 与输入 CSV、source_manifest、protocol_version 绑定；缺任一绑定仍为 blocked。
- 核对 TestA/TestB/CRAG test 只验证 blind 身份，不写入 selector、threshold、postprocess 选择证据。
- 核对检查异常回退到 source 或配对层，并保留原报告而非覆盖。
- 核对文档检查通过不改变 data_check_gate 的 planned/blocked 状态。
- 核对报告行数与输入 CSV 行数可解释，缺失样本有明确 next_action。
