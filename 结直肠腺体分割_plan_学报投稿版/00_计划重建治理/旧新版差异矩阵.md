- purpose: 记录旧版哪些内容保留、重写、恢复、降级或禁止消费。

## 2. 当前已发现的缩水点

| 主题 | 旧版强项 | 新版缺口 | 必须修复 |
|---|---|---|---|
| 参数冻结 | 全局参数冻结表与阶段参数细则 | 新方法有候选描述但部分值未锁定或无来源 | 恢复逐参数：值、单位、来源、可调范围、冻结状态 |
| 训练入口 | full split、smoke、resume、CSV、scheduler、early stop | N1 曾将 smoke batch 上限泄漏到 formal | 复用旧 trainer 规范并加正式/探针强隔离 |
| 评估入口 | val_objdice、testA/testB、逐样本七指标 | N1 仅 loss，未接正式评估 | 接入旧 `run_eval.py` 级完整评估 |
| checkpoint | val_objdice_max，identity、SHA、RNG | N1 曾使用 val_loss_min，评估不完整 | 在协议层裁决后统一实现 |
| 汇总 | raw、mean/std、failure、handoff | 新版 schema 声明多，实际生成少 | 实现并验证 N1/N2/N3 专用汇总 |
| Gate | 训练、指标、稳定性、visual、handoff | contract/runtime pass 容易被误读为正式结果 pass | 分层 Gate，禁止扩大解释 |
| 投稿准入 | 中文/英文起步、写稿、最终 readiness | 当前 active 文件缩水 | 恢复有效当前裁决文件并标明结果资格 |
| 依据链 | 旧阶段正文经常写明论文/代码/参数 | 新版部分文件写成 planned interface | 每文件逐项来源矩阵和代码落点 |
| 回退 | 阶段级 keep/drop/stop/Plan B | 新版部分回退停留在声明 | 每个方法、筛查、正式 GlaS、总验收独立回退段 |
| tests | 旧工程有部分实现测试 | journal N1 测试不足 | 先定义必要测试，不用 pytest 缺失冒充通过 |

## 3. 保留、重写、恢复、禁止消费

### 保留并重写

- 旧版 `00_总览与规范` 的结构和规则。
- 旧版数据协议、训练步骤、验证测试、三 seed、mean/std、阶段验收。
- 旧工程 train/eval/test/summarize 的接口思想。
- GlaS/DCAN/MILD-Net/指标参考资料的来源映射。

### 新版保留但必须加深

- N1/N2/N3 方法协议。
- val 筛查和正式 GlaS 两阶段分离。
- blind、lineage、manifest 和结果 eligibility。
- Plan B 的有限触发边界。

### 必须恢复为 active 细则的候选文件

- 旧 11 的中文保底条件、英文起步条件、写稿进入判断、最终 readiness、stage gate matrix、stop loss decision。
- 旧 10 的结果包检查、主表/消融表回填、可视化和失败案例细则。
- 旧 02/03 的错误排查、训练/验证/测试和统计细则。

最终是否恢复为正式路径，须在对应批次正文重写和 diagnostics 后裁决。

### 禁止当前消费

- 旧实验结果、旧 checkpoint、旧 metrics、旧 summary、旧 Gate、旧 handoff。
- 新 N1 当前已归档的 smoke-cap invalid runs。
- 任何 synthetic runtime 作为真实 GlaS formal evidence。
- 任何未通过独立指标复核的 CSV。

## 4. 新方法科学变量矩阵

| 方法 | 主变量 | 必须冻结 | 允许筛查 | 禁止 |
|---|---|---|---|---|
| N1 | instance-aware contour supervision | target 定义、宽度/radius、loss、lambda、输入输出 | 预注册候选、仅 train/val | 使用 TestA/TestB/CRAG 调参 |
| N2 | region/contour feature fusion | N1 checkpoint、融合位置、通道、参数预算 | 仅一个预注册融合候选 | 同时改 target/loss/融合位置 |
| N3 | contour-guided instance recovery | marker/elevation/postprocess、阈值、面积 | 仅 val17 后处理候选 | 用 test 选择 watershed 参数 |
| Plan B | DCAN-inspired adaptation | adaptation identity、差异、一次触发 | 仅定义的单次候选 | 称 strict DCAN replication、无限搜索 |

## 6. 当前结论

- 新计划重建尚未完成。
- 旧计划流程可以作为严谨基线迁移。
- 新方法必须保留科学边界，不能简单回退为旧模块拼接。
- 所有正式实验暂停，直到各批次计划正文、来源、参数、代码映射、验收和 diagnostics 收口。
