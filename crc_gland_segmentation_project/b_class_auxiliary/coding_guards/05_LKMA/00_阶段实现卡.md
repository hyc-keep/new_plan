## 3. 当前阶段唯一目标
- 用一句话说清: 在不改变 B1 数据、训练、评估和统计协议的条件下，验证 `R34UNet + LKMA v1 (bottleneck, k=15)` 是否带来稳定、可解释且成本可接受的对象级收益。
- 当前阶段进入条件: 研究记录已完成，研究门禁报告状态为 pass，且 B1 current_standard 的真实资产可按 `frozen_baseline_with_warning` 消费；`original_gate_b1=false` 不得改写。
- 上一阶段已经交付: B1 current_standard 三 seed 的真实训练、测试、独立指标复核和基线资产；其 Object Dice stability warning 必须在 C1 lineage、比较表和解释中继续保留。

## 4. 为什么现在做这个,而不是下一个阶段
- 当前阶段必须先解决的阻断: 先验证大感受野/形态上下文模块是否值得保留，才能决定 06_Boundary 消费 `baseline + LKMA` 还是退回 B1 current_standard。
- 如果现在提前做下一个阶段,会破坏什么: 直接加入 Boundary 会同时改变对象分离变量，使 LKMA 的收益来源无法归因，也会把未裁决的 C1 结果传给下游。
- 下一个阶段此刻还不允许进入的原因: C1 尚未完成主版本三 seed、独立复核、成本比较、keep/backup/drop 和 Gate_C1 handoff。

## 5. 当前阶段允许改动 / 禁止改动
- 明确允许改: 只实现 LKMA v1 的 depth-wise large-kernel spatial block、可选 1x1 channel mixing、可选 direct residual；只在 bottleneck 放置主版本；最多按协议补一个互斥备选；新增 C1 专属模型/实验配置、汇总和比较入口；补充 C1 所需 manifest、lineage、成本字段和可视化资产。
- 明确禁止改: 不得改 train/val/TestA/TestB、seed、loss、optimizer、scheduler、AMP、best selector、threshold、后处理、评估实现、数据增强或输入尺寸；不得加入 attention map/gating、多分支、多尺度 kernel mix、stride/pooling、上下采样、Boundary Head、Distance loss、完整 VAN/WTConv/RepLKNet backbone、未记录 projection shortcut；不得把 screening 当正式结果；不得改写 B1 metrics、checkpoint 或原始 Gate。
- 如果越界,会破坏哪份正式协议: 越界会破坏 `结直肠腺体分割_plan_优化版/01_实验执行/05_LKMA/00_阶段总协议.md`、插入位置协议、实验步骤协议和保留删除标准协议、全局参数冻结表、评估口径以及 B1 conditional handoff lineage。
