## 本轮 B1 数值故障与最小修复

本轮 B1 三个正式 run 均从第 1 个 epoch 产生 NaN，训练日志、验证日志和 checkpoint 已判无效并排除当前 Gate。根因定位为 `scripts/train.py` 曾按 `stage_code=B1` 自动打开 ResNet34 encoder BN policy，使原始 `train_proto_v1` 隐式继承 v3 行为；同时 `src/engine/trainer.py` 在 loss/gradient 非有限时没有阻断，导致 NaN 被错误写成 early stopping 和 baseline_ready。

本轮只做两项代码修复：

- `scripts/train.py`：BN policy 改为只由 model config 显式控制，`train_proto_v1` 不再按 B1 身份自动继承 v3。
- `src/engine/trainer.py`：在 backward 前检查 loss finite，在 optimizer step 前检查 unscaled gradient finite；发现 NaN/Inf 立即抛错，禁止生成正常完成状态。
- `configs/model/resnet34_unet.yaml`：将 `model_version` 从 `current_standard` 统一为 canonical B1 身份 `r34_unet_v1`，消除 experiment config 与 run_meta 的 lineage 不一致。

学习率、loss、batch、AMP、scheduler、BN 模型默认行为、threshold、eval protocol 和 seed 均未改变。修复后必须重新完成静态检查、runtime、smoke 和 B1 三 seed fresh 重训；旧 B1 三个结果不得续用。
