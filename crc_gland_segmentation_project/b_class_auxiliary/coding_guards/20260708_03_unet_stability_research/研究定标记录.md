## 6. 当前阶段边界初判
- 当前阶段应该做什么: 先把“在 A1 冻结协议之上只改 `train_seed` 做三次重复、产出 TestA 与 TestB 分开的 raw per-seed 表与 `mean +- std` 聚合表、记录跨 seed 错误模式一致性、裁决异常 run、形成稳定基线交接包”研究清楚,并在下一步阶段锁定里把唯一目标限定为 A2_UNet_GlaS_v1_seed3407、seed1234、seed2025 三次正式 run
- 当前阶段不该做什么: 不提前切到 `04_Baseline` 的 ResNet34-U-Net; 不提前引入 LKMA、Boundary Head、Distance-aware Loss; 不提前把 CRAG 或跨数据集比较纳入本轮; 不借重复实验之名改增强、阈值、optimizer、batch_size、后处理; 不把某个难看 seed 删除或替换后再汇总
- 为什么现在不能直接进入下一个阶段: 因为当前仍停在“研究”工作流阶段,并且工程里还缺 A2_UNet_GlaS_v1_seed 三个 experiment config、reports 表目录下的 unet_seed_results.csv 与 unet_mean_std_summary.csv、reports 阶段报告目录下的 unet_stability_note.md,以及三 seed 聚合与交接脚本能力等 A2 正式对象; 在没有先做阶段锁定和 Pre-check 的情况下直接编码,会把唯一变量边界、冻结继承、汇总 schema 和红线都留成口头规则
- 如果现在跳过边界锁定,最容易犯什么错: 会在某个 seed 上临时改 batch_size 或阈值造成协议漂移却仍并表; 会先算聚合再补 raw,导致均值无法回查; 会把 TestA 与 TestB 混算成单一总值; 会在异常 run 未按 `bug_source_rank` 排查前就把 bug 当成真实 seed 波动写进结论

## 7. 对后续阶段锁定的输入
- 后续 `00_阶段实现卡.md` 必须写清的关键点: 唯一目标只能是“在已通过的 A1 冻结协议之上,只改 train_seed(3407、1234、2025)完成三次正式 run 并产出 TestA 与 TestB 分开的 mean±std 稳定性证据”; 允许改动应限定为新增 A2_UNet_GlaS_v1_seed 三个 experiment config、扩展 summarize_stage 与 compare_runs 的三 seed 聚合与交接能力、以及 reports 表目录与 reports 阶段报告目录下的稳定性产物; 禁止改动应明确包括训练配置 `crc_gland_segmentation_project/configs/train/unet_flow_v1.yaml`、评估配置 `crc_gland_segmentation_project/configs/eval/eval_proto_v1.yaml`、best_selector=val_objdice_max、threshold_source=val17、eval_cast_policy、boundary 与连通域口径以及 UNet 结构本身

- 哪些问题如果不解决,阶段锁定就不能开始: 本轮没有新的研究层阻塞项; 研究已足以证明本阶段唯一变量、冻结继承边界、汇总 schema、Gate_A2 门槛与工程缺口,因此阶段锁定可以开始; 但阶段锁定必须显式写明“三次 run 只能通过不同 A2_*_seed* 配置启动、其余协议一律继承 A1,且真正训练需在 GPU 环境执行”

- 研究结论状态: `allow_stage_lock`
