## 三、三份操作说明的统一模式总结

所有三份操作说明都遵循相同的结构，这是一个**优秀的可复用模板**：

## 四、建议作为 Skill 规范的内容

### 4.1 操作说明强制要求

对于每个实验阶段（01/02/03/04/...），必须在交付时包含 `操作说明_如何运行本阶段.md`，且必须包含以下六个章节：

#### 必需章节清单
| 章节 | 内容要求 | 示例 |
|------|---------|------|
| **一、这个阶段在做什么** | 1-2 段话说明阶段定位、核心产物、与前后阶段关系 | "本阶段用三个 seed 验证训练稳定性，产出 mean±std 表" |
| **二、先读哪些文件** | 推荐阅读顺序（≥3 个文件），新手友好 | "1. 清单 2. README 3. 当前阶段为什么能 pass" |
| **三、配置引用关系/运行顺序** | 技术架构图或流程图（ASCII 或 Markdown） | experiment 入口 → model/train/eval 子配置 |
| **四、完整执行流程与命令** | 逐步命令清单（每个命令必须带参数说明和默认值） | `python train.py --config <必填> --device cuda <默认>` |
| **五、跑完怎么确认成功** | 明确的通过标准（文件存在性 + 指标范围 + gate 状态） | `workflow_gate_status=pass`、`testB ObjDice ≈ 0.79` |
| **六、常见坑** | 实战踩坑经验（≥2 条） | "命令必须在项目根目录执行"、"别把 smoke 当正式 run" |

#### 格式规范
- 所有命令前加 `cd /path/to/project_root` 声明（避免路径错误）
- 参数说明格式：`--param`：说明文字，默认 `value`
- 必填参数用 **必填** 标注，可选参数说明默认值
- 产物路径用相对路径（相对项目根目录）

#### 诚实边界要求
- 必须明确说明本阶段**做什么**、**不做什么**
- 如果有已知限制（如"只验证入口预飞，不代表 forward 已成立"），必须写在对应步骤的说明中
- 参考指标必须附带"重跑因环境略有浮动属正常"等免责说明

- "请参考源码"、"详见 README"（必须自包含）
- "根据情况调整参数"（必须给出明确的默认值和调整场景）

- 只有命令没有参数说明（每个参数都要解释）

✅ **应该**：
- 每个命令都可以直接复制粘贴运行
- 每个参数都有"必填"或"默认值"说明
- 每个步骤都有产物路径和验证方法
