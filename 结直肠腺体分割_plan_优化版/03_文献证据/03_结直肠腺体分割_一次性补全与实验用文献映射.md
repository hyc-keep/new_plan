## 1. 文档定位

### 1.1 这份文档负责什么

1. 把后续路线设计拆成若干固定场景
2. 给每个场景指定应优先查看的正式深提取稿
3. 告诉你进入正式稿后，具体应该搜什么信息

### 1.2 什么时候先看这份

- 当你准备锁实验协议、模块顺序或阅读顺序时
- 当你不想每次都从 `62` 篇里手动遍历时
- 当你已经知道当前处于哪个实验阶段，但不知道先看哪几篇时

### 1.3 这份文档的输入和输出

- 输入问题：`我现在要解决什么实验问题，先看哪几篇，进入后重点搜什么`
- 输出结果：`阶段 -> 论文 -> 重点字段 -> 使用目的`

### 1.4 这份文档不负责什么

- 不负责逐篇无遗漏总索引，那是 `02`
- 不负责按关键词直接反查，那是 `04`
- 不负责替代单篇正式稿中的原始证据细节

---

## 2. 使用方法

后面不要再用下面两种低效方式：

- 每次做实验前，从 `62` 篇里一篇篇手动遍历
- 每次只凭印象回忆“好像某篇提过这个点”

推荐流程：

1. 先在本文件中定位你当前处于哪个阶段
2. 按本文件给出的“先看论文”列表打开对应正式深提取稿
3. 在对应正式稿中重点查看参数、协议、公式、页码和图表索引
4. 只有当正式稿还不能回答问题时，再去查原始 PDF 或 `_tmp_fulltext_archive`
