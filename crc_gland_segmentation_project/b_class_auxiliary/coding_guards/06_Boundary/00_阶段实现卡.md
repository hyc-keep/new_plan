## 3. 当前阶段唯一目标
- 用一句话说清: 在不改变 B1 数据、训练、评估和统计协议的条件下，验证 `R34UNet + 最小 BoundaryHead` 是否带来稳定、可解释且成本可接受的边界和对象级收益。
- 当前阶段进入条件: 06 研究定标记录已建立，研究 alignment gate 已真实通过；05 LKMA 已判定为 `drop`，因此当前 base 回到 B1 baseline，并继承 B1 `valid_with_stability_warning`。
- 上一阶段已经交付: B1 baseline 的真实训练、测试、独立指标复核和可消费资产；其 `stability_warning=true`、`original_gate_b1=false` 不得在 06 中改写。

## 4. 为什么现在做这个,而不是下一个阶段
- 当前阶段必须先解决的阻断: 需要先验证显式边界监督是否改善边界分离、黏连处理和轮廓恢复，才能决定后续 Distance-aware Loss 是否消费 Boundary 结果或回到 baseline。
- 如果现在提前做下一个阶段,会破坏什么: 同时加入距离监督会改变形状变量，无法归因 Boundary Head 的效果，也会把未裁决的 Boundary 结果传给后续阶段。
- 下一个阶段此刻还不允许进入的原因: 06 尚未完成代码现状核对、target/metric 协议裁决、runtime/smoke、三 seed 正式运行、独立复核、成本比较和 Gate_D1。

## 5. 当前阶段允许改动 / 禁止改动
- 明确允许改: 只允许实现 decoder final 接入的单分支 BoundaryHead；主版本固定 `boundary_width=3`、`boundary_loss=BCE`、`lambda_boundary=0.3`；允许新增 Boundary logits 的训练监督、Boundary 可视化和边界误差分析；允许新增 06 专属配置、manifest、summary 和正式报告入口；允许最多一个由主版本 screening 结果触发的备选 width=5。
- 明确禁止改: 不得改 train68/val17/TestA60/TestB20、输入尺寸、归一化、三 seed、BCE+Dice 主分割损失、AdamW、学习率、scheduler、epoch/patience、AMP、best selector、threshold、后处理、connected components 或历史 B1/C1 结果；不得加入完整 GSCNN/DCAN/DEA-Net、多分支反馈、距离图、topology loss、marker/中心点监督、多个接入位置、多个 loss/lambda/width 组合搜索或 encoder-decoder 重构。
- 如果越界,会破坏哪份正式协议: 越界会破坏 `结直肠腺体分割_plan_优化版/01_实验执行/06_Boundary/00_阶段总协议.md`、06 标签和实验步骤协议、全局参数冻结表、评估口径、B1 conditional handoff lineage 以及 Gate_D1 的单变量归因要求。
