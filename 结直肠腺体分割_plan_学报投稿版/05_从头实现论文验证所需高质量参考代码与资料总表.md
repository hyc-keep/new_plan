## Object F1 正式口径

正式 primary identity 按 GlaS 挑战论文“at least 50% overlap”语义冻结：

\[
match(S_j,G_i)=\mathbb{1}\left[\frac{|S_j\cap G_i|}{|G_i|}\ge 0.5\right],
\qquad
F1_{obj}=\frac{2TP}{2TP+FP+FN}.
\]

- primary：`glas_paper_overlap_ge_0_5`，边界为 `>=0.5`，进入正式主表。
- compatibility：本地 DCAN `metrics.py` 使用 `>0.5`，仅作 `local_dcan_overlap_gt_0_5` audit，不进入正式主表。
- 当前未获得并验证官方 MATLAB 脚本，`official_matlab_verified=false`；不得声称官方代码 parity。
- 实现前必须有恰好 `0.5` 的边界单测；primary 计 TP、compatibility 不计 TP。
- 必须生成 `object_f1_boundary_diff.csv` 和差异报告，记录 sample/object、overlap ratio、primary 判定、compatibility 判定与聚合差异。缺任一项，对象级结果不得进入正式主表。

## 最小参考资料集合

### 官方与通用底座

1. `PyTorch`：模型、AMP `GradScaler`、optimizer、checkpoint、DataLoader 与 deterministic API。
2. `torchvision`：ResNet34 encoder 与官方权重加载身份。
3. `segmentation_models_pytorch`：只作架构/encoder-decoder 组织参考，不替代统一训练入口。
4. `SciPy` 与 `scikit-image`：形态学、连通域、distance transform、watershed。
5. `MedPy` 或经独立单测的等价实现：HD95 交叉核对。
6. `pandas` 与 `matplotlib`：schema 化汇总和图表，不参与结果生成语义。

### 任务论文与实现锚点

1. GlaS Challenge：数据角色、TestA/TestB 分开报告、Object F1/Object Dice/Object Hausdorff。
2. DCAN：object/contour 联合方向；本地代码只作行为复核。
3. MILD-Net：contour 分支相邻证据，不与 DCAN/N1 身份混同。
4. Gated-SCNN：N2 gate 公式动机，不照搬城市街景配置。
5. DEA-Net：边界增强与融合动机，不提供本项目默认参数。
6. TA-Net：region/marker/elevation 的 watershed 思路，不提供可直接照搬的完整参数。

参考资料是否纳入，不按固定旧计数判断；以当前 evidence manifest 的实际枚举和 digest 为准。禁止把任何历史“共若干篇/仓库”计数写成当前事实。
