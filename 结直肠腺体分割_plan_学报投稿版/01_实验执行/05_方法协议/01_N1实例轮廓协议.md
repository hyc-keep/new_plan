## Target 数学定义

设图像域 \(\Omega=\{0,\ldots,H-1\}\times\{0,\ldots,W-1\}\)，实例图 \(Y:\Omega\to\{0,1,\ldots,K\}\)，\(S_k=\{x:Y(x)=k\}\)。结构元素 \(B_w\) 是以中心为原点的 8 邻域方形，半径 \(w\in\{1,2,3\}\)，图外统一视作背景 0。候选集合严格封闭为：

\[
C_k^{grad}(w)=(S_k\oplus B_w)\setminus(S_k\ominus B_w),
\]
\[
C_k^{in}(w)=S_k\setminus(S_k\ominus B_w),
\qquad
C_k^{out}(w)=(S_k\oplus B_w)\setminus S_k.
\]

最终二值 target 为 \(C^{q,w}=\bigvee_{k=1}^{K}C_k^q(w)\)，其中 \(q\in\{grad,in,out\}\)。重叠一律布尔 OR，不计重叠次数。禁止对 \([Y>0]\) 整图求边界后冒充本 target。

### 小实例规则三选一

当 \(S_k\ominus B_w=\varnothing\) 时，仅允许预注册一个策略：

- `small_keep_defined`：保留上述集合运算的原始结果；
- `small_force_inner`：令该实例贡献 \(C_k=S_k\)，适用于三种 q；
- `small_ignore_loss`：target 仍按原式生成，同时产生 `valid_mask=0` 于 \(S_k\oplus B_w\)，该区域不计 contour loss。

三者不得按样本动态选择。默认尚未选定，06 manifest 必须给唯一枚举值；未选即 blocked。

### 触边、空图与标签规则

图外 padding 恒为背景，因此触边实例只生成图内可观察轮廓，禁止补造图外像素。`K=0` 时返回全零 target 与全一 valid mask。实例 ID 可不连续但必须是非负整数；NaN、负值、浮点非整数直接报 target validation failure。输入实例之间理论上不重叠；同一像素多 ID 属资产错误而非 OR 规则处理对象。

## Target 校验硬门

每批未来资产必须记录并通过：输入/输出 shape 相同；target/valid_mask 值域为 `{0,1}`；背景空图全零；每个非小实例按 q/w 重算完全一致；触边无越界；实例 ID permutation 后 target 不变；禁止出现由整图二值外边缘替代造成的 touching seam 丢失；固定微型合成例覆盖单像素、2×2、接触双实例、触边实例和空图。任一失败时 `target_valid=false`、N1 blocked。

## Loss 有限候选

令 contour logit 为 \(z\)、target 为 \(c\)、valid mask 为 \(m\)。仅允许：

\[
L_{bce}=\frac{\sum_xm_x\,BCEWithLogits(z_x,c_x;pos\_weight=p)}{\sum_xm_x},
\quad p\in\{1,2,4\}.
\]

正式集合定义为 `pos_weight ∈ {1,2,4}`。Dice 使用 \(\epsilon=10^{-6}\)：

\[
L_{dice}=1-\frac{2\sum m\sigma(z)c+\epsilon}{\sum m\sigma(z)+\sum mc+\epsilon}.
\]

总 loss 仅两族：`BCE(p)`；或 `BCE+Dice: λ_bce L_bce + λ_dice L_dice`，权重集合仅 `{(1,1),(1,0.5)}`。区域主损失保持冻结，轮廓辅助总系数仅 `{0.25,0.5}`。不得新增 focal/Tversky 或连续调权。论文未提供这些值，它们均为项目候选。

## 回退条件
