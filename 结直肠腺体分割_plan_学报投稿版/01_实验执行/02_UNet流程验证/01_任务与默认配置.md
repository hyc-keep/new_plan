## 3. 本阶段唯一允许处理的变量

### 3.1 当前阶段唯一有效任务边界

当前阶段统一只承认下面这组边界：

- 任务：`2D` 结直肠组织病理腺体分割
- 主数据集：`GlaS`
- 训练 split：`train68`
- 验证 split：`val17`
- 正式测试 split：`testA60 / testB20`
- 主指标：`F1 / Object Dice / Object Hausdorff`
- 补充指标：`Dice / IoU / HD95 / Boundary F1`
- 模型角色：`UNet` 只承担流程排雷与最小可复现基线角色

对应符号固定为：

- 原始实例标签：`M_inst(x) in {0, 1, ..., K}`
- 当前训练目标：`y(x) in {0,1}`
- 预测 `logit`：`z = f_theta(x)`
- 预测概率：`p = sigmoid(z)`

### 3.2 当前阶段明确不做的事情

当前阶段不允许借“默认配置”之名提前打开：

- `CRAG`
- 外部对比
- `ResNet34-U-Net`
- `LKMA`
- `Boundary Head`
- `Distance-aware Loss`
- stain normalization
- postprocess
- TTA
- 测试集回调阈值

这样规定的原因是：

- `A1` 要先证明基础训练和评估链可信。
- 默认配置文件只负责给首版协议定起点，不负责把后续变量一并打开。
- 如果现在同时打开多个变量，后续就无法判断闭环跑通究竟来自基础协议还是来自额外模块。

## 4. 阶段门控表达式

### 4.1 任务定义与 benchmark 角色

- 当前结论：本阶段固定任务为 `GlaS` 上的 `2D` 腺体分割首版流程验证，目标是在 `train68 / val17 / testA60 / testB20` 上完成标准 `UNet` 闭环。
- 规则类型：`官方协议固定项 + 路线层已锁定`
- 适用阶段：`02_UNet流程验证`
- 直接依据：路线依据来自 02_路线与投稿/01、02_路线与投稿/02；文献依据来自 `结直肠腺体分割_plan_优化版/03_文献证据/05_腺体任务经典论文/01_GlaS-Challenge.md`
- 采用原因：`A1` 阶段的核心是把单一 benchmark 的主链路跑通，而不是提前扩大实验面。
- 不采用的相邻方案：不采用当前阶段同时引入 `CRAG`；不采用把 `UNet` 首轮写成最终主模型实验。
- 代码落点：`crc_gland_segmentation_project/configs/data/glas.yaml`，configs/experiment/A1_UNet_GlaS_v1_seed3407.yaml
- 运行记录字段：`stage`，`dataset`，`split_train`，`split_val`，`split_testa`，`split_testb`
- 验收方式：检查正式 run 只消费 `GlaS` 固定 split；检查 run_meta.yaml 不出现 `CRAG` 或外部对比字段。

### 4.2 标签形式与前景定义

- 当前结论：训练标签固定为 `instance_to_binary`，前景定义固定为 `y(x) = 1[M_inst(x) > 0]`。
- 规则类型：`工程冻结规则`
- 适用阶段：数据阶段冻结后，`A1` 开始继承；后续 `A2`、`Baseline` 和模块阶段继续继承
- 直接依据：路线依据来自 02_路线与投稿/02；文献依据来自 01_GlaS-Challenge.md、04_MILD-Net.md；上游依据来自 `01_数据协议/*`
- 核心公式或定义参考：`y(x) = 1[M_inst(x) > 0]`

### 4.3 输入通道、归一化与 resize 规则

- 当前结论：输入固定为 `RGB`，归一化固定为 `ImageNet mean/std`，图像 resize 使用双线性插值，mask resize 使用最近邻，最终张量尺寸固定为 `512x512`。
- 规则类型：`论文支持的候选范围 + 工程冻结规则`
- 适用阶段：`A1` 首轮开始生效；通过后冻结到 `A2` 与 `Baseline`
- 直接依据：路线依据来自 02_路线与投稿/01、02_路线与投稿/02；文献依据来自 02_U-Net.md、05_ResNet.md；参数冻结依据来自 `结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/02_参数冻结总表.md`
- 核心公式或定义参考：

```text
x_raw -> Resize_bilinear(512, 512) -> x_float = x_raw / 255
x_norm[c,h,w] = (x_float[c,h,w] - mu_c) / sigma_c
y_bin -> Resize_nearest(512, 512)
```

- 采用原因：统一 `RGB + 512x512 + ImageNet mean/std` 可以兼顾腺体上下文、单卡可训练性和后续 `ResNet34-U-Net` 兼容性。
- 不采用的相邻方案：不采用灰度输入；不采用 mask 双线性插值；不采用 `U-Net 2015` 的 `572 -> 388` valid-conv 尺寸链作为当前工程默认值。

- 验收方式：检查 train/val/test 配置全都记录 `512x512` 和相同归一化；检查 resize 后 mask 仍为二值。

### 4.4 标准 `UNet` 结构角色与继承边界

- 当前结论：当前正式模型固定为标准单输出 `UNet`，输出单通道前景 `logit`，仅继承 `U-shaped encoder-decoder + skip connection` 的结构思想，不照搬原论文训练 recipe。
- 规则类型：`论文直接支持 + 工程冻结规则`

- 核心公式或定义参考：

```text

p = sigmoid(z)
decoder_l = Decode(Concat(F_l^enc, Up(F_{l+1}^dec)))
```

- 采用原因：标准 `UNet` 最适合在最低复杂度下暴露数据、loss、评估与记录链中的基础问题。
- 不采用的相邻方案：不采用预训练 encoder 版本；不采用 attention 变体；不采用边界头、距离头或多任务输出。

### 4.6 主损失函数

- 当前结论：首版主损失固定为 `L_seg = L_BCE + L_Dice`。
- 规则类型：`论文支持的候选范围 + 工程冻结规则`
- 适用阶段：`A1` 开始生效；至少继承到 `Baseline`
- 直接依据：文献依据来自 `结直肠腺体分割_plan_优化版/03_文献证据/07_综述与写作支撑/02_Loss-Survey.md`；冻结依据来自 `结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/02_参数冻结总表.md`
- 核心公式或定义参考：

```text
L_BCE = - 1/N * sum_i [y_i log(p_i) + (1-y_i) log(1-p_i)]
L_Dice = 1 - (2 * sum_i p_i y_i + eps) / (sum_i p_i + sum_i y_i + eps)
L_seg = L_BCE + L_Dice
```

- 采用原因：它在首轮阶段兼顾数值稳定性和区域重叠约束，适合作为腺体分割的最小可解释损失组合。
- 不采用的相邻方案：不采用单独 `BCE`；不采用单独 `Dice`；不采用复杂 focal/topology/distance 类损失提前进入 `A1`。
- 代码落点：src/losses/seg_losses.py，configs/train/unet_flow_v1.yaml
- 运行记录字段：`loss_name`，`loss_version`，`bce_weight`，`dice_weight`，`loss_eps`
- 验收方式：检查训练日志至少保留 `loss_total / loss_bce / loss_dice`；检查 config.yaml 明确写出 loss 版本与参数。

### 4.7 默认优化与停止协议

- 当前结论：默认训练协议固定为 `AdamW + lr 1e-3 + weight_decay 1e-4 + ReduceLROnPlateau(monitor=val_objdice) + epoch_max 120 + early_stopping 20 + AMP on`。

### 4.8 `batch size` 与 smoke check 边界

- 当前结论：正式 run 的 `batch_size` 取单卡稳定不溢出的最大值，但 `A1` 阶段只允许记录一个正式值；正式 run 前必须先完成一次不入表的 smoke check。

补充冻结说明：

```text
batch_size_candidate = max {b | smoke_check(train_step, b) stable and no OOM}
a1_formal_batch_size = batch_size_written_in_first_formal_run_meta
a1_batch_ready = smoke_check_pass and a1_formal_batch_size_recorded
```

这里必须额外写死三条执行边界：

- smoke check 只负责确定 `batch_size_candidate`，不承担正式结果身份。
- 一旦第一条正式 `A1` run 成功写入 run_meta.yaml，该值就固定记为 `a1_formal_batch_size`，后续 `03_UNet稳定性` 不允许静默改写。
