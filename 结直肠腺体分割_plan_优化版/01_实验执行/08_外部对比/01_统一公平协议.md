# 统一公平协议

本文件不是给 `08_外部对比` 再补一页“大家尽量公平”的原则说明，也不是把 `结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/02_UNetPP适配方案.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/03_DeepLabV3P适配方案.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/04_AttUNet适配方案.md` 前面再套一层重复模板。

它在当前阶段中的唯一职责是：

> 作为 `08_外部对比` 的统一母协议，把 `current_mainline`、统一数据协议、统一训练协议、统一评估协议、最小必要差异、来源边界、三 seed 补证、主表资格和下游 handoff 一次写死。自本文件起，外部对比不再允许“方法接进来后再解释是否公平”，所有正式 run、正式表格和正式结论都只能在这条冻结链内成立。

从现在开始，`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/02_UNetPP适配方案.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/03_DeepLabV3P适配方案.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/04_AttUNet适配方案.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/05_增强对比进入条件.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/06_引用值与复现值标记规则.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/07_阶段验收.md` 都只能展开本文件，不得另起一套公平口径。若冲突，以本文件为准回改。

## 1. 文件角色与执行边界

### 1.1 当前文件负责什么

当前文件只负责冻结下面八件事：

1. `08_外部对比` 阶段唯一合法的公平比较起点，也就是 `current_mainline vs external architectures`。
2. 首批外部方法共享的统一数据、训练、评估和结果记录协议。
3. `allowed_gap` 的合法边界，也就是哪些差异属于“最小必要适配”，哪些差异已经破坏公平。
4. `reproduced / † / *`、`single_seed / mean+-std`、`direct_comparison_eligible` 与主表资格的绑定关系。
5. `need_three_seed` 的统一触发方式，以及单 seed 结果何时必须升级为正式聚合结果。
6. 同层方法文件必须继承的字段链、禁止重开的变量和统一来源边界。
7. 当前公平协议在代码中的入口函数、前置断言、最低交付物和下游 handoff 资产。
8. 当前文件自身的独立 `回退条件`、`文件质量自检` 和独立 `Diagnostics 闭环`。

### 1.2 当前文件不负责什么

当前文件不重新定义下面这些内容：

- `07_Distance` 的正式 `keep / downgrade / drop` 结论。
- `01_数据协议`、`结直肠腺体分割_plan_优化版/01_实验执行/04_Baseline/02_训练协议.md` 中已经冻结的数据、训练和评估协议本体。
- `02/03/04` 三个方法文件各自的结构本体与接入细节。
- `结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/05_增强对比进入条件.md` 对增强外部是否进入的阶段决策。
- `结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/06_引用值与复现值标记规则.md` 的具体表注话术与表格呈现细节。
- `结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/07_阶段验收.md` 对整个 `08` 阶段的最终通过裁决。
- `09/10/11` 下游阶段各自的角色定义与总裁决逻辑。

### 1.3 为什么它必须单独存在

如果没有一份独立的统一公平协议，`08_外部对比` 最容易出现的不是“完全没做”，而是下面几种伪完成状态：

- 每个外部方法都能跑，但实际沿用不同训练口径、不同 checkpoint 选择器或不同阈值来源。
- 主表里写了 `UNet++ / DeepLabV3+ / Attention U-Net`，却说不清哪些是统一协议复现值，哪些是引用值，哪些带特殊测试设置。
- `02/03/04` 各自重写 `best_selector`、阈值来源、后处理和测试链，导致同层内部出现多套公平定义。
- `09/10/11` 想消费外部层时，只能读到“外部方法做过”，却读不到“哪些值真能进主表、哪些值只能当引用对照、哪些值必须降级出表”。

因此，本文件的角色不是“补充说明”，而是把 `fairness_protocol_fixed` 真正压成 `Gate_H1` 的前置门。

## 3. 本阶段唯一允许处理的变量

### 3.1 当前文件真正要证明什么

`08_外部对比` 的统一公平协议要证明的不是“又跑了几个通用模型”，而是：

> 在 `Distance` 已经给出唯一 `current_mainline`、数据训练评估协议已经冻结的前提下，`UNet++ / DeepLabV3+ / Attention U-Net` 能否在“只换结构、不换协议”的边界内形成可复查、可分层、可进入主表第一层的正式比较结果；同时所有来源标记、直比资格、聚合方式和下游 handoff 字段也已经一次写死。

### 3.2 当前阶段唯一合法起点

本阶段不重新选择主线，而是严格继承 `07_Distance` 的 handoff：

```text
current_mainline = (distance_decision_level == keep) ? distance_kept_base : distance_input_base
```

这意味着：

- 只有 `Distance` 已正式收口，`08` 才允许启动。
- `distance_kept_base` 和 `distance_input_base` 不允许在本阶段被同时当成比较起点。
- 所有正式外部 run 都必须在字段链上追溯到同一个 `current_mainline`。

### 3.3 当前阶段唯一公平变量

本阶段唯一允许变化的核心变量固定为：

```text
external_run_fair = unified_data_proto + unified_train_proto + unified_eval_proto + documented_minimal_gap_only
```

其中：

- `unified_data_proto`：不改 benchmark、不改 split、不改标签口径、不改输入尺寸和归一化。
- `unified_train_proto`：不改优化器、不改训练预算、不改 selector、不改单独方法训练技巧。
- `unified_eval_proto`：不改单独阈值来源、不改单独后处理、不改单独 TTA、不改 `TestA / TestB` 分开汇报。
- `documented_minimal_gap_only`：只允许留下最小必要适配，且必须结构化记录。

### 3.4 最小必要差异主式

```text
allowed_gap = minimal_necessary_gap(external_architecture)
```

这条式子只允许两类差异存在：

1. 架构本体接入当前统一框架所必需的输入输出适配。
2. 文献与工程实现之间无法消除、但已经显式记录的最小差异。

以下内容不自动属于 `allowed_gap`：

- 更强预训练
- 私有后处理
- TTA
- 特殊测试链
- 默认仓库训练技巧
- 自定义阈值策略
- 额外监督头
- 模型家族外的结构替换

### 3.5 三 seed 与直比资格主式

```text
need_three_seed = (gap_to_mainline <= 1.5pt) OR (rank == 2)
```

```text
direct_comparison_eligible = same_benchmark AND same_split AND same_metric_protocol AND no_major_test_gap
```

固定解释如下：

- `need_three_seed` 只对关键竞争者触发，防止全部方法机械补三 seed，也防止关键竞争者只用单 seed 结论。
- `direct_comparison_eligible` 是主表与 task-specific direct comparison 的共同准入门，不满足时只能降级为参考说明或引文区。

### 3.6 `fairness_protocol_fixed` 与阶段门关系

本文件必须把 `fairness_protocol_fixed` 压成显式门，而不是抽象口号：

```text
fairness_protocol_fixed = current_mainline_frozen
                          AND unified_data_proto_frozen
                          AND unified_train_proto_frozen
                          AND unified_eval_proto_frozen
                          AND allowed_gap_rule_fixed
                          AND source_boundary_fixed
                          AND main_table_eligibility_fixed
                          AND handoff_field_fixed
```

它与阶段总门的关系固定为：

```text
Gate_H1 = current_mainline_ready
          AND fairness_protocol_fixed
          AND core_external_runs_complete
          AND source_boundary_ready
          AND external_table_ready
          AND handoff_ready
```

也就是说，只要本文件没有真正把公平协议写成可执行硬规则，`Gate_H1` 就不能视为成立。

## 4. 阶段门控表达式

下面所有核心规则都写成最终协议格式。后面写代码、写同层适配文件和写结果表时，只允许把这些规则实现出来，不允许擅自补新的隐含规则。

### 4.1 `current_mainline` 唯一起点规则

- 当前结论：外部对比阶段只允许从 `Distance` 正式结论导出的唯一 `current_mainline` 出发。
- 规则类型：`handoff 冻结规则 + 阶段起点规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/00_阶段总协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/00_阶段总协议.md`
- 核心公式或定义参考：`current_mainline = (distance_decision_level == keep) ? distance_kept_base : distance_input_base`
- 采用原因：如果起点不唯一，后面的公平协议和下游主表资格都会失去统一参照物。
- 不采用的相邻方案：不采用“哪个主线看起来更强就先拿哪个比”；不采用把 `distance_kept_base` 和 `distance_input_base` 同时写入同层主表。
- 代码落点：crc_gland_segmentation_project/configs/experiment/external_h1_v1.yaml、crc_gland_segmentation_project/scripts/run_external.py
- 运行记录字段：`current_mainline`、`mainline_source_note`、`distance_decision_level`
- 验收方式：检查所有正式外部 run 和汇总表都只引用同一个 `current_mainline` 字段值。

### 4.2 统一数据协议规则

- 当前结论：所有正式外部 run 必须共享同一数据协议。
- 规则类型：`数据冻结规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/01_数据协议/00_阶段总协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/04_Baseline/02_训练协议.md`
- 核心公式或定义参考：`unified_data_proto = dataset + split + label_rule + input_spec + normalization`
- 采用原因：只要 split、标签或输入规格变化，比较就不再是同一 benchmark 上的公平比较。
- 不采用的相邻方案：不允许某个外部方法单独改输入尺寸、重划 split、复制一套新数据目录或改标签口径。
- 代码落点：`crc_gland_segmentation_project/configs/data/glas.yaml`、`crc_gland_segmentation_project/splits/glas/*.csv`
- 运行记录字段：`data_proto_version`、`dataset_code`、`input_size`、`label_rule`、`normalization_version`
- 验收方式：检查所有正式外部 run 都复用同一数据协议版本。

### 4.3 统一训练协议规则

- 当前结论：外部方法默认共享同一训练协议，首轮只允许替换架构本体。
- 规则类型：`训练冻结规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/04_Baseline/02_训练协议.md`
- 核心公式或定义参考：`unified_train_proto = aug + loss + optimizer + scheduler + epoch_budget + checkpoint_rule`
- 采用原因：公平比较的底线是“只换结构，不换训练游戏规则”。
- 不采用的相邻方案：不沿用第三方仓库默认优化器、默认 epoch、默认 selector、默认 warmup 或默认 tricks 直接横比主线。
- 代码落点：`crc_gland_segmentation_project/scripts/train.py`、crc_gland_segmentation_project/configs/external/*.yaml、crc_gland_segmentation_project/src/eval/checkpoint_selector.py
- 运行记录字段：`train_proto_version`、`aug_version`、`loss_version`、`optimizer_name`、`best_selector`、`physical_batch_size`、`grad_accum_steps`、`effective_batch_size`
- 验收方式：检查外部方法没有单独改动训练主协议。

补充冻结说明：

```text
effective_batch_size = physical_batch_size * grad_accum_steps
target_effective_batch_size = current_mainline_effective_batch_size
```

这条补充规则的固定含义是：

- 公平比较看的是 `effective_batch_size`，不能只看某个仓库表面写的 `batch size`。
- 若某个外部架构因为显存占用更高，无法直接保持与 `current_mainline` 相同的 `physical_batch_size`，则必须优先用 `grad_accum_steps` 把 `effective_batch_size` 尽量补齐到主线水平。
- 只有在显存或实现约束下仍然无法完全补齐时，才允许留下“最小必要差异”，并且必须把原因写入 `fairness_adjustment / allowed_gap_summary`。
- 不允许某个外部方法在没有记录的情况下，把更小的 `effective_batch_size`、不同的更新频率或不同的梯度累积策略直接混入正式公平比较。
- 若 `effective_batch_size` 与主线不一致，则该差异必须和 `implementation_source / adaptation_note / paper_setting_gap / fairness_adjustment` 一起进入正式字段链与阶段表。

### 4.4 统一评估协议规则

- 当前结论：所有外部方法必须共享同一评估协议。
- 规则类型：`评估冻结规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/04_评估口径与官方脚本对齐.md`
- 核心公式或定义参考：`unified_eval_proto = inference_chain + metric_chain + split_reporting + source_boundary + eval_cast_policy + boundary_metric_impl + connected_components_impl`
- 采用原因：阈值来源、`dtype` 处理、边界指标实现、连通域实现、后处理、TTA 和 `TestA / TestB` 汇报方式都会直接改变结论。
- 不采用的相邻方案：不允许某个外部方法带特殊后处理、特殊测试链、特殊 TTA 或合并 `TestA / TestB` 后再与主线横比；不允许某个外部方法在 `AMP` 低精度 `logits / probabilities` 上直接做正式 `threshold`。
- 代码落点：crc_gland_segmentation_project/src/eval/run_eval.py、crc_gland_segmentation_project/src/metrics/object_metrics.py、crc_gland_segmentation_project/src/metrics/boundary_metrics.py
- 运行记录字段：04 阶段 Gate 冻结的 `eval_proto_version`、`eval_cast_policy`、`threshold_value`、`threshold_source`、`boundary_metric_width`、`boundary_metric_impl`、`connected_components_impl`、`connected_components_connectivity`、`postprocess_version`、`tta_version`
- 验收方式：检查推理链、指标链和结果导出链与主线完全对齐；检查正式评估时 `threshold` 之前的 `logits / probabilities` 已保持为或显式转换为 `float32`；检查边界指标与连通域实现没有在不同外部方法之间漂移。

补充冻结说明：

```text
eval_cast_policy = logits/probabilities must be kept or cast to float32 before thresholding
unified_eval_pipeline = logits(float32) -> sigmoid(float32) -> threshold -> binary mask(uint8/bool) -> connected components -> metrics(float32)
boundary_metric_width = 3 px
boundary_metric_impl = skimage.segmentation.find_boundaries(mode=inner) + binary_dilation
connected_components_impl = scipy.ndimage.label
connected_components_connectivity = 8
```

也就是说：

- 即使训练或推理前向开启 `AMP`，正式外部比较链里的 `threshold` 之前也必须先回到 `float32`
- 不允许某个外部方法因为第三方仓库默认写法不同，就在 `float16 / bfloat16` 概率图上直接二值化后再与主线横比
- `Boundary F1` 的正式实现固定为 `3 px` 宽度，不允许某个外部方法单独改成其它边界宽度或其它边界提取方式
- 连通域统计固定使用 `scipy.ndimage.label` 和 `8-connectivity`，不允许在对象级指标链中为单个方法偷偷切换实现

### 4.5 最小必要差异记录规则

- 当前结论：外部方法若无法完全无差异接入，只允许保留最小必要差异，且必须结构化记录。
- 规则类型：`适配冻结规则 + 来源记录规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/00_阶段总协议.md`、`02/03/04` 同层方法文件
- 核心公式或定义参考：`allowed_gap = minimal_necessary_gap(external_architecture)`
- 采用原因：真实世界的外部适配可以有少量工程差异，但差异必须被压到最小并且可追溯。
- 不采用的相邻方案：不接受“原仓默认就是这样”的免责说法；不把更强预训练、测试增强、私有后处理和额外监督头伪装成结构本体。
- 代码落点：external_reproduction_note.md、run_meta.yaml、crc_gland_segmentation_project/reports/tables/*.csv
- 运行记录字段：`implementation_source`、`adaptation_note`、`paper_setting_gap`、`fairness_adjustment`、`allowed_gap_summary`
- 验收方式：检查每个正式外部方法都能追溯到实现来源、适配说明和公平修正说明。

### 4.6 三 seed 补证规则

- 当前结论：首批外部方法首轮固定先跑 `seed3407`；满足竞争条件时必须补齐三次重复。
- 规则类型：`成本控制规则 + 稳定性补证规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/00_阶段总协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/07_阶段验收.md`
- 核心公式或定义参考：`need_three_seed = (gap_to_mainline <= 1.5pt) OR (rank == 2)`
- 采用原因：先用单 seed 控制成本，再为关键竞争者补齐稳定性证据。
- 不采用的相邻方案：不一开始全部三 seed；也不把所有方法都停留在单次 screening 后直接进入正式结论层。
- 代码落点：crc_gland_segmentation_project/scripts/run_external.py、crc_gland_segmentation_project/scripts/compare_runs.py
- 运行记录字段：`train_seed`、`need_three_seed_followup`、`rank_in_external_group`、`aggregation`
- 验收方式：检查三 seed 触发完全按统一规则执行，且 `single_seed / mean+-std` 身份明确。

### 4.7 来源标记与直比准入规则

- 规则类型：`主表资格规则 + 写作分层规则`
