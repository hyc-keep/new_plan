# 可视化与失败案例清单

## 1. 文件角色与执行边界

### 1.1 当前文件负责什么

这份文件不是“图像素材备忘录”，而是 `10_结果汇总` 阶段里专门负责 `figures_ready` 子门的协议文件。

它当前只负责把下面八件事写死：

1. 什么样的图像资产才算正式结果包证据，而不是临时导出图或好看截图。
2. `GlaS`、`CRAG`、外部对比、主线逐步消融、局部放大和失败案例六类图像资产的唯一身份与最小集合。
3. image / gt / pred 三元组、样本级追溯字段、错误类型文本说明和正文候选图分层在图像层里的唯一合法写法。
4. 图像资产与 `official_result_record`、正式表格层、`stage summary`、投稿结果包检查表和下游总验收之间的字段映射关系。
5. reproduced / † / *、single_seed / mean +- std、GlaS / CRAG、TestA / TestB / test40、A / B / C 在可视化层中的来源边界与写作边界。
6. 失败案例记录、筛选顺序、正文候选图与备份图分层、图像命名规则和目录冻结规则。
7. 当前文件自己的独立 `回退条件`、`代码落地接口`、`冲突裁决记录`、`文件质量自检` 和独立 `Diagnostics 闭环`。
8. 当前文件作为 `结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/00_阶段总协议.md` 中 `figures_ready` 子门展开协议的正式职责。

### 1.2 当前文件不负责什么

当前文件不重新定义下面这些内容：

- 哪些 run 先算正式结果；这些由 `结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/01_结果记录模板.md` 冻结。
- 哪些结果具有正式入表资格；这些由 `结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/02_主表与消融表回填规则.md` 冻结。
- 各阶段为什么保留、降级或删除模块；这些由 `02-09` 阶段 handoff 和 `stage summary` 给出。
- 投稿结果包最终落到 ready / ready_with_minor_gaps / not_ready 的哪一档；这些由 `结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/05_投稿结果包检查表.md` 和 `11_总验收与止损` 下游文件裁决。

### 1.3 为什么当前文件必须独立存在

结果汇总阶段最容易出现的伪完成状态不是“没导图”，而是：

- 有很多成功案例，但没有失败案例和文字级错误说明。
- 图像已经导出，但说不清它对应哪条正式 run、哪张表、哪个 split 和哪个版本。
- 图里只剩裁剪局部，没有原图和真值，后面无法支撑写稿或审稿回复。
- 正文候选图、补充材料图和普通调试图混在一起，到了结果包阶段还要重新猜。
- `CRAG` 已有表格，却没有与 A / B / C 等级一致的图像证据层。
- 外部方法图像存在，但来源边界不清，后面很容易把 `†` 或 `*` 结果伪装成公平视觉对比。

如果图像层不独立冻结，后面 04_stage_summary模板.md、05_投稿结果包检查表.md、11_总验收与止损/* 都会再次面对“表已经有了，但图到底能不能作为正式证据”的灰区。

## 3. 本阶段唯一允许处理的变量

### 3.1 图像证据层的统一定义

从现在开始，正式图像证据层固定理解为：

```text
figure_evidence
= success_cases
+ failure_cases
+ comparison_panels
+ zoom_evidence
+ paper_candidate_registry
+ textual_error_notes
```

并且：

```text
figure_asset
= image
+ gt
+ pred
+ sample_trace
+ source_boundary
+ error_note
+ eval_implementation_chain
```

其中：

- `success_cases`：证明主线收益仍然成立的代表样本。
- `failure_cases`：证明局限性、剩余错误类型和 stop-loss 边界的代表样本。
- `comparison_panels`：主线逐步消融、`CRAG` 比较和外部对比的标准并排图。
- `zoom_evidence`：边界、黏连、复杂轮廓和局部失败的放大图。
- `paper_candidate_registry`：正文候选、补充候选、备份图和普通导出图的正式分层。
- `textual_error_notes`：与每张关键图配套的错误类型、短说明和图像来源身份。

### 3.2 当前文件唯一合法输入资产

当前文件只允许消费下面这些正式资产：

- `official_result_record`
- `formal_tables`
- `visual_dir`
- `result_source_type`
- `aggregation`
- sample_id / split_role / dataset / run_name
- success_failure_summary / abnormal_case
- `conclusion_grade`

当前文件明确不允许：

- 先凭审美挑图，再倒推要补哪条 run 或哪条表格。
- 用未进入正式记录层的临时截图伪装成正文候选图。
- 只基于论文引文值或特殊设置结果生成“公平视觉对比图”。
- 把 GlaS / CRAG、TestA / TestB / test40、single_seed / mean +- std 混成不可追溯的视觉资产。

### 3.3 当前文件在 `Gate_J1` 中的角色

`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/00_阶段总协议.md` 把 `figures_ready` 写成正式子门。

当前文件就是这个子门的展开协议：

```text
figures_ready
= success_cases_ready
AND failure_cases_ready
AND comparison_panels_ready
AND zoom_ready
AND paper_candidates_ready
AND naming_trace_ready
AND text_notes_ready
```

如果 `figures_ready = false`，后面的 `stage summary`、结果包检查和总验收都默认不得宣称正式完成。

## 4. 阶段门控表达式

后面统一只承认下面九条总原则：

1. 先有正式 run 和正式表格，再允许出正式图。
2. 先保留 image / gt / pred 三元组，再讨论局部放大和裁剪。
3. 先保留成功案例与失败案例成对证据，再讨论正文候选图。
4. 先区分 GlaS / CRAG 与 TestA / TestB / test40，再做联合图像说明。
5. 先区分 reproduced / † / * 和 single_seed / mean +- std，再讨论是否构成公平视觉对比。
6. 先写清错误类型和短说明，再把失败图推进到 summary 或结果包。
7. 先把正文候选图与补充 / 备份图区分开，再讨论写稿版面。
8. 先保留可追溯字段，再允许压缩成论文图版。
9. 只留漂亮图、不留失败图、不留文字说明的结果包，一律视为未完成。

如果一张图只能证明“模型最好看的一面”，却不能解释失败、局限和来源边界，这张图默认不具备正式证据资格。

## 5. 核心规则卡片

### 5.1 成功案例与失败案例成对规则

- 当前结论：每个关键阶段、每个关键 benchmark 都必须同时保留成功案例和失败案例，不能只留优势图
- 规则类型：`工程冻结规则 + 写作边界规则`
- 适用阶段：`10_结果汇总`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/00_阶段总协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/05_投稿结果包检查表.md`
- 核心公式或定义参考：`case_pair = success_case + failure_case`
- 采用原因：论文结果层必须同时支撑贡献和局限，不能只服务“主观好看”
- 不采用的相邻方案：不采用只留成功样本；不采用失败图留到最后再补；不采用失败图没有文字说明
- 代码落点：reports/figures/glas/*、reports/figures/crag/*、notes/failure_case_notes.md
- 运行记录字段：`dataset`, `split_role`, `sample_id`, `model_name`, `error_type`, `short_note`, `paper_candidate`
- 验收方式：检查 `GlaS` 和 `CRAG` 都同时存在成功图、失败图和对应说明

### 5.2 image / gt / pred 三元组规则

- 当前结论：每张正式关键图必须能回到完整的 image / gt / pred 三元组，局部放大图不能脱离原图上下文独立存在
- 规则类型：`工程冻结规则`
- 适用阶段：`10_结果汇总`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/01_结果记录模板.md`、`结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/05_代码工程映射与实现策略.md`
- 核心公式或定义参考：`figure_triplet = image + gt + pred`
- 采用原因：没有原图和真值的图无法支持审计、写稿、审稿回复和错误定位
- 不采用的相邻方案：不采用只留预测图；不采用只留局部裁剪；不采用放大图没有原图索引
- 代码落点：reports/figures/*、scripts/validate_sources.py
- 运行记录字段：`image_path`, `gt_path`, `pred_path`, `zoom_bbox`, `source_image_id`, `eval_cast_policy`, `boundary_metric_width`, `boundary_metric_impl`, `connected_components_impl`, `connected_components_connectivity`
- 验收方式：检查每张关键图都能追溯到原图路径、真值路径和预测路径

### 5.3 `GlaS` 主线逐步对比图规则

- 当前结论：`GlaS` 正式主线对比图只允许按 `UNet -> ResNet34-U-Net -> + LKMA -> + Boundary Head -> + Distance-aware Loss` 的固定顺序组织
- 规则类型：`路线冻结规则`
- 适用阶段：`10_结果汇总`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/02_主表与消融表回填规则.md`、`结直肠腺体分割_plan_优化版/02_路线与投稿/02_结直肠腺体分割_分阶段实验路线与执行标准.md`
- 核心公式或定义参考：`visual_ablation_order = [UNet, Baseline, LKMA, Boundary, Distance]`
- 采用原因：图像层必须与表格层和主线实验顺序完全一致，避免视觉叙事与消融表冲突
- 不采用的相邻方案：不按“最好看”重新排列模型；不把已删除模块继续放入正文主线图；不混入未正式入表版本
- 代码落点：reports/figures/glas/ablation/、scripts/compare_runs.py
- 运行记录字段：`ablation_order`, `model_name`, `supports_mainline`, `can_enter_ablation_table`
- 验收方式：检查主线对比图的模型顺序、图注语义和正式消融表一致

### 5.4 `CRAG` 图像资产规则

- 当前结论：`CRAG` 图像层至少必须保留 `UNet`、`ResNet34-U-Net`、`current_mainline 的成功 / 失败案例，以及 ResNet34-U-Net vs current_mainline` 的正式对比图
- 规则类型：`handoff 规则 + 结果解释规则`
- 适用阶段：`10_结果汇总`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/09_CRAG验证/03_结果解释规则.md`、`结直肠腺体分割_plan_优化版/01_实验执行/09_CRAG验证/04_阶段验收.md`
- 核心公式或定义参考：`crag_visual_chain = UNet + Baseline + current_mainline + grade_consistent_cases`
- 采用原因：`CRAG` 是补充 benchmark，图像层必须服务其 A / B / C 解释边界，而不是额外制造主张
- 不采用的相邻方案：不只留成功案例；不脱离 A / B / C 等级单独挑图；不把 `CRAG` 图写成比 `GlaS` 更强的主证据
- 代码落点：reports/figures/crag/*、reports/stage_reports/crag_stage_summary.md
- 运行记录字段：`dataset`, `sample_id`, `model_name`, `conclusion_grade`, `allowed_claim`, `paper_candidate`
- 验收方式：检查 `CRAG` 图像资产与 A / B / C 等级、阶段总结和下游 handoff 语气一致

### 5.5 外部视觉对比规则

- 当前结论：外部视觉对比只允许服务“正式入表且具备来源边界说明”的方法，至少保留 `current_mainline vs 最强正式外部方法` 的代表图
- 规则类型：`来源边界规则 + 写作边界规则`
- 适用阶段：`10_结果汇总`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/06_引用值与复现值标记规则.md`、`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/02_主表与消融表回填规则.md`
- 核心公式或定义参考：`external_visual_entry = can_enter_main_table AND source_boundary_clear`
- 采用原因：外部视觉对比最容易因为来源不清和特殊设置差异而失真，必须先服从入表资格
- 不采用的相邻方案：不把 `†`、`*` 结果无说明地画成公平并排图；不为未进入正式表格的方法大量出图；不把参考区图像冒充主文对比图
- 代码落点：reports/figures/external_compare/、scripts/validate_sources.py
- 运行记录字段：`result_source_type`, `direct_comparison_eligible`, `has_special_test_setting`, `paper_candidate`, `eval_cast_policy`, `boundary_metric_width`, `boundary_metric_impl`, `connected_components_impl`, `connected_components_connectivity`
- 验收方式：检查外部视觉对比图都能解释“为什么能比”或“为什么仅供参考”

### 5.6 失败案例字段与错误类型规则

- 当前结论：每个失败案例都必须同时保留图像文件和文字说明，错误类型固定先统一为 miss / over-seg / under-seg / adhesion / boundary_break
- 规则类型：`工程冻结规则`
- 适用阶段：`10_结果汇总`
- 直接依据：旧版 03_可视化与失败案例清单.md 可继承内容、`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/01_结果记录模板.md`
- 核心公式或定义参考：`failure_case = sample_id + dataset + model + error_type + short_note + entry_target`
- 采用原因：失败案例是局限性证据，不是仅靠图片直觉描述的素材
- 不采用的相邻方案：不只留图片不写说明；不允许每次临时发明新的错误类型；不允许说明中缺样本和模型身份
- 代码落点：notes/failure_case_notes.md、reports/figures/*/failure/
- 运行记录字段：`sample_id`, `dataset`, `model_name`, `error_type`, `short_note`, `paper_candidate`, `linked_run_name`
- 验收方式：检查每个失败案例都具备固定字段和固定错误类型之一

### 5.7 失败案例筛选顺序规则

- 当前结论：失败案例固定按“共同失败但主线改善有限 -> baseline 失败而主线改善明显 -> 主线仍失败且最能暴露限制”三层顺序筛选
- 规则类型：`结果解释规则`
- 适用阶段：`10_结果汇总`
- 直接依据：旧版 03_可视化与失败案例清单.md 可继承内容、`结直肠腺体分割_plan_优化版/01_实验执行/09_CRAG验证/03_结果解释规则.md`
- 核心公式或定义参考：`failure_priority = shared_failure -> improved_failure -> unresolved_failure`
- 采用原因：这样筛选才能同时支撑优点、边界和局限，不会让结果包只剩“漂亮图”
- 不采用的相邻方案：不按主观好看度选失败图；不完全忽略仍然失败的限制样本；不只挑最极端样本不管代表性
- 代码落点：notes/failure_case_notes.md、scripts/summarize_stage.py
- 运行记录字段：`failure_priority`, `improved_vs_baseline`, `still_failed_in_mainline`
- 验收方式：检查失败案例层能同时展示改善样本和未解决样本

### 5.8 正文候选图分层规则

- 当前结论：图像资产必须至少分成 main_text / supplement / backup / not_for_paper 四层，正文候选图与普通导出图不得混放
- 规则类型：`写作冻结规则 + 工程冻结规则`
- 适用阶段：`10_结果汇总`
- 直接依据：旧版 03_可视化与失败案例清单.md 可继承内容、`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/05_投稿结果包检查表.md`
- 核心公式或定义参考：`paper_candidate in {main_text, supplement, backup, not_for_paper}`
- 采用原因：正文图选择是写稿入口条件之一，必须在结果汇总阶段先形成分层
- 不采用的相邻方案：不采用正文候选与普通图混在一个目录；不采用只靠文件名猜哪张进主文
- 代码落点：reports/figures/paper_candidates/*、submission_package_check.md
- 运行记录字段：`paper_candidate`, `used_in_stage_summary`, `used_in_submission_package`
- 验收方式：检查每张正式关键图都有明确候选层级和去向

### 5.9 图像到 summary 与结果包规则

- 当前结论：图像资产必须能直接支撑 `stage summary`、投稿结果包检查和总验收局限性段落，不能停留在“目录里有图”
- 规则类型：`交接放行规则`
- 适用阶段：`10_结果汇总`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/04_stage_summary模板.md`、`结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/05_投稿结果包检查表.md`
- 核心公式或定义参考：`figure_handoff = figure_evidence + failure_notes + paper_candidate_registry`
- 采用原因：图像层的完成标志不是导出若干 png，而是下游能直接消费其结论和局限性证据
- 不采用的相邻方案：不允许只交图不给文字说明；不允许只交局部裁剪不给样本追溯；不允许 summary 重新猜图像结论
- 代码落点：reports/stage_reports/*.md、submission_package_check.md
- 运行记录字段：`used_in_stage_summary`, `used_in_submission_package`, `supports_limitations`, `linked_table_row`, `eval_cast_policy`, `boundary_metric_width`, `boundary_metric_impl`, `connected_components_impl`, `connected_components_connectivity`

### 6.5 正文候选图最低集合

如果后面要形成可投稿结果包，正文候选图至少准备：

1. 1 组数据样例或任务示意图。
2. 1 组主线逐步对比图。

- 图像来自哪个 split。
- 当前图是否只反映单 seed。
- 外部对比图是否具备公平来源边界。

这些字段一旦漏掉，后面 summary、结果包检查和总验收都会整体返工。

## 8. 回退条件

### 8.1 独立回退触发条件

只要出现下面任意一条，当前文件就必须先回退修正，而不是直接进入 `stage summary`、结果包层或总验收：

- 关键图无法追到正式 run、样本 ID、split 或模型版本。
- 正式图像层只有成功案例，没有失败案例，或失败案例没有文字说明。
- 局部放大图无法回到完整 image / gt / pred 三元组。
- 外部视觉对比图的来源边界不清，或把 `†`、`*` 结果画成公平主对比图。
- `CRAG` 图像资产与 A / B / C 等级、允许 / 禁止写法不一致。
- 正文候选图、补充图和备份图没有正式分层，导致结果包无法直接消费。

### 8.2 固定回退顺序

回退时统一按下面顺序排查：

1. 先检查图像是否能回到正式 run、正式表格和样本字段。
2. 再检查 image / gt / pred 三元组和局部放大追溯关系是否完整。
3. 再检查失败案例字段、错误类型和筛选顺序是否可追溯。
4. 再检查来源边界、聚合身份和外部视觉对比资格是否统一。
5. 最后再检查正文候选图分层、summary 引用和结果包状态是否真的闭环。

### 8.3 回退后的重新放行条件

发生回退后，只有同时满足下面条件，才允许重新声明 `figures_ready = true`：

- 图像追溯链、失败案例字段和命名规则已补齐。
- 关键图的 image / gt / pred、样本字段、错误说明和候选层级重新对齐。
- `GlaS`、`CRAG` 和外部对比图层的来源边界与表格资格重新一致。
- 当前文件重新满足第 `5` 节全部规则卡片和第 `6-7` 节全部目录、字段和接口要求。
