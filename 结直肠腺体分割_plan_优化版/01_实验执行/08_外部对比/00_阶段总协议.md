## 3. 本阶段唯一允许处理的变量

### 3.1 当前阶段真正要证明什么

`08_外部对比` 阶段真正要证明的，不是“又跑了几个通用模型，主表看起来更丰富”，而是：

> 在数据协议、训练协议、评估协议、当前主线模型和结果记录规则全部冻结的前提下，`UNet++ / DeepLabV3+ / Attention U-Net` 作为首批通用外部对比，是否能在统一公平协议下形成可复查、可解释、可进入投稿主表第一层的正式外部比较层；同时任务内 gland-specific direct comparison 的来源边界、进入条件和后续补强路径也已经被写成唯一协议。

因此，本阶段通过的含义同时包括：

- `current_mainline` 已经唯一化，外部对比没有反向重开主线搜索。
- 首批外部方法已经被压成“只换结构、不换规则”的最小统一协议比较。
- `reproduced / † / *`、`single_seed / mean +- std`、direct comparison 准入和增强外部进入条件已经有正式边界。
- `09_CRAG验证`、`10_结果汇总` 和 `11_总验收与止损` 已经能直接读取当前阶段的正式外部层资产与结论。

### 3.2 当前阶段不允许用什么替代正式结论

当前明确不允许把下面这些情况写成“外部对比已完成”：

- 只接了第三方仓库并跑出一个数字，就宣称外部层已经成立。
- 不同外部方法各自沿用不同训练口径、不同阈值或不同后处理，却仍写成同一主表横比。
- 只写 `UNet++ / DeepLabV3+ / Attention U-Net` 三个名字，却没有实现来源、适配差异和来源边界记录。
- 首批外部层还没收口，就提前扩展 `nnU-Net / TransUNet / Swin-Unet`。
- 只给通用外部对比，却没有同步准备任务内 gland-specific direct comparison 区域。

### 3.3 为什么当前阶段必须拆成多层阶段门

`08_外部对比` 最容易发生的错误是把“几个外部方法能跑”误写成“外部主表层已经成立”。

因此，当前阶段必须把放行条件拆成：

- 主线冻结门：负责确认当前比较建立在已唯一化的 `current_mainline` 上。
- 公平协议门：负责确认数据、训练、评估和记录规则真正统一。
- 运行完整门：负责确认首批外部方法最小复现闭环与必要时的三 seed 补证已齐。
- 来源边界门：负责确认 `reproduced / † / *` 与 direct comparison 准入边界已经固定。
- 主表形成门：负责确认外部主结果表、任务内 direct comparison 草稿和增强外部进入结论已成型。
- 交接门：负责确认 `09_CRAG验证`、`10_结果汇总` 与 `11_总验收与止损` 已可直接读取本阶段资产。

## 4. 设计边界(做什么/不做什么)

### 4.1 当前阶段直接继承哪些冻结项

本文件直接继承并不得擅自改动的上位规则如下：

- 主任务：`2D gland segmentation`
- 主数据集：`GlaS`
- 正式 split：`train68 / val17 / testA60 / testB20`
- 标签口径：`mask > 0`
- 输入通道：`RGB`
- 输入尺寸：`512 x 512`
- 归一化：`ImageNet mean/std`
- 主增强包：沿用当前主线已冻结版本
- 主损失：`L_seg = L_BCE + L_Dice`
- 优化协议：`AdamW + lr 1e-3 + weight_decay 1e-4 + ReduceLROnPlateau + epoch_max 120 + early_stopping 20 + AMP on`
- 评估链：`logits -> sigmoid -> threshold -> binary mask -> connected components -> metrics`
- 主指标优先级：`F1 / Object Dice / Object Hausdorff`
- 补充指标：`Dice / IoU / HD95 / Boundary F1`
- `best_selector = val_objdice_max`
- `eval_cast_policy = logits/probabilities must be kept or cast to float32 before thresholding`
- `threshold_source = val17`
- `boundary_metric_width = 3 px`
- `boundary_metric_impl = project_custom_erosion_xor_3x3_ones_8conn_border0_tol3px`
- `connected_components_impl = scipy.ndimage.label`
- `connected_components_connectivity = 8`
- `train_proto_version = train_proto_v1`
- `eval_proto_version` 由 04 阶段当前轮 Gate 冻结
- `postprocess_version`：当前轮待由研究定标、阶段锁定、Pre-check/Gate 冻结；`none_in_v3` 仅作历史 lineage，不是当前值。
- `tta_version = none_in_v1`
- 正式种子集合：`3407 / 1234 / 2025`
- `Distance` 阶段形成的正式结论、handoff 资产和 `current_mainline` 上游语义必须完整继承

### 4.2 当前阶段唯一合法起点

外部对比阶段不重新定义主模型，而是由 `Distance` 阶段正式结论唯一决定比较起点：

```text
current_mainline = (distance_decision_level == keep) ? distance_kept_base : distance_input_base
```

这意味着：

- 只有 `07_Distance` 已形成正式 `keep / downgrade / drop` 结论并完成 handoff，`08_外部对比` 才允许启动。
- `downgrade` 与 `drop` 都不能被偷换成“继续把 `Distance` 版本默认当主线”。
- 任何正式外部对比 run 都必须能追溯到 `Distance` 的 decision note、stage manifest 和 `current_mainline` 定义。

### 4.3 当前阶段固定比较名单与唯一变量

当前阶段默认只允许先完成首批三类通用外部方法：

```text
External_core = {UNet++, DeepLabV3+, Attention_U-Net}
```

本阶段唯一允许变化的核心变量只有“在完全继承 `current_mainline` 数据、训练、评估和记录协议的前提下，替换外部架构本体并记录最小必要差异”：

```text
external_run = unified_data_proto + unified_train_proto + unified_eval_proto + external_architecture_only
```

```text
allowed_gap = minimal_necessary_gap(external_architecture)
```

本阶段明确不允许借机改动：

- 数据协议、split、输入尺寸和归一化
- 主增强包、主损失、optimizer、scheduler 和训练轮数口径
- `best checkpoint` 选择器、阈值来源、后处理与 TTA 默认口径
- `current_mainline` 的结构与模块保留结论
- 把第三方仓库默认的测试脚本、私有后处理或多阶段训练流程直接混入统一主表
- 在首批外部层未收口前，默认扩展到 `nnU-Net / TransUNet / Swin-Unet`

### 4.4 当前阶段正式阶段门

当前文件固定采用下面的阶段门语义：

```text
Gate_H1 = current_mainline_ready
          AND fairness_protocol_fixed
          AND core_external_runs_complete
          AND source_boundary_ready
          AND external_table_ready
          AND handoff_ready
```

各子门定义如下：

- `current_mainline_ready`：`Distance` 正式结论已发布，`current_mainline` 唯一合法，且当前阶段所有配置都引用同一主线说明。
- `fairness_protocol_fixed`：`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/01_统一公平协议.md` 中的数据、训练、评估和记录规则已落入实现与结果表，没有出现方法间协议漂移。
- `core_external_runs_complete`：`UNet++ / DeepLabV3+ / Attention U-Net` 已完成最小闭环；若接近主线或排名第二，已完成三 seed 补证。
- `source_boundary_ready`：`reproduced / † / *`、`single_seed / mean +- std`、direct comparison 准入和不可直比情形已正式写清。
- `external_table_ready`：外部主表草稿、任务内 direct comparison 草稿和增强外部进入结论已经成型。
- `handoff_ready`：`09_CRAG验证`、`10_结果汇总` 和 `11_总验收与止损` 已能直接读取当前阶段资产，不再需要口头解释。

### 4.5 三 seed 补证触发条件

首批外部对比默认先允许最小闭环，但以下情形必须升级为三 seed：

```text
need_three_seed = (gap_to_mainline <= 1.5pt) OR (rank == 2)
```

解释如下：

- 当某个外部方法与 `current_mainline` 差距足够小，单 seed 容易夸大或低估真实差距，必须补三 seed。
- 当某个外部方法位列第二强，即使与主线差距略大，也会直接影响主表排序与写作表述，必须补三 seed。
- 若方法明显落后且仅承担“首批覆盖层”角色，可先保留单 seed，但必须显式标注 `aggregation = single_seed`。

## 5. 如何判断设计有效(验收标准)

下面所有核心规则都写成最终协议格式。后面写代码时，只允许把这些规则实现出来，不允许擅自补新的隐含规则。

### 5.1 `current_mainline` 唯一起点规则

- 当前结论：外部对比阶段只允许从 `Distance` 正式结论导出的唯一 `current_mainline` 出发，不允许重新选择或混用主线
- 规则类型：`handoff 冻结规则 + 阶段起点规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/00_阶段总协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/07_Distance/04_保留或降级标准.md`
- 核心公式或定义参考：`current_mainline = (distance_decision_level == keep) ? distance_kept_base : distance_input_base`
- 采用原因：如果起点不唯一，后面的外部比较、公平协议和下游主表都会失去统一参照物
- 不采用的相邻方案：不采用“哪个主线看起来更强就先拿哪个比”；不采用把 `distance_kept_base` 和 `distance_input_base` 同时写进同一层主表
- 代码落点：crc_gland_segmentation_project/configs/experiment/external_h1_v1.yaml、crc_gland_segmentation_project/scripts/run_external.py、crc_gland_segmentation_project/reports/stage_reports/external_stage_summary.md

- 验收方式：检查所有正式 run 和汇总表都只引用同一个 `current_mainline` 字段值

### 5.2 统一公平协议规则

- 当前结论：所有正式外部 run 都必须遵守“统一数据协议 + 统一训练协议 + 统一评估协议 + 仅替换外部架构”的最小公平比较框架

- 核心公式或定义参考：`external_run = unified_data_proto + unified_train_proto + unified_eval_proto + external_architecture_only`
- 采用原因：只有把比较压到“只换结构”这一层，外部结果才具备进入主表的公平性
- 不采用的相邻方案：不采用第三方仓库原生训练脚本直接横比；不采用方法间各用一套阈值、后处理或测试时增强
- 代码落点：crc_gland_segmentation_project/scripts/run_external.py、crc_gland_segmentation_project/src/external/adapters/、crc_gland_segmentation_project/src/eval/run_eval.py

### 5.3 首批外部名单冻结规则

- 当前结论：首批通用外部对比名单固定为 `UNet++ / DeepLabV3+ / Attention U-Net`
- 规则类型：`路线冻结规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/02_路线与投稿/02_结直肠腺体分割_分阶段实验路线与执行标准.md`、`结直肠腺体分割_plan_优化版/03_文献证据/01_经典基线与对比方法/04_UNet++.md`、`结直肠腺体分割_plan_优化版/03_文献证据/01_经典基线与对比方法/03_DeepLabV3+.md`、`结直肠腺体分割_plan_优化版/03_文献证据/01_经典基线与对比方法/01_Attention_U-Net.md`

- 采用原因：这三类方法分别覆盖强改进 U 型、语义上下文解码和轻量注意力增强三条代表性外部路线，足以构成第一层主表外部层

- 验收方式：检查首批主表草稿完整覆盖三类方法，且没有未经触发就加入增强外部方法

### 5.4 最小必要差异记录规则

- 当前结论：任何外部方法如果无法完全复刻统一协议，只允许保留实现所必需的最小差异，并必须逐项留痕
- 规则类型：`工程冻结规则 + 来源记录规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/01_统一公平协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/05_代码工程映射与实现策略.md`
- 核心公式或定义参考：`allowed_gap = minimal_necessary_gap(external_architecture)`
- 采用原因：真实世界的外部适配难免存在少量实现差异，但只有把差异压到最小并记录清楚，结果才可解释
- 不采用的相邻方案：不采用“仓库默认就是这样”作为免责；不把结构依赖、训练技巧和测试后处理一起打包进未说明差异

- 运行记录字段：`adaptation_note`, `paper_setting_gap`, `fairness_adjustment`, `implementation_trust_level`
- 验收方式：检查每个正式外部方法都有最小必要差异说明，且没有未记录的重要协议偏移

### 5.5 三 seed 补证规则

- 当前结论：当外部方法与主线差距足够小或排名第二时，必须从单 seed 升级为三 seed 聚合
- 规则类型：`结果稳健性规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/07_阶段验收.md`、`结直肠腺体分割_plan_优化版/02_路线与投稿/03_结直肠腺体分割_投稿层级自查与止损判断.md`
- 核心公式或定义参考：`need_three_seed = (gap_to_mainline <= 1.5pt) OR (rank == 2)`
- 采用原因：接近主线或处于次优位置的方法会直接影响排序解释和写作说服力，不能只凭单 seed
- 不采用的相邻方案：不采用“某次跑得好看就直接进主表”；不采用对所有明显落后方法机械补三 seed 浪费资源
- 代码落点：crc_gland_segmentation_project/scripts/run_external.py、crc_gland_segmentation_project/scripts/summarize_stage.py
- 运行记录字段：`aggregation`, `seed_list`, `gap_to_mainline`, `rank_in_external_group`
- 验收方式：检查所有触发条件命中的方法均产出 `mean +- std` 聚合结果

### 5.6 来源标记与 direct comparison 准入规则

- 当前结论：`reproduced / † / *` 必须与 direct comparison 准入边界一起固定，只有同 benchmark、同 split、同 metric protocol 且无重大测试差异时才允许写成直接横比
- 规则类型：`来源边界规则 + 写作边界规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/06_引用值与复现值标记规则.md`
- 核心公式或定义参考：`direct_comparison_eligible = same_benchmark AND same_split AND same_metric_protocol AND no_major_test_gap`
- 采用原因：如果来源标记和直比资格不分开，主表很容易把引文值误写成可公平横比的复现值
- 不采用的相邻方案：不采用 benchmark 不一致还写 direct comparison；不采用对引文值隐藏 `† / *` 标记
- 代码落点：crc_gland_segmentation_project/reports/tables/external_main_table_draft.csv、crc_gland_segmentation_project/reports/tables/task_specific_direct_comparison_draft.csv、crc_gland_segmentation_project/scripts/validate_sources.py
- 运行记录字段：`result_source_type`, `quoted_from_paper`, `direct_comparison_eligible`, `has_special_test_setting`
- 验收方式：检查所有主表条目都具备来源标记；检查 direct comparison 草稿中不存在不满足准入条件的条目

### 5.7 增强外部进入条件规则

- 当前结论：`nnU-Net / TransUNet / Swin-Unet` 只允许在首批外部层收口后，且说服力和资源条件同时满足时进入
- 规则类型：`止损规则 + 扩展准入规则`
- 适用阶段：`08_外部对比`
- 直接依据：`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/05_增强对比进入条件.md`、`结直肠腺体分割_plan_优化版/01_实验执行/08_外部对比/07_阶段验收.md`
- 核心公式或定义参考：`Gate_H1_plus = current_mainline_ready AND core_external_ready AND external_main_table_ready AND task_specific_direct_ready AND persuasion_gap_still_exists AND resource_budget_ok AND implementation_trustworthy AND fairness_adaptable`
- 采用原因：增强外部方法成本更高、实现更复杂，必须先证明首批外部层已经闭环且确有补强必要
- 不采用的相邻方案：不采用“为了表更满”而无条件扩方法；不采用实现来源不可信还强行加入
- 代码落点：[project_root]/notes/plus_external_entry_decision.md、crc_gland_segmentation_project/configs/external/extra/、crc_gland_segmentation_project/scripts/plan_external_plus.py
- 运行记录字段：`plus_external_candidate`, `entry_decision`, `resource_budget_ok`, `implementation_trustworthy`, `need_more_persuasion`
- 验收方式：检查增强外部是否有正式进入决议；若未进入，也必须留下明确的“不进入”结论

## 7. 回退条件

### 7.1 独立回退触发条件

只要出现下面任意一条，本阶段都必须先回退修正，而不是直接进入 `09_CRAG验证`：

- `Distance` 尚未形成唯一合法 `current_mainline`，却已经启动正式外部比较
- 不同外部方法实际使用了不同数据、训练、评估或记录协议，却被写进同一主表
- `implementation_source`、`adaptation_note`、`paper_setting_gap` 或 `fairness_adjustment` 缺失，导致最小必要差异无法追溯
- `reproduced / † / *` 标记与结果来源不一致，或 direct comparison 准入条件未满足却仍写成直接横比
- 首批外部层尚未收口，就提前扩展 `nnU-Net / TransUNet / Swin-Unet`
- 主表草稿、stage manifest、阶段总结和下游 handoff 之间对同一方法的资格判断不一致
- `TestA / TestB` 被混写或聚合，导致结果口径与上游评估协议冲突

### 7.2 固定回退顺序

回退时统一按下面顺序排查：

1. 检查 `current_mainline` 是否唯一、是否来自 `Distance` 正式 handoff。
2. 检查统一公平协议是否真的落入配置、训练、评估和结果记录。
3. 检查每个外部方法的实现来源、最小必要差异和运行字段链是否完整。
4. 检查 `reproduced / † / *`、direct comparison 准入与主表资格是否一致。
5. 只有前面都确认无误后，才允许讨论是否进入增强外部或直接止损收口。

### 7.3 回退后的重新放行条件

发生回退后，只有同时满足下面条件，才允许重新声明 `08_外部对比` 阶段通过：

- 问题来源已经按固定回退顺序定位并修复
- 修复动作、影响范围和复验结果已写入 external_reproduction_note.md 或阶段总结
- external_main_table_draft.csv、task_specific_direct_comparison_draft.csv、external_stage_manifest.csv 与文字结论重新对齐
- 当前阶段重新满足第 `5` 节全部规则卡片与第 `6` 节的最低交付物、交接资产和下游起点要求
