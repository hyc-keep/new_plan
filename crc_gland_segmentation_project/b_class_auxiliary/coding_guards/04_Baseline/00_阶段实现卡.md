## 3. 当前阶段唯一目标

- 用一句话说清: 在原始计划的数据、统一学习率、batch、loss、scheduler、selector、threshold、seed 和评估全部不变时，只将 plain UNet encoder 升级为 ResNet34 residual encoder，建立 A2/B1 成对结构 baseline。
- 当前阶段进入条件: 原始协议复现研究定标结论为 allow_stage_lock，`b_class_auxiliary/coding_guards/04_Baseline/research_alignment_gate_report.md` 为 pass，v1/v2/v3 均保留为历史真实轮次，未修改任何结果文件。
- 上一阶段已经交付: 历史 v1/v2/v3 资产均保留其真实状态；本轮不把历史结果当作当前 fresh run，也不修改其 Gate。

## 4. 为什么现在做这个,而不是下一个阶段

- 当前阶段必须先解决的阻断: canonical 配置和旧阶段证据曾残留 v3 训练引用与身份，必须先完成原始协议 config、contract、lineage 和 freshness 闭合。
- 如果现在提前做下一个阶段,会破坏什么: 直接加入 LKMA、Boundary 或 Distance 会把 baseline 协议与模块效应混合，无法归因。
- 下一个阶段此刻还不允许进入的原因: 历史 Gate_B1=false、正式 handoff=false；本轮必须先完成六个 fresh run、独立复核和当前 Gate。

## 5. 当前阶段允许改动 / 禁止改动

- 明确允许改: 六份 A2/B1 experiment config 的原始协议身份和训练入口引用；`train_proto_v1`、`eval_proto_v1`、稳定 run_name、seed 映射、contract lineage 和 current-round freshness；隔离 runtime、smoke 和对应真实证据。
- 明确禁止改: 不使用 v2/v3 差分学习率或 BN policy；不替换 BatchNorm 为 GroupNorm；不增加 gradient accumulation；不改 physical batch=2、统一 lr=1e-3、AdamW、weight decay、BCE+Dice、augmentation、scheduler、early stopping、max epoch、AMP、best selector、threshold、eval protocol、seed、split、病例、指标、统计公式或 Gate；不混入 LKMA/Boundary/Distance/TTA/后处理；不复用历史结果冒充当前轮。
- 如果越界,会破坏哪份正式协议: 会破坏 `结直肠腺体分割_plan_优化版/01_实验执行/04_Baseline/02_训练协议.md`、`结直肠腺体分割_plan_优化版/01_实验执行/04_Baseline/03_对比与判断规则.md`、`结直肠腺体分割_plan_优化版/01_实验执行/04_Baseline/04_阶段验收.md` 的原始协议与公平比较边界。
