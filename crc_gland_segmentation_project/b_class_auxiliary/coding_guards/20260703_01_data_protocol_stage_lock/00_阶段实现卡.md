## 3. 当前阶段唯一目标
- 用一句话说清: 把 `01_数据协议` 当前应执行的唯一动作锁定为“恢复并冻结正式输入层”，并把缺失正式 GlaS 与 CRAG 主数据资产这一事实写成后续工程必须正面处理的正式阻塞
- 当前阶段进入条件: 研究定标记录已形成，`研究结论状态 = allow_stage_lock`，且 research_alignment_gate_report 已经是 pass
- 上一阶段已经交付: 研究阶段已经证明当前最重要的问题不是“还没研究清楚”，而是“当前工作区缺少计划声明的正式主数据资产”，因此允许进入阶段锁定

## 4. 为什么现在做这个,而不是下一个阶段
- 当前阶段必须先解决的阻断: 当前还没有把“正式输入层未就位”写成唯一工程目标和唯一允许改动边界；如果不先锁定，后续很容易把恢复数据资产、生成 split、写配置和开始训练混在一起
- 如果现在提前做下一个阶段,会破坏什么: 会破坏 `01_数据协议` 作为正式输入层 gate 的角色，让 `UNet` 阶段背负本应由数据阶段解决的目录、split、标签和交接问题，导致后面所有模型对比失去同一输入层前提
- 下一个阶段此刻还不允许进入的原因: Pre-check 之前必须先有一张正式阶段卡，把当前唯一目标、允许改动、禁止改动、未来工程落点和最小验证计划写死；否则 Pre-check 只能检查一堆漂移中的假设，而不是检查被锁定的阶段边界

## 5. 当前阶段允许改动 / 禁止改动
- 明确允许改: 当前阶段只允许在后续编码阶段处理正式数据资产恢复、split 清单生成、数据配置、数据检查输出、数据预览输出、来源说明与相应工具脚本；本轮阶段锁定本身只允许新增当前阶段卡和阶段门禁报告
- 明确禁止改: 不允许提前进入 Pre-check 四件套，不允许写训练入口、不允许改模型结构、不允许改 loss/optimizer/scheduler、不允许改评估实现、不允许写运行证据、不允许写学习型说明文
- 如果越界,会破坏哪份正式协议: 会破坏 `结直肠腺体分割_plan_优化版/01_实验执行/00_总览与规范/00_执行导航.md` 规定的“先数据协议、再 UNet”的固定顺序，也会破坏 `crc_gland_segmentation_project/.trae/skills/制度完成定义.md` 对阶段锁定完成的定义

## 9. 当前未决问题与阻断项
- 当前仍未解决: 当前正式 GlaS 与 CRAG 主数据目录究竟是被清理后尚未拷回、还是实际位于其它路径仍未声明；当前本地整理版 CRAG 的最终固定目录命名和来源说明文本也还没有在工程区正式落盘
- 如果这些问题不解,后续会卡在哪: 会卡在 Pre-check 之后的正式编码入口，因为没有真实主数据资产就无法合法生成正式 split、配置、检查资产、预览资产和交接资产；即使勉强写代码，也无法证明后续训练到底在读取哪一版正式输入层
- 本轮是否允许继续进入 Pre-check: `是`；因为阶段锁定的职责是把目标、边界和阻塞锁死，而不是现在就解决所有工程问题。只要阶段门禁通过，就允许进入 Pre-check 去真实审查代码库现状、前置依据和未来工程落点

- 阶段锁定状态: `allow_precheck`
