# 02_UNet流程验证 操作说明（如何运行本阶段）

> 目的：让任何人不看源码也能把 02 阶段（单 seed UNet 完整训练→测试→评估→可视化）跑一遍。
> 所有命令都在项目根目录执行：`cd /home/featurize/work/Paper/crc_gland_segmentation_project`

## 一、这个阶段在做什么

用**单个 seed（3407）**跑通完整 UNet 训练链，验证「数据→模型→loss→反向→优化→验证→测试→评估→可视化→记录」整条流程在单 GPU + AMP 下真实可跑。产出正式 run：`experiments/A1_UNet_GlaS_v1_seed3407/`。

这是「流程验证」，不是「稳定性验证」（多 seed 是 03 阶段的事）。

## 三、配置引用关系（重要）

`configs/experiment/A1_UNet_GlaS_v1_seed3407.yaml` 是总入口，它引用：
- `configs/model/unet_v1.yaml` —— 模型结构
- `configs/train/unet_flow_v1.yaml` —— 训练超参（epoch、patience、lr 等）

改任何超参都改对应子配置，不要散着改。

## 四、完整执行流程与命令

### 步骤 0（可选）：本地 smoke check（先验证能跑，不出正式结果）

```bash
python scripts/train.py --config configs/experiment/A1_UNet_GlaS_v1_seed3407.yaml --device cuda --smoke-check
```
- `--smoke-check`：只跑最小训练循环（1 epoch 级），验证 dataloader/forward/loss/backward/optimizer.step 通路。
- 产物落在独立 smoke 目录，不污染正式 run。**确认无报错后再跑正式训练。**

### 步骤 1：正式训练（单 GPU + AMP，从头 fresh run）

```bash
python scripts/train.py --config configs/experiment/A1_UNet_GlaS_v1_seed3407.yaml --device cuda
```
- `--config`：**必填**。
- `--device`：默认 `cuda`（不可用自动回退 CPU，但服务器务必保持 cuda）。
- `--smoke-check`：**不要加**，加了就只是 smoke 不是正式训练。
- `--resume-from-last`：仅在训练中断需续跑时加，从 `checkpoints/last.ckpt` 恢复；正常 fresh run 不加。
- 产物：`experiments/A1_UNet_GlaS_v1_seed3407/` 下 `checkpoints/best.ckpt`、`train_log.csv`、`val_metrics.csv`、`run_meta.yaml`。
- 早停：patience=20，正常在 best_epoch 后 20 轮停。

### 步骤 2：测试评估（加载 best.ckpt，跑 testA/testB）

```bash
python scripts/test.py --config configs/experiment/A1_UNet_GlaS_v1_seed3407.yaml --device cuda
```
- `--config`：**必填**。
- `--checkpoint`：默认 `None`（自动加载该 run 的 `checkpoints/best.ckpt`）；要评估特定权重才传。
- `--max-visual-samples`：每个 split 导出多少张最差案例可视化，默认 `5`。
- `--skip-visuals`：只要指标、不要可视化时加。
- `--max-samples-per-split`：默认 `None`（全量）；仅本地 CPU 快速联调时限制样本数。
- 产物：`testA_metrics.csv`、`testB_metrics.csv`、`metric_crosscheck_note.md`、`summaries/`、可视化。输出应含 `metric_crosscheck_result=pass`。

### 步骤 3：单 run 阶段汇总（A1 用法，不带 --stage a2）

```bash
python scripts/summarize_stage.py --config configs/experiment/A1_UNet_GlaS_v1_seed3407.yaml
```
- `--config`：**必填**，单 run 汇总模式（**注意**：不带 `--stage a2`，那是 03 阶段的多 seed 聚合）。
- `--run-name`：默认从 config 读，一般不用传。
- `--debug-note` / `--stage-summary-output` / `--manifest-output`：都有默认路径，一般不用改。

## 五、跑完怎么确认成功

- `experiments/A1_UNet_GlaS_v1_seed3407/run_meta.yaml`：`smoke_check=false`、`stop_reason=early_stopping`、`device=cuda`、`amp_active=true`
- `metric_crosscheck_result=pass`（sample_mean 与 aggregate 一致）
- 正式 A1 参考结果：testA objdice≈0.6949、testB objdice≈0.7510（重跑因环境略有浮动属正常）

## 六、常见坑

- **smoke 与正式 run 别混**：`--smoke-check` 产出的是 smoke 目录，不是正式 A1，看结果前先确认 `run_meta.yaml` 里 `smoke_check=false`。
- **device 默认 cuda**：服务器别手贱改成 cpu，会慢 40 倍且不报错。
- **summarize_stage 两种模式**：02 阶段用 `--config`（单 run）；03 阶段用 `--stage a2`（多 seed 聚合）。别用错。
- 命令必须在 `crc_gland_segmentation_project/` 目录下跑。
