## 4. P04-001 的事实记录

### 4.1 已确认数值

当前 TestB Object Dice 的稳定性比较为：

```text
B1 std = 0.014335403187570029
A2 std = 0.012146
差值约为 0.002189
```

当前 B1 的 Object Dice 均值高于 A2；Object F1 稳定性已通过；其他主要对象级指标方向没有发现同类阻断。

### 4.2 已完成的排除审计

已确认：

1. 三个 B1 seed 使用相同 TestB 样本集合，样本 ID 完全对齐。
2. `eval_proto_v1`、`current_standard`、阈值和连通域口径一致。
3. 正式 prediction PNG、GT PNG 与逐样本 CSV 已独立复核，三 seed 两个 split 均无 mismatch。
4. Object Dice 已使用实例标签化后的参考公式逐样本复算，最大误差约为数乘 `1e-12` 量级。
5. 三个正式 run 的 best epoch、早停和 checkpoint 选择没有发现异常：best epoch 为 47/47/48，均按 `val_objdice_max` 选择并正常早停。
6. 当前日志没有证明学习率、早停或 checkpoint 选择是 P04-001 的确定根因，因此不能凭猜测修改这些因素。

### 4.3 科学解释

P04-001 只能支持以下结论：

> 在三个预先固定的 seed、当前 TestB 和 `ddof=0` 统计规则下，B1 的 TestB Object Dice 离散度略高于 A2。

不能扩写为：

- B1 模型完全失败；
- B1 不可复现；
- B1 的评估代码错误；
- B1 所有指标均不稳定；
- 未来任意 seed 都会失败。

## 6. 后续实验停止与回退条件

出现以下任一情形时，停止后续主线，回退到 04 做受控稳定性迭代：

1. 后续模块的均值增益小于或接近 B1 的 seed 间波动。
2. 后续模块在三个 seed 上改进方向不一致。
3. Object F1 或 Object Dice 均值相对 B1 明显下降。
4. TestB 的主要指标出现明显退化。
5. 后续结果无法区分模块收益与 B1 的训练随机性。
6. 后续模块改变了当前锁定的数据、split、评估协议、阈值或统计口径。
7. 发现当前 B1 的身份、checkpoint、预测资产或指标实现存在新的实质错误。

### 6.1 回退后的执行方式

1. 先归档当前 B1、配置、汇总、Gate 和相关流程证据。
2. 生成 `archive_manifest.md`，标记 `historical_archive_only=true`、`valid_for_current_gate=false`。
3. 保持正式科学对象使用标准 run_name，不在正式对象名中写 `v1/v2/v3`。
4. 新轮次只改变一个可检验变量，继续使用预先固定的三个 seed，除非新统计设计已正式冻结。
5. 重新执行 stage contract、runtime、smoke、正式训练、测试、独立复核、summary 和 Gate。
6. 新轮次结果不得覆盖本轮 B1。

## 7. 论文写作边界

### 7.1 baseline 结果段

论文应报告：

- 三个固定 seed 的逐 seed 结果；
- mean ± std；
- TestA/TestB 分开统计；
- 主要对象级、像素级、边界级和距离指标；
- 与 A2 的公平比较；
- Object Dice stability warning。

推荐事实表述：

> Under the same evaluation protocol, the ResNet34-U-Net baseline improved the mean object-level performance over A2 on the GlaS test subsets. However, the TestB Object Dice variability across the three fixed seeds was slightly higher than that of A2, indicating a minor residual sensitivity to training randomness.

### 7.2 discussion / limitation 段

应说明：

- 三 seed 是预先固定的重复设计，而不是事后挑选。
- std 描述当前三个训练随机状态的离散程度，不是显著性检验或所有未来复现的保证。
- 当前 warning 不改变正式 PNG/GT 指标复核和主要均值结果的有效性。
- 后续模块是否改善稳定性，需要用相同 seed、相同协议和逐 seed 结果进一步判断。

不得写：

- “B1 在所有稳定性指标上均优于 A2”；
- “B1 完全稳定”；
- “B1 的提升已被统计显著性证明”；
- “因为没有通过内部 Gate，所以 B1 结果无效”。

### 7.3 投稿影响边界

P04-001 是当前工程的内部稳定性放行限制，不是文献统一规定的绝对投稿门槛。它需要在论文中透明报告，但单独不能推出论文必然不能投稿或录用。最终投稿质量仍取决于方法贡献、完整消融、外部验证、公平比较、失败案例和结论是否与证据匹配。
