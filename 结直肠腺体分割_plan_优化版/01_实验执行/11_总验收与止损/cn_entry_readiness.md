# CN Entry Readiness

本文件不是中文写稿备忘录，也不是把 03_中文保底条件.md 简单抄一遍的轻量模板。

它在 `11_总验收与止损` 阶段中的唯一职责是：

> 把上游已经冻结的中文保底规则、阶段门槛、止损动作、结果包资产、图像资产和来源边界，正式压成可被下游直接消费的中文起步裁决输出；后续 writing_entry_decision.md 与 final_readiness_judgement.md 只能读取本文件已经填写的中文档位、缺失资产、阻塞项和进入理由，不允许再跳过本文件另行主观改判。

这里的 `CN Entry Readiness` 不是重新定义中文规则，而是把 03_中文保底条件.md 的冻结规则落成真正可填写、可回查、可交接的中文裁决层。它关心的是：当前项目到底能不能开始中文写稿，要不要边补图表，缺什么，卡在哪里，以及这些判断分别由哪份正式证据支撑。

---

## 计划 lineage 与下游 handoff

总验收当前/未来消费必须记录 `source_stage`、`source_manifest`、`source_protocol_version`、`source_run_name`、`consumer_stage`、`consumer_file`、`consumption_boundary`；缺失任一字段或计划/config/run_name/run_meta/manifest/gate/handoff 不同步时总验收 blocked，回到治理/阶段锁定。

## 回退条件

任一阶段 Gate、结果来源、版本链或交接资产不完整时停止总验收，按阶段顺序回退到最早缺口。

## 代码落地接口

总验收入口消费阶段 gate、manifest、summary 和 handoff 资产；只允许对真实产物做状态裁决。

## 冲突裁决记录

当前协议与历史结果冲突时保留历史身份和数字，当前/未来只消费当前冻结 lineage。

## 文件质量自检

检查七字段、全部阶段 Gate、来源边界、结果资产、回退链、代码入口和下游交接。

## Diagnostics 闭环

落盘后复核标题、字段、状态口径和历史边界；问题修复后再复核文档检查。

## 1. 文件角色与执行边界

### 1.1 当前文件负责什么

当前文件只负责冻结下面八件事：

1. 中文稿起步状态只允许填写为 可直接开写 / 可开写但需边补图表 / 暂不开写 三档之一的唯一输出值。
2. 面向下游固定输出 `cn_ready`、`cn_ready_reason`、`cn_missing_assets`、`cn_blockers`、`removed_claims`、`downgraded_claims`、`entry_reason` 的字段边界。
3. 把 03_中文保底条件.md 的中文三档、硬门槛、章节最低可写要求，映射成可直接回填的检查表、判定规则和证据映射表。
4. 把 01_阶段门槛总表.md 的阶段状态、02_止损与回退规则.md 的动作边界、`10_结果汇总` 的正式资产，收束成中文起步的唯一判断入口。
5. 明确哪些缺口属于 `可开写但需边补图表` 的轻缺口，哪些缺口必须把结论打回 `暂不开写`。
6. 把中文裁决结果正式交接给 writing_entry_decision.md 与 final_readiness_judgement.md，避免下游重复解释中文状态。
7. 写清独立 `回退条件`、固定回退顺序和重新放行条件，防止中文写稿带着未收口证据进入下游总裁决。
8. 保留本文件自己的 `代码落地接口`、`冲突裁决记录`、强版 `文件质量自检` 和独立 `Diagnostics 闭环`。

### 1.2 当前文件不负责什么

当前文件不重新定义下面这些内容：

- 中文三档状态本体、中文最低章节要求和中文保底硬门槛本体；这些以 03_中文保底条件.md 为准。
- Pass / Pass with gaps / Not pass 的项目总状态本体；这些以 final_readiness_judgement.md 为准。
- 英文起步、英文降调和英文写稿轨道本体；这些以 04_英文起步条件.md 为准。
- 模块级 retain / downgrade / remove / rollback 动作本体；这些以 02_止损与回退规则.md 为准。
- 正式结果包、表格、图像和 `package_status` 本体；这些以 `结直肠腺体分割_plan_优化版/01_实验执行/10_结果汇总/00_阶段总协议.md` 为准。
- 论文正文表述、语言润色和章节写作风格本体；当前文件只回答“能不能起稿，以及依据是什么”。

### 1.3 为什么当前文件必须独立存在

如果只有 03_中文保底条件.md 而没有当前文件，总验收阶段很容易出现下面这些伪完成状态：

- 中文规则已经写清，但没有真正把唯一档位填出来。
- 主线和表格大体齐了，就口头认为“中文可以先写”。
- `removed_claims`、`downgraded_claims` 和 `entry_reason` 没有正式落盘，下游只剩模糊印象。
- writing_entry_decision.md 与 final_readiness_judgement.md 需要中文状态时，只能重新人工解释而不是读取正式输出。

因此，当前文件必须独立承担“把中文保底规则变成正式裁决产物”的职责。

## 3. 本阶段唯一允许处理的变量

### 3.1 当前文件真正要证明什么

本文件真正要证明的，不是“现在看起来差不多能写中文”，而是：

> 在阶段门槛、止损动作、主线冻结、正式结果包、图像资产、来源边界和章节可落笔性都已经由上游压实后，当前项目可以被正式写成一个唯一中文状态，并伴随正式理由、缺失资产、阻塞项和进入理由，不再留下“靠口头理解继续推进”的灰区。

因此，本文件通过的含义同时包括：

- 中文状态已经从规则层落成结果层。
- 结果层输出已经足以直接交给写稿进入裁决。
- 当前文件可以解释为什么能写、为什么只能边补图表写，或为什么暂时不能写。
- 当前文件可以明确指出哪些 claim 被删、哪些 claim 被降调，而不是让下游自己猜。

### 3.2 当前文件与 `Gate_11`、`Gate_CN` 的关系

本文件不重新定义 `Gate_11` 与 `Gate_CN`，但它是两者落地到中文裁决输出的正式承接层：

```text
Gate_11 = stage_matrix_ready
          AND project_status_fixed
          AND stop_loss_fixed
          AND cn_ready_fixed
          AND en_ready_fixed
          AND writing_entry_fixed
```

```text
Gate_CN = cn_state_fixed
          AND cn_asset_boundary_fixed
          AND cn_chapter_minimum_fixed
          AND cn_rollback_boundary_fixed
          AND cn_downstream_schema_fixed
```

当前文件真正负责消费并落盘的是：

```text
cn_entry_output_ready = cn_ready
                        + cn_ready_reason
                        + cn_missing_assets
                        + cn_blockers
                        + removed_claims
                        + downgraded_claims
                        + entry_reason
```

只有当上游 `cn_ready_fixed` 已经成立，且当前文件把上述输出字段真实填成可交接资产时，中文链路才算真正闭环。

### 3.3 当前文件不允许用什么替代正式裁决

当前明确不允许把下面这些情况当成“中文起步已完成”：

- 只有检查表，没有最终字段输出。
- 只有中文档位，没有 `cn_ready_reason`、`cn_missing_assets` 和 `cn_blockers`。
- 只有“可写”结论，没有 `removed_claims` 与 `downgraded_claims`。
- 只有上游规则复述，没有证据映射和交接对象。
- 只有口头说法“先写中文再补图”，但没有把缺口写进正式字段。

## 4. 阶段门控表达式

### 4.1 当前文件唯一合法输入

本文件不制造新的实验事实，只消费已经冻结的正式资产：

```text
cn_entry_inputs = stage_gate_matrix
                + stop_loss_actions
                + result_package_assets
                + figure_assets
                + source_boundary_assets
                + chapter_writability_evidence
```

其中关键来源固定为：

- `stage_gate_matrix`：各阶段 threshold_passed / handoff_assets_ready / next_stage_ready / stage_pass 以及 `stage11_pass`。
- `stop_loss_actions`：模块层、外部对比层、`CRAG` 层和写作层的 retain / downgrade / remove / rollback 结论。
- `result_package_assets`：`GlaS` 主结果表、主线消融表、第一批外部对比表、`CRAG` 表、summary、`package_status`，以及 seed_reporting_mode / main_seed_set / additional_seed_set / combined_seed_count 的正式记录边界。
- `figure_assets`：主线逐步图、边界放大图、失败案例图、`CRAG` 图像。
- `source_boundary_assets`：reproduced / † / *、表注边界、`CRAG` 角色和局限性边界，以及主 `3-seed` 结论与 supplementary `additional_reporting` 的写作边界。
- `chapter_writability_evidence`：摘要、方法、实验、结论是否能在不补关键实验的前提下直接落笔。
